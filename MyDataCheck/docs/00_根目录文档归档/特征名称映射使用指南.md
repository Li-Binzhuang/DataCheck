## 功能介绍

特征名称映射功能允许您在数据对比前，为指定文件的特征列名添加前缀和/或后缀，从而实现列名统一后再进行对比。

### 适用场景

当两个CSV文件的特征列名不完全一致时，例如：
- 一个文件的列名有前缀（如 `model_age`），另一个没有（如 `age`）
- 一个文件的列名有后缀（如 `age_v1`），另一个没有（如 `age`）
- 列名格式不统一，需要通过添加前缀/后缀来匹配

## 使用步骤

### 步骤1：准备数据文件

确保您有两个需要对比的CSV文件，例如：

**file1.csv（模型特征表）：**
```csv
id,name,age,income,score
1,Alice,25,50000,85
2,Bob,30,60000,90
```

**file2.csv（接口特征表）：**
```csv
id,name,model_age,model_income,model_score
1,Alice,25,50000,85
2,Bob,30,60000,90
```

### 步骤2：上传文件

1. 打开MyDataCheck Web界面
2. 进入"数据对比"页面
3. 上传两个CSV文件

### 步骤3：配置基本参数

设置以下基本参数：
- **主键列索引**：用于匹配两个文件的数据行（如：0表示第一列）
- **特征起始列索引**：特征列从哪一列开始（如：2表示从第三列开始）
- **其他选项**：根据需要勾选（如：转换特征值为数值）

### 步骤4：启用特征名称映射

1. 勾选"启用特征名称映射"复选框
2. 映射配置区域会自动展开

### 步骤5：配置映射参数

根据您的需求配置以下参数：

#### 选择映射文件
- **模型特征表 (file1)**：为第一个文件的特征列添加映射
- **接口/灰度/从库特征表 (file2)**：为第二个文件的特征列添加映射

#### 输入前缀和/或后缀
- **特征列名前缀**：添加到列名前面的字符串（如：`model_`）
- **特征列名后缀**：添加到列名后面的字符串（如：`_v1`）

> 💡 提示：可以只填写前缀、只填写后缀，或者两者都填写

### 步骤6：执行对比

点击"执行对比"按钮，系统会：
1. 读取两个文件
2. 应用特征名称映射
3. 根据映射后的列名进行对比
4. 生成对比报告

### 步骤7：查看结果

对比完成后，您可以：
- 查看实时输出日志
- 下载对比报告（差异记录、特征统计、合并数据）

## 配置说明

### 映射文件选项

| 选项 | 说明 | 适用场景 |
|------|------|---------|
| 模型特征表 (file1) | 为第一个文件添加映射 | file1的列名缺少前缀/后缀 |
| 接口/灰度/从库特征表 (file2) | 为第二个文件添加映射 | file2的列名缺少前缀/后缀 |

### 前缀和后缀

| 参数 | 说明 | 示例 | 结果 |
|------|------|------|------|
| 前缀 | 添加到列名前面 | `model_` | `age` → `model_age` |
| 后缀 | 添加到列名后面 | `_v1` | `age` → `age_v1` |
| 前缀+后缀 | 同时添加 | `api_` + `_new` | `age` → `api_age_new` |

### 映射规则

1. **只影响特征列**：映射只应用于特征列（从`feature_start`开始的列），主键列不受影响
2. **内存操作**：映射在内存中进行，不会修改原始CSV文件
3. **完全匹配**：映射后的列名必须与另一个文件的列名完全一致才能匹配

## 实际案例

### 案例1：为file1添加前缀

**场景描述：**
- file1的列名：`age`, `income`, `score`
- file2的列名：`model_age`, `model_income`, `model_score`
- 需要对比这两个文件

**解决方案：**

1. 勾选"启用特征名称映射"
2. 选择映射文件：`模型特征表 (file1)`
3. 输入前缀：`model_`
4. 后缀留空
5. 执行对比

**映射效果：**
```
file1原始列名：age, income, score
映射后列名：  model_age, model_income, model_score
file2列名：    model_age, model_income, model_score
结果：        ✅ 完全匹配，可以正常对比
```

### 案例2：为file2添加后缀

**场景描述：**
- file1的列名：`age_v1`, `income_v1`, `score_v1`
- file2的列名：`age`, `income`, `score`
- 需要对比这两个文件

**解决方案：**

1. 勾选"启用特征名称映射"
2. 选择映射文件：`接口/灰度/从库特征表 (file2)`
3. 前缀留空
4. 输入后缀：`_v1`
5. 执行对比

**映射效果：**
```
file1列名：    age_v1, income_v1, score_v1
file2原始列名：age, income, score
映射后列名：  age_v1, income_v1, score_v1
结果：        ✅ 完全匹配，可以正常对比
```

### 案例3：同时添加前缀和后缀

**场景描述：**
- file1的列名：`age`, `income`, `score`
- file2的列名：`api_age_new`, `api_income_new`, `api_score_new`
- 需要对比这两个文件

**解决方案：**

1. 勾选"启用特征名称映射"
2. 选择映射文件：`模型特征表 (file1)`
3. 输入前缀：`api_`
4. 输入后缀：`_new`
5. 执行对比

**映射效果：**
```
file1原始列名：age, income, score
映射后列名：  api_age_new, api_income_new, api_score_new
file2列名：    api_age_new, api_income_new, api_score_new
结果：        ✅ 完全匹配，可以正常对比
```

## 常见问题

### Q1：映射后找不到匹配的特征列怎么办？

**A：** 请检查以下几点：
1. 映射后的列名是否与另一个文件的列名完全一致（包括大小写）
2. 前缀和后缀是否输入正确（注意下划线等特殊字符）
3. 是否选择了正确的映射文件（file1或file2）

**示例：**
```
错误配置：
  file1列名：age
  前缀：model（缺少下划线）
  映射后：modelage
  file2列名：model_age
  结果：❌ 不匹配

正确配置：
  file1列名：age
  前缀：model_（包含下划线）
  映射后：model_age
  file2列名：model_age
  结果：✅ 匹配
```

### Q2：可以为两个文件都添加映射吗？

**A：** 目前只支持为一个文件添加映射。如果两个文件的列名都需要调整，建议：
1. 先处理其中一个文件的列名
2. 或者在上传前预处理CSV文件的列名

### Q3：映射会影响主键列吗？

**A：** 不会。映射只影响特征列（从`feature_start`开始的列），主键列和主键列之前的列都不受影响。

### Q4：映射会修改原始文件吗？

**A：** 不会。映射只在内存中进行，不会修改原始CSV文件。您的原始文件保持不变。

### Q5：如何取消映射？

**A：** 取消勾选"启用特征名称映射"复选框即可恢复默认行为（不使用映射）。

### Q6：前缀和后缀都为空会怎样？

**A：** 系统会显示警告提示，并跳过映射，使用原始列名进行对比。

### Q7：映射配置可以保存吗？

**A：** 可以。点击"保存配置"按钮，映射配置会保存到`data_comparison/config.json`文件中，下次可以通过"加载配置"按钮恢复。

## 注意事项

### ⚠️ 重要提示

1. **列名大小写敏感**
   - 映射后的列名必须与目标文件的列名完全一致
   - 包括大小写、空格、特殊字符等

2. **只映射特征列**
   - 主键列不受映射影响
   - 只有从`feature_start`开始的列才会被映射

3. **一次只能映射一个文件**
   - 不能同时为file1和file2添加映射
   - 如果需要，请选择其中一个文件进行映射

4. **前缀或后缀至少填一个**
   - 如果前缀和后缀都为空，映射无效
   - 系统会显示警告并跳过映射

5. **映射不修改原文件**
   - 映射只在内存中进行
   - 原始CSV文件保持不变

### 💡 最佳实践

1. **先测试小文件**
   - 在处理大文件前，先用小文件测试映射配置是否正确

2. **检查列名**
   - 在配置映射前，先查看两个文件的列名
   - 确保映射后的列名能够匹配

3. **保存配置**
   - 配置正确后，记得保存配置
   - 下次可以直接加载，无需重新配置

4. **查看日志**
   - 执行对比时，注意查看日志输出
   - 日志会显示映射前后的列名示例
