# 表关联关系总览

## 一、业务流程总览

```
授信申请 → 授信产品 → 用信申请 → 资产申请 → 借据 → 还款计划
    ↓         ↓          ↓          ↓        ↓       ↓
额度记录   规则执行   额度记录   额度记录  放款流水
```

## 二、核心业务流程

### 2.1 授信流程
1. **授信申请表** (`ods_mx_aprv_approve_credit_apply_df`)
   - 客户提交授信申请
   - 触发规则引擎执行（规则执行记录表）
   - 生成额度记录（额度记录表）

2. **授信产品表** (`ods_mx_aprv_approve_credit_apply_product_df`)
   - 为每个授信申请配置产品方案
   - 关联字段: `credit_apply_id` → 授信申请表的`id`

### 2.2 用信流程
3. **用信申请表** (`ods_mx_aprv_approve_use_credit_apply_df`)
   - 客户使用授信额度
   - 关联字段: `credit_apply_id` → 授信申请表的`id`
   - 关联字段: `asset_id` → 资产申请进件表的`seq_no`

4. **资产申请进件表** (`ods_mx_ast_asset_loan_apply_df`)
   - 生成借款申请
   - 关联字段: `seq_no` ← 用信申请表的`asset_id`
   - 关联字段: `loan_apply_no` → 借据信息表和三方放款流水表

### 2.3 放款流程
5. **借据信息表** (`ods_mx_ast_asset_loan_info_df`)
   - 生成借据
   - 关联字段: `loan_apply_no` ← 资产申请进件表的`loan_apply_no`
   - 关联字段: `loan_no` → 还款计划表

6. **三方放款流水表** (`ods_mx_ast_asset_pay_founder_loan_flow_df`)
   - 记录第三方支付平台的放款流水
   - 关联字段: `loan_apply_no` ← 资产申请进件表的`loan_apply_no`

7. **还款计划表** (`ods_mx_ast_asset_repay_plan_df`)
   - 生成还款计划
   - 关联字段: `loan_no` ← 借据信息表的`loan_no`

### 2.4 额度管理
8. **额度记录表** (`ods_mx_aprv_cust_credit_limit_record_df`)
   - 记录额度变更历史
   - 关联字段: `seq_no` ← 授信申请表的`id`
   - 关联字段: `loan_apply_no` ← 资产申请进件表的`loan_apply_no`

### 2.5 规则引擎
9. **规则执行记录表** (`ods_mx_aprv_approve_rule_record_df`)
   - 记录规则引擎的执行记录
   - 关联字段: `apply_id` ← 授信申请表的`id`

### 2.6 辅助表
10. **账户标签表** (`ods_mx_cust_cust_account_tag_df`)
    - 存储客户标签信息
    - 关联字段: `cust_no` → 其他表的`cust_no`

11. **业务数据存储表增量表** (`ods_mx_allinone_t_kepler_scene_metric_data_df`)
    - 存储业务场景的指标数据
    - 关联字段: `apply_id` → 申请表的`id`

## 三、关键关联关系

### 3.1 一对一关系
- 资产申请 → 借据: 1:1 (100%)
- 资产申请 → 放款流水: 1:1 (100%)
- 借据 → 还款计划: 1:N (平均5.2期)

### 3.2 一对多关系
- 授信申请 → 授信产品: 1:1-2 (平均1.5个产品)
- 授信申请 → 用信申请: 1:1-3 (平均1.25次)
- 授信申请 → 规则执行: 1:6.5 (平均6.5条规则)
- 借据 → 还款计划: 1:4-8 (根据期数)

### 3.3 多对一关系
- 用信申请 → 授信申请: N:1 (转化率87.5%)
- 资产申请 → 用信申请: 1:1 (100%)
