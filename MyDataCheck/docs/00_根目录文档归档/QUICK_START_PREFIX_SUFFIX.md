# 列名前后缀匹配 - 快速开始指南

## 什么是列名前后缀匹配？

当两个 CSV 文件的列名存在前后缀差异时，可以通过指定前缀和后缀来自动匹配列名。

**例子**：
- 模型表列名：`inst_telcel_scat_account_count_7d`
- 接口表列名：`local_all_sms_inst_telcel_scat_account_count_7d_v3`
- 前缀：`local_all_sms_`
- 后缀：`_v3`

## 如何使用？

### 步骤 1：打开 CSV 数据对比页面

在左侧菜单中点击"csv数据对比"

### 步骤 2：上传或指定文件

- 选择"上传文件"或"服务器路径"
- 上传/指定模型特征表（第一个文件）
- 上传/指定接口/灰度/从库特征表（第二个文件）

### 步骤 3：配置列名匹配（可选）

在"列名匹配配置"部分：
- **列名前缀**：输入接口表列名的前缀（如：`local_all_sms_`）
- **列名后缀**：输入接口表列名的后缀（如：`_v3`）

如果不需要前后缀匹配，可以留空。

### 步骤 4：执行对比

点击"执行对比"按钮，系统会自动：
1. 读取模型表的列名
2. 应用前后缀进行转换
3. 在接口表中查找对应的列
4. 对匹配的列进行数据对比

### 步骤 5：查看结果

执行完成后，会生成以下输出文件：
- `差异特征汇总.csv` - 差异特征统计
- `差异数据明细.csv` - 具体差异数据
- `特征统计.csv` - 特征对比统计

## 常见场景

### 场景 1：只有前缀不同

```
模型表: feature_a, feature_b
接口表: api_feature_a, api_feature_b

配置:
前缀: api_
后缀: (留空)
```

### 场景 2：只有后缀不同

```
模型表: feature_a, feature_b
接口表: feature_a_v2, feature_b_v2

配置:
前缀: (留空)
后缀: _v2
```

### 场景 3：前后缀都不同

```
模型表: feature_a, feature_b
接口表: api_feature_a_v3, api_feature_b_v3

配置:
前缀: api_
后缀: _v3
```

### 场景 4：不需要前后缀匹配

```
模型表: feature_a, feature_b
接口表: feature_a, feature_b

配置:
前缀: (留空)
后缀: (留空)
```

## 保存和加载配置

### 保存配置

点击"保存配置"按钮，当前的所有配置（包括列名前后缀）会被保存到 `data_comparison/config.json`

### 加载配置

点击"加载配置"按钮，会自动加载之前保存的配置，包括列名前后缀设置

## 常见问题

**Q: 如果前后缀配置不正确会怎样？**

A: 系统会尝试直接匹配列名。如果直接匹配也失败，该列会被标记为在接口表中不存在，对比结果中会显示为差异。

**Q: 前后缀是否区分大小写？**

A: 是的，前后缀匹配是区分大小写的。请确保前后缀的大小写与实际列名一致。

**Q: 可以同时使用前缀和后缀吗？**

A: 可以。系统会先应用前缀，再应用后缀，最终形成 `前缀 + 原列名 + 后缀` 的形式。

**Q: 如果模型表中的列在接口表中不存在怎样？**

A: 该列的所有数据都会被标记为差异，因为接口表中没有对应的列进行对比。
