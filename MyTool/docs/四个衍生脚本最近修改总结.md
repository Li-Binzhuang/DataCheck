# 四个衍生脚本最近修改总结

## 所有修改（按时间倒序）

### 1. **空值填充-999** 
**修改时间**: 2026-01-28  
**修改内容**: 将特征计算中的空值填充从 `fillna(0)` 改为 `fillna(-999)`

**注释标识**:
```python
# zlf update: 特征值为空时填充-999
```

**修改统计**:
- 第一板块：15次 fillna(-999)
- 第二板块：8次 fillna(-999)
- 第三板块：7次 fillna(-999)
- BOSS板块：31次 fillna(-999)
- **总计**：61次 fillna(-999)

**原因**:
- ✅ 统一空值标识，便于下游模型识别缺失值
- ✅ 区分真实零值与缺失值
- ✅ 提高模型效果

**受影响特征示例**:
- 总查询次数、机构大类计数、占比计算
- 余额汇总、逾期金额、授信额度
- 使用率计算、收入比计算
- 预防类型、责任类型统计

### 2. **cnt特征空值填充0**
**修改时间**: 2026-01-28  
**修改内容**: count类特征保持填充0（不改为-999）

**原因**:
- ✅ count类特征的空值代表"计数为0"，不是缺失值
- ✅ 填充0符合业务逻辑
- ✅ 与其他特征的-999区分开

**受影响特征示例**:
- `*_cnt` - 各类计数特征
- `*_count` - 统计类特征
- `unique_cnt` - 唯一值计数

**注意**: 这类特征在代码中通常使用 `fillna(0)` 或 `fillna(0.0)`

### 3. **输出保留6位小数**
**修改时间**: 2026-01-28  
**修改内容**: CSV输出时对数值特征保留6位小数

**注释标识**:
```python
# zlf update: 对数值特征列保留6位小数
```

**修改代码**:
```python
_features_to_write = features_df.copy()
_round_cols = [c for c in _features_to_write.columns 
               if c not in {"apply_id", "request_time"}]
_features_to_write[_round_cols] = _features_to_write[_round_cols]\
    .apply(pd.to_numeric, errors="coerce").round(6)
_features_to_write.to_csv(csv_path, index=False, encoding="utf-8-sig")
```

**修改统计**:
- 第一板块：已有 round(6)，无需修改
- 第二板块：已有 round(6)，无需修改
- 第三板块：已有 round(6)，无需修改
- BOSS板块：新增 round(6) 处理

**原因**:
- ✅ 统一四个板块的输出格式
- ✅ 避免浮点数特征值被截断
- ✅ 保证特征精度

**受影响特征示例**:
- 均值特征：`*_mean`
- 标准差特征：`*_std`
- 使用率特征：`*_utilization`
- 占比特征：`*_ratio`

### 11. **第三板块total_cnt填充修复** ⭐ 最新
