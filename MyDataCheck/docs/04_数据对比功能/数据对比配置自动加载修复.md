# 数据对比配置自动加载修复

## 问题描述

**用户反馈**: 
- 数据对比配置可以正常保存
- 但是刷新页面后没有自动加载上次保存的配置
- 连续执行两次对比时，取的字段不对

## 问题分析

### 根本原因

**页面加载时没有自动调用配置加载函数**

检查代码发现：
1. ✅ `loadCompareConfig()` 函数存在且逻辑正确
2. ✅ 配置文件已正常保存
3. ❌ **页面加载时没有自动调用该函数**

### 影响

1. **首次访问页面**
   - 使用默认配置（关键列=0，特征起始列=1）
   - 不会加载上次保存的配置

2. **刷新页面后**
   - 配置重置为默认值
   - 上次保存的配置丢失

3. **连续执行对比**
   - 第一次：使用当前页面配置（可能是默认值）
   - 第二次：仍使用默认值（因为没有自动加载）
   - 导致字段取值不正确

## 解决方案

### 修复内容

**文件**: `MyDataCheck/static/js/data-compare.js`

**添加自动加载代码**:
```javascript
// ========== 页面加载时自动加载配置 ==========
document.addEventListener('DOMContentLoaded', function() {
    // 检查是否在数据对比页面
    const comparePage = document.getElementById('page-compare');
    if (comparePage) {
        console.log('[INFO] 页面加载完成，自动加载数据对比配置...');
        // 延迟500ms加载配置，确保页面元素已完全初始化
        setTimeout(() => {
            loadCompareConfig();
        }, 500);
    }
});
```

**优化loadCompareConfig函数**:
- 添加详细的调试日志
- 优化提示信息（添加emoji）
- 记录每个配置项的加载过程

### 工作流程

```
页面加载
    ↓
DOMContentLoaded事件触发
    ↓
检查是否在数据对比页面
    ↓
延迟500ms（确保元素初始化）
    ↓
调用loadCompareConfig()
    ↓
从服务器加载配置文件
    ↓
应用配置到页面元素
    ↓
显示成功提示
```

## 常见问题

### Q1: 刷新页面后配置还是没有加载？

**A**: 检查以下几点：
1. 打开浏览器控制台，查看是否有错误
2. 检查是否有"[INFO] 页面加载完成"日志
3. 检查配置文件是否存在
4. 尝试手动点击"加载配置"按钮

### Q2: 控制台显示"配置文件不存在"？

**A**: 
1. 先保存一次配置
2. 检查文件是否生成：`ls -l MyDataCheck/data_comparison/config.json`
3. 如果文件存在但仍报错，检查文件权限

### Q3: 配置加载成功但页面没有更新？

**A**: 
1. 检查控制台日志，确认配置内容
2. 检查元素ID是否正确
3. 尝试清除浏览器缓存
4. 重启Web服务

### Q4: 为什么延迟500ms加载？

**A**: 
- 确保页面元素已完全初始化
- 避免在元素未就绪时设置值
- 如果仍有问题，可以增加延迟时间
