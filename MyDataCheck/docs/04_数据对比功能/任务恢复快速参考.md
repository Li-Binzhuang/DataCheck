# 任务恢复功能 - 快速参考

## 问题：刷新页面后如何继续看到任务进度？

### 解决方案：任务恢复功能

✅ **已实现** - 刷新页面后自动检测并恢复任务

---

## 使用方法

### 1. 自动恢复（推荐）

```
1. 任务执行中，页面显示 "network error"
2. 刷新页面（F5 或 Cmd+R）
3. 系统自动弹出提示："检测到有正在执行的任务，是否恢复查看？"
4. 点击"确定"
5. 自动显示历史日志 + 实时更新新日志
```

### 2. 手动查看（可选）

```javascript
// 在浏览器控制台执行
showTaskHistory();  // 显示所有历史任务
```

## 工作原理

### 后端
```python
# 1. 创建任务时保存状态
TaskManager.create_task("接口数据对比", "api_comparison")

# 2. 执行过程中更新进度
TaskManager.update_task(task_id, progress=50, total=100)

# 3. 所有输出自动保存到文件
TaskManager.add_log(task_id, "正在处理第100行...")
```

### 前端
```javascript
// 1. 页面加载时检查
checkRunningTasks()  // 查询 /api/tasks?status=running

// 2. 如果有任务，显示提示
confirm("检测到有正在执行的任务，是否恢复查看？")

// 3. 恢复任务
recoverTask(task_id)
  → 加载历史日志
  → 开始轮询新日志（每2秒）
```

## 常见问题

### Q1: 刷新页面后没有弹出恢复提示？

**A**: 可能原因：
1. 任务已经完成
2. 任务被停止
3. Web服务重启（守护线程被终止）

**解决方法**：
```bash
# 检查任务状态
curl http://localhost:5001/api/tasks

# 查看输出文件
ls -lht MyDataCheck/outputdata/api_comparison/ | head -5
```

### Q2: 恢复后看不到新日志？

**A**: 检查：
1. 任务是否还在运行（查看进程）
2. 浏览器控制台是否有错误
3. 轮询是否正常（每2秒应该有请求）

### Q3: 历史日志太多，加载很慢？

**A**: 可以限制加载数量：
```javascript
// 只加载最后1000条
loadTaskLogs(taskId, 1000);
```

### Q4: 如何查看已完成的任务？

**A**: 
```javascript
// 方法1: 在控制台执行
showTaskHistory();

// 方法2: API查询
fetch('/api/tasks?status=completed')
    .then(r => r.json())
    .then(d => console.log(d.tasks));
```
