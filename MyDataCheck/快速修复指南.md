# 🚀 PKL功能快速修复指南

## 问题
- Python 3.15 (alpha) 不支持 pandas
- PKL 解析和转换功能无法使用

## 一键修复（推荐）

### 方法1: 使用自动安装脚本

```bash
cd MyDataCheck
./setup_python312.sh
```

这个脚本会自动：
1. ✅ 检查并安装 Python 3.12
2. ✅ 删除旧的虚拟环境
3. ✅ 创建新的 Python 3.12 虚拟环境
4. ✅ 安装所有依赖（pandas, numpy, Flask等）
5. ✅ 验证安装是否成功

### 方法2: 手动执行（如果脚本失败）

```bash
# 1. 安装 Python 3.12
brew install python@3.12

# 2. 进入项目目录
cd MyDataCheck

# 3. 删除旧虚拟环境
rm -rf .venv

# 4. 创建新虚拟环境
python3.12 -m venv .venv

# 5. 激活虚拟环境
source .venv/bin/activate

# 6. 升级 pip
pip install --upgrade pip

# 7. 安装依赖
pip install -r requirements.txt

# 8. 验证安装
python -c "import pandas; print('✅ pandas版本:', pandas.__version__)"
```

## 安装完成后

启动 Web 服务：

```bash
cd MyDataCheck
source .venv/bin/activate
python web_app.py
```

然后在浏览器访问 `http://localhost:5000`，PKL 功能就可以正常使用了！

## 常见问题

### Q: Homebrew 显示自动更新提示？
**A**: 这是正常的信息提示，可以忽略，不影响安装。

### Q: 安装 Python 3.12 需要多长时间？
**A**: 通常需要 5-10 分钟，取决于网络速度。

### Q: 安装后仍然报错？
**A**: 确保：
1. 虚拟环境已激活（`source .venv/bin/activate`）
2. 重启了 Web 服务
3. 检查 Python 版本：`python --version` 应该显示 3.12.x

### Q: 不想安装 Python 3.12？
**A**: 可以尝试安装 Python 3.11：
```bash
brew install python@3.11
python3.11 -m venv .venv
```
