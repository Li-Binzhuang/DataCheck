# CSV合并功能优化说明

## 问题
1. 点击"执行合并"按钮没有反应
2. 处理大文件（20万行）速度慢，没有进度展示

## 解决方案

### 1. 修复按钮无响应问题
**原因**: JavaScript 调用了未定义的函数 `addOutputLine()` 和 `updateStatus()`

**修复**:
- 将 `addOutputLine()` 替换为 `appendOutput()`（全局函数）
- 添加 `updateMergeStatus()` 函数处理状态更新
- 文件: `static/js/merge-csv.js`

### 2. 大文件处理优化

#### 后端优化 (`web/routes/merge_csv_routes.py`)
- **流式处理**: 使用 SSE (Server-Sent Events) 实时推送进度
- **分块读取**: 纵向合并使用 `chunksize=50000` 分块处理，避免内存溢出
- **临时文件**: 上传文件先保存到临时目录，处理完自动清理
- **进度推送**: 每处理一个文件或每5个块推送一次进度

#### 前端优化 (`static/js/merge-csv.js`)
- **SSE 接收**: 使用 Fetch API 的 ReadableStream 接收服务器推送的进度
- **实时展示**: 解析 SSE 消息，实时更新进度条和输出日志
- **格式化显示**: 行数使用千分位分隔符显示（如 200,000）

## 性能提升

### 纵向合并（追加行）
- **内存占用**: 从一次性加载全部数据 → 每次只加载 5万行
- **处理速度**: 支持百万级数据处理
- **进度展示**: 实时显示处理进度（5% → 100%）

### 横向合并（追加列）
- **内存优化**: 使用 `low_memory=False` 优化 pandas 读取
- **进度展示**: 显示文件读取和合并进度
- **重复列处理**: 支持自动检测或指定列名移除重复列

## 技术细节

### SSE 消息格式
```json
{
  "type": "progress",
  "message": "已处理 50,000 行...",
  "percent": 25
}

{
  "type": "success",
  "output_file": "merged_20250313_143022.csv",
  "total_rows": 200000,
  "total_columns": 10,
  "download_url": "/download/merge_csv/merged_20250313_143022.csv",
  "percent": 100
}

{
  "type": "error",
  "message": "错误信息"
}
```

### 分块处理逻辑
```python
# 第一个块：写入表头
chunk.to_csv(output_path, mode='w', index=False, encoding='utf-8-sig')

# 后续块：追加数据（不写表头）
chunk.to_csv(output_path, mode='a', index=False, header=False, encoding='utf-8-sig')
```

## 测试建议

1. **小文件测试**: 2个文件，各1000行 → 验证基本功能
2. **大文件测试**: 2个文件，各10万行 → 验证性能和进度
3. **横向合并**: 测试重复列移除功能
4. **错误处理**: 测试文件格式错误、网络中断等异常情况

## 注意事项

1. 纵向合并要求所有文件列结构相同
2. 横向合并要求所有文件行数相同
3. 分块大小 `CHUNK_SIZE=50000` 可根据服务器内存调整
4. 临时文件会在处理完成后自动清理
