# 停止Web服务说明

## 方法1：使用停止脚本（推荐）

```bash
cd 场景1_接口数据对比
./stop_web.sh
```

脚本会自动查找并停止占用端口5001的进程。

## 方法2：手动停止

### 查找进程
```bash
# 查找占用端口5001的进程
lsof -ti:5001

# 或者查找web_app.py进程
ps aux | grep web_app.py | grep -v grep
```

### 停止进程
```bash
# 方法1：使用kill命令（优雅停止）
kill $(lsof -ti:5001)

# 方法2：强制停止（如果方法1无效）
kill -9 $(lsof -ti:5001)
```

## 方法3：在前台运行时按Ctrl+C

如果服务是在前台运行的（直接运行 `python3 web_app.py`），可以：
1. 切换到运行服务的终端窗口
2. 按 `Ctrl+C` 停止服务

**注意**：如果服务是在后台运行的（使用 `&` 或 `nohup`），Ctrl+C 无法停止，需要使用方法1或方法2。

## 方法4：使用pkill命令

```bash
# 停止所有web_app.py进程
pkill -f web_app.py

# 强制停止
pkill -9 -f web_app.py
```

## 验证服务已停止

停止后，可以验证端口是否已释放：

```bash
# 检查端口5001是否还被占用
lsof -ti:5001

# 如果没有输出，说明端口已释放
```

## 常见问题

### Q: 为什么Ctrl+C无法停止服务？

**A:** 可能的原因：
1. 服务在后台运行，不在当前终端
2. Flask的debug模式可能影响信号处理
3. 进程被其他进程管理工具管理（如nohup、screen等）

**解决方案**：使用停止脚本或手动kill进程。

### Q: 停止后端口仍被占用？

**A:** 可能是进程没有完全退出，可以：
1. 等待几秒后再次检查
2. 使用 `kill -9` 强制停止
3. 重启电脑（最后手段）

### Q: 如何防止服务意外停止？

**A:** 如果需要在后台长期运行，建议使用：
- `nohup python3 web_app.py &` - 后台运行，忽略挂起信号
- `screen` 或 `tmux` - 终端复用工具
- 系统服务（systemd/supervisor）- 生产环境推荐
