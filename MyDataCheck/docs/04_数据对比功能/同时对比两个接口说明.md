# 同时对比两个接口（接口A和接口B）使用说明

## 📋 需求说明

需要同时对比两批数据：
- **接口A**: 例如生产环境接口
- **接口B**: 例如灰度环境接口

使用同一份输入数据，分别调用两个接口，对比结果。

---

## ✅ 当前支持情况

### 支持的功能

1. ✅ **多场景配置**: 可以在config.json中配置多个场景
2. ✅ **顺序执行**: 系统会依次执行所有启用的场景
3. ✅ **独立输出**: 每个场景有独立的输出文件前缀
4. ✅ **相同输入**: 多个场景可以使用同一份输入文件

### 当前限制

- ⚠️ **顺序执行**: 场景是依次执行的，不是并行执行
- ⚠️ **执行时间**: 如果有N个场景，总时间约为单个场景时间×N

## 🔧 配置方法

### 方法1: 使用配置文件（推荐）

在 `api_comparison/config.json` 中配置两个场景：

```json
{
  "scenarios": [
    {
      "name": "接口A对比",
      "enabled": true,
      "description": "对比生产环境接口A",
      "input_csv_file": "test_data.csv",
      "output_file_prefix": "api_a_result",
      "api_url": "http://10.129.2.25:8081/riskanalysis/Cdc/features",
      "thread_count": 180,
      "timeout": 60,
      "convert_feature_to_number": true,
      "add_one_second": false,
      "column_config": {
        "feature_start_column": 2
      },
      "api_params": [
        {
          "param_name": "applyId",
          "column_index": 0,
          "is_time_field": false
        },
        {
          "param_name": "baseTime",
          "column_index": 1,
          "is_time_field": true,
          "add_t_separator": true,
          "convert_date_to_time": true
        }
      ]
    },
    {
      "name": "接口B对比",
      "enabled": true,
      "description": "对比灰度环境接口B",
      "input_csv_file": "test_data.csv",
      "output_file_prefix": "api_b_result",
      "api_url": "http://10.129.2.25:8082/riskanalysis/Cdc/features",
      "thread_count": 180,
      "timeout": 60,
      "convert_feature_to_number": true,
      "add_one_second": false,
      "column_config": {
        "feature_start_column": 2
      },
      "api_params": [
        {
          "param_name": "applyId",
          "column_index": 0,
          "is_time_field": false
        },
        {
          "param_name": "baseTime",
          "column_index": 1,
          "is_time_field": true,
          "add_t_separator": true,
          "convert_date_to_time": true
        }
      ]
    }
  ],
  "global_config": {
    "default_thread_count": 180,
    "default_timeout": 60,
    "default_convert_feature_to_number": true
  },
  "output_config": {
    "output_intermediate_files": true,
    "output_error_records": true
  }
}
```

### 关键配置说明

| 配置项 | 接口A | 接口B | 说明 |
|--------|-------|-------|------|
| name | "接口A对比" | "接口B对比" | 场景名称 |
| input_csv_file | "test_data.csv" | "test_data.csv" | **相同**的输入文件 |
| output_file_prefix | "api_a_result" | "api_b_result" | **不同**的输出前缀 |
| api_url | 接口A的URL | 接口B的URL | **不同**的接口地址 |

## 📊 输出结果

### 文件结构

执行完成后，会在 `outputdata/api_comparison/` 目录下生成：

```
outputdata/api_comparison/
├── api_a_result_01301530_api_data.csv              # 接口A返回的数据
├── api_a_result_01301530_compare_analysis_report.csv  # 接口A对比分析报告
├── api_a_result_01301530_compare_feature_stats.csv    # 接口A特征统计
├── api_a_result_01301530_compare_全量数据合并.csv      # 接口A全量合并数据
├── api_b_result_01301530_api_data.csv              # 接口B返回的数据
├── api_b_result_01301530_compare_analysis_report.csv  # 接口B对比分析报告
├── api_b_result_01301530_compare_feature_stats.csv    # 接口B特征统计
└── api_b_result_01301530_compare_全量数据合并.csv      # 接口B全量合并数据
```

### 输出说明

| 文件类型 | 说明 |
|---------|------|
| `*_api_data.csv` | 接口返回的原始数据 |
| `*_analysis_report.csv` | 对比分析报告（异常记录） |
| `*_feature_stats.csv` | 特征统计（按异常率排序） |
| `*_全量数据合并.csv` | 全量数据合并（包含原始值和接口值） |

## 📈 执行流程

```
开始
  ↓
读取配置文件
  ↓
场景1: 接口A对比
  ├─ 读取输入文件
  ├─ 调用接口A获取数据
  ├─ 对比数据
  └─ 生成报告
  ↓
场景2: 接口B对比
  ├─ 读取输入文件（相同）
  ├─ 调用接口B获取数据
  ├─ 对比数据
  └─ 生成报告
  ↓
完成
```

---

## ⏱️ 执行时间估算

假设单个场景执行时间为 T：
- **接口A对比**: T
- **接口B对比**: T
- **总时间**: 约 2T

**示例**:
- 如果1000条数据，单个接口耗时5分钟
- 两个接口总耗时约10分钟

## 💡 优化建议

### 1. 减少数据量

如果只是验证功能，可以使用较小的数据集：
```json
{
  "input_csv_file": "test_data_sample_100.csv"
}
```

### 2. 调整线程数

根据服务器性能调整线程数：
```json
{
  "thread_count": 100  // 降低线程数可以减少服务器压力
}
```

### 3. 禁用中间文件输出

如果只需要对比报告，可以禁用中间文件：
```json
{
  "output_config": {
    "output_intermediate_files": false
  }
}
```

### 4. 临时禁用某个场景

如果只想执行其中一个场景：
```json
{
  "scenarios": [
    {
      "name": "接口A对比",
      "enabled": true,  // 启用
      ...
    },
    {
      "name": "接口B对比",
      "enabled": false,  // 禁用
      ...
    }
  ]
}
```

## 🔍 对比两个接口的结果

执行完成后，可以对比两个接口的分析报告：

### 方法1: 手动对比

1. 打开 `api_a_result_*_analysis_report.csv`
2. 打开 `api_b_result_*_analysis_report.csv`
3. 对比异常特征和异常率

### 方法2: 使用数据对比功能

可以使用MyDataCheck的"数据对比"功能，对比两个接口的输出：

1. 将 `api_a_result_*_api_data.csv` 作为文件1
2. 将 `api_b_result_*_api_data.csv` 作为文件2
3. 执行数据对比

## ⚠️ 注意事项

1. **输入文件**: 两个场景使用相同的输入文件
2. **输出前缀**: 必须使用不同的输出前缀，避免文件覆盖
3. **接口地址**: 确保两个接口地址都可访问
4. **执行顺序**: 场景按配置顺序依次执行
5. **时间戳**: 两个场景会使用相同的时间戳后缀

## 🎯 总结

### 当前方案

✅ **支持**: 通过配置多个场景，可以同时对比两个接口  
✅ **简单**: 只需修改配置文件，无需修改代码  
✅ **灵活**: 可以配置任意数量的场景  
⚠️ **顺序**: 场景是依次执行的，不是并行执行

### 适用场景

- ✅ 对比生产和灰度环境
- ✅ 对比不同版本的接口
- ✅ 对比不同服务器的接口
- ✅ A/B测试

### 不适用场景

- ❌ 需要严格同时执行（当前是顺序执行）
- ❌ 需要实时对比（当前是批量对比）
