# 端口配置说明

## 当前配置

**Web服务端口**：5001  
**访问地址**：http://localhost:5001

---

## 为什么使用5001端口？

macOS系统的AirPlay Receiver默认占用5000端口，为避免冲突，Web服务已切换到5001端口。

---

## 启动服务

```bash
cd MyDataCheck
./start_web.sh
```

服务启动后，访问：
```
http://localhost:5001
```

---

## 停止服务

```bash
./stop_web.sh
```

---

## 如果需要切换回5000端口

### 方法1：关闭AirPlay Receiver
1. 打开"系统设置"
2. 进入"通用" → "隔空播放与接力"
3. 关闭"隔空播放接收器"

### 方法2：手动修改配置
```bash
# 修改 web/app.py
sed -i '' 's/port=5001/port=5000/g' web/app.py

# 修改 start_web.sh
sed -i '' 's/5001/5000/g' start_web.sh

# 修改 stop_web.sh
sed -i '' 's/5001/5000/g' stop_web.sh
```

## 检查端口占用

### 查看5001端口
```bash
lsof -i:5001
```

### 查看5000端口
```bash
lsof -i:5000
```

---

## 常见问题

### Q: 端口仍然被占用？
```bash
# 停止所有web_app进程
pkill -9 -f web_app.py

# 检查端口
lsof -i:5001

# 如果有其他进程占用，手动停止
kill -9 <PID>
```

### Q: 如何使用其他端口？
修改 `web/app.py` 中的端口配置：
```python
app.run(
    debug=True,
    host='0.0.0.0',
    port=8080,  # 改为你想要的端口
    threaded=True
)
```
