# 数据对比配置保存功能完成报告

## 功能概述

为"数据对比"模块添加了配置保存和加载功能，用户可以通过Web界面保存当前的对比配置，并在需要时快速加载。

## 实现内容

### 1. 配置文件

**位置**：`MyDataCheck/data_comparison/config.json`

**结构**：
```json
{
  "scenarios": [
    {
      "name": "场景名称",
      "enabled": true,
      "description": "场景描述",
      "sql_file": "Sql文件名.csv",
      "api_file": "接口文件名.csv",
      "sql_key_column": 0,
      "api_key_column": 0,
      "sql_feature_start": 1,
      "api_feature_start": 1,
      "convert_feature_to_number": true,
      "output_prefix": "compare"
    }
  ],
  "global_config": {
    "default_convert_feature_to_number": true,
    "default_sql_key_column": 0,
    "default_api_key_column": 0,
    "default_sql_feature_start": 1,
    "default_api_feature_start": 1
  }
}
```

### 2. Web界面更新

#### 新增按钮

在"数据对比"页面的按钮组中添加了两个新按钮：

- **💾 保存配置**：保存当前的对比配置到`config.json`
- **📂 加载配置**：从`config.json`加载已保存的配置

#### 提示信息

添加了配置文件位置的提示：
```
💡 提示：配置将保存到 data_comparison/config.json
```

### 3. JavaScript功能

#### saveCompareConfig()

保存当前配置到服务器：
- 收集所有表单字段的值
- 构建配置对象（包含场景和全局配置）
- 调用`/api/compare/config/save` API
- 显示保存结果

#### loadCompareConfig()

从服务器加载配置：
- 调用`/api/compare/config/load` API
- 解析返回的配置数据
- 更新所有表单字段
- 更新文件信息显示
- 如果文件都存在，启用执行按钮

### 4. 后端API

API已在之前实现，本次更新确认其正常工作：

- **POST /api/compare/config/save**：保存配置
- **GET /api/compare/config/load**：加载配置

### 5. 配置管理模块

**文件**：`MyDataCheck/data_comparison/job/config_manager.py`

提供的功能：
- `save_config()`：保存配置到文件
- `load_config()`：从文件加载配置
- `add_scenario()`：添加场景
- `update_scenario()`：更新场景
- `delete_scenario()`：删除场景
- `get_scenario()`：获取指定场景

### 6. 配置说明文档

**文件**：`MyDataCheck/data_comparison/CONFIG_GUIDE.md`

包含内容：
- 配置文件结构说明
- 字段详细说明
- 使用方式（Web界面、手动编辑、命令行）
- 配置示例
- 输出文件说明
- 常见问题解答

## 使用流程

### 保存配置

1. 在Web界面配置对比参数：
   - 上传Sql文件和接口文件
   - 设置主键列索引
   - 设置特征起始列索引
   - 设置输出文件前缀
   - 勾选是否转换特征值为数值

2. 点击"💾 保存配置"按钮

3. 系统提示"配置保存成功！"

### 加载配置

1. 点击"📂 加载配置"按钮

2. 系统自动填充所有配置字段

3. 如果配置中的文件存在，自动启用"执行对比"按钮

### 命令行执行

```bash
cd MyDataCheck/data_comparison
python execute_data_comparison.py
```

脚本会自动读取`config.json`并执行所有启用的场景。

## 配置文件特点

### 1. 场景支持

- 支持多个场景配置
- 每个场景可以独立启用/禁用
- Web界面当前保存单场景，命令行支持多场景

### 2. 全局配置

- 提供默认值配置
- 新场景可以继承全局配置
- 减少重复配置

### 3. 灵活性

- 可以通过Web界面保存
- 可以手动编辑JSON文件
- 可以通过命令行执行

### 4. 可追溯性

- 配置文件包含`last_updated`时间戳
- 可以查看配置历史
