# 接口请求失败记录功能验证报告

## 功能概述
在MyDataCheck项目的接口数据对比功能中，新增了请求失败记录输出功能。当选中输出中间文件时，系统会自动将接口请求失败的记录输出到单独的CSV文件。

## 输出文件格式

### 文件命名
```
{输出前缀}_{时间戳}_api_data_error_records.csv
```

### 文件字段
| 字段名 | 说明 | 数据来源 |
|--------|------|----------|
| applyid | 申请ID | 从api_params配置或CSV表头自动识别 |
| baseTime | 基准时间 | 从api_params配置或CSV表头自动识别 |
| 错误code | 错误信息 | 从errors字典中提取 |

## 字段提取逻辑

### applyId字段提取
1. 优先从 `api_params` 配置中查找 `custNo`、`applyId`、`apply_id`、`cust_no` 等字段
2. 如果配置中没有，从CSV表头中查找包含这些关键词的列
3. 使用找到的列索引从原始数据行中提取值

### baseTime字段提取
1. 从 `api_params` 配置中查找第一个 `is_time_field=true` 的字段
2. 如果配置中没有，从CSV表头中查找包含 `basetime`、`time`、`date` 等关键词的列
3. 使用找到的列索引从原始数据行中提取值

## 错误类型记录

系统会记录以下类型的错误：

### 1. 参数错误
```
参数 'applyId' 的列索引无效: column_index=5, 行长度=3
参数 'baseTime' 为空: 行 10, 列 1
```

### 2. 请求错误
```
接口请求失败: params={'applyId': '123', 'baseTime': '2024-01-15T10:30:00.000'}
JSON解析失败: params={'applyId': '456', 'baseTime': '2024-01-15T11:00:00.000'}
```

### 3. 处理异常
```
处理异常: division by zero
```

## 配置说明

### 默认行为
- `output_error_records` 默认为 `true`
- 只有在 `output_intermediate_files=true` 时才会输出错误记录
- 如果没有错误，不会生成错误记录文件

### 配置选项

#### 启用错误记录输出（默认）
```json
{
  "output_config": {
    "output_intermediate_files": true,
    "output_error_records": true
  }
}
```

#### 关闭错误记录输出
```json
{
  "output_config": {
    "output_intermediate_files": true,
    "output_error_records": false
  }
}
```

#### 仅报告模式（不输出错误记录）
```json
{
  "output_config": {
    "output_intermediate_files": false
  }
}
```

## 使用场景

### 1. 排查接口问题
- 快速定位哪些请求失败
- 分析失败原因
- 统计失败率

### 2. 数据质量检查
- 发现输入数据的问题
- 检查参数配置是否正确
- 验证时间格式是否正确

### 3. 重试机制
- 使用错误记录中的applyId和baseTime
- 重新构造输入文件
- 对失败的记录进行重试

## 性能影响

### 内存占用
- 错误记录在内存中以字典形式存储
- 只记录错误信息，不保存完整的请求/响应数据
- 对内存影响很小

### 处理速度
- 错误记录的写入是在所有请求完成后进行
- 不影响并发请求的处理速度
- 写入速度很快（通常<1秒）

### 文件大小
- 每条错误记录约100-200字节
- 1000条错误记录约100-200KB
- 对磁盘空间影响很小

## 注意事项

### 1. 必须启用中间文件输出
只有当 `output_intermediate_files=true` 时，才会输出错误记录文件。

### 2. 字段自动识别
系统会自动识别applyId和baseTime字段，但建议在api_params中明确配置。

### 3. 错误信息格式
错误信息会包含详细的上下文，便于排查问题。

### 4. 文件编码
输出文件使用UTF-8编码，可以用Excel直接打开。
