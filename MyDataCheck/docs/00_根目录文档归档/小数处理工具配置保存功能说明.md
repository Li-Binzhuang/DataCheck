# 小数处理工具 - 配置保存功能说明

## 📋 功能概述

为小数处理工具添加了配置保存和加载功能，用户可以保存常用的配置参数，下次使用时自动加载，提高使用效率。

## ✨ 新增功能

### 1. 配置保存
- 点击"💾 保存配置"按钮，将当前配置保存到服务器
- 配置文件路径：`data_comparison/decimal_config.json`
- 保存的配置包括：
  - 文件路径
  - 源列名称
  - 参考列名称
  - 小数处理方式
  - 对比方式
  - 容差值
  - 输出文件前缀
  - 输出全量数据选项
  - 文件输入模式

### 2. 配置加载
- 点击"📂 加载配置"按钮，从服务器加载已保存的配置
- 页面加载时自动尝试加载配置（如果存在）
- 加载配置后，所有表单字段自动填充

### 3. 自动加载
- 打开小数处理工具页面时，自动加载上次保存的配置
- 如果没有保存的配置，使用默认值

## 🔧 技术实现

### 前端实现 (data-compare.js)

#### 新增函数

1. **saveDecimalConfig()**
   - 收集当前页面的所有配置参数
   - 发送POST请求到 `/api/compare/decimal/config/save`
   - 显示保存成功/失败提示

2. **loadDecimalConfig(forceLoad)**
   - 发送GET请求到 `/api/compare/decimal/config/load`
   - 将配置数据填充到表单字段
   - forceLoad参数：是否强制覆盖当前已上传的文件

3. **自动加载逻辑**
   - 页面加载完成后500ms自动调用loadDecimalConfig()
   - 不会覆盖用户已上传的文件（除非点击"加载配置"按钮）

### 后端实现 (compare_routes.py)

#### 新增路由

1. **POST /api/compare/decimal/config/save**
   - 接收配置数据（JSON格式）
   - 保存到 `data_comparison/decimal_config.json`
   - 返回保存结果

2. **GET /api/compare/decimal/config/load**
   - 读取 `data_comparison/decimal_config.json`
   - 返回配置数据
   - 如果文件不存在，返回错误信息

### 配置文件格式

```json
{
  "file_path": "PATH:/data/diff.csv",
  "source_column": "API值",
  "reference_column": "CSV值",
  "decimal_method": "round",
  "compare_mode": "exact",
  "tolerance_value": 0.01,
  "output_prefix": "decimal_processed",
  "output_full_data": false,
  "file_input_mode": "path"
}
```

### 配置字段说明

| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| file_path | string | 文件路径（上传文件或服务器路径） | "PATH:/data/diff.csv" |
| source_column | string | 源列名称（需要处理的列） | "API值" |
| reference_column | string | 参考列名称（提供小数位数的列） | "CSV值" |
| decimal_method | string | 小数处理方式 | "round", "double_round", "truncate", "ceil", "none" |
| compare_mode | string | 对比方式 | "exact", "tolerance", "last_digit", "last_digit_2" |
| tolerance_value | number | 容差值（仅对比方式为tolerance时有效） | 0.01 |
| output_prefix | string | 输出文件前缀 | "decimal_processed" |
| output_full_data | boolean | 是否输出全量数据 | true/false |
| file_input_mode | string | 文件输入模式 | "upload", "path" |

## 📝 使用说明

### 保存配置

1. 配置好所有参数（文件、列名、处理方式等）
2. 点击"💾 保存配置"按钮
3. 看到"✅ 配置保存成功"提示

### 加载配置

#### 方式1：自动加载
- 打开小数处理工具页面
- 系统自动加载上次保存的配置
- 如果有配置，表单字段自动填充

#### 方式2：手动加载
- 点击"📂 加载配置"按钮
- 系统加载配置并填充表单
- 会覆盖当前已上传的文件

### 配置优先级

1. 用户手动输入/上传 > 自动加载的配置
2. 点击"加载配置"按钮 > 用户当前输入

## 🎯 使用场景

### 场景1：重复处理相同类型的文件
- 第一次配置好列名、处理方式等参数
- 保存配置
- 以后只需上传新文件，其他参数自动填充

### 场景2：多个处理任务切换
- 为不同的处理任务保存不同的配置
- 通过加载配置快速切换

### 场景3：团队协作
- 配置文件保存在服务器
- 团队成员可以共享配置

## ⚠️ 注意事项

1. **配置文件位置**
   - 配置保存在服务器：`data_comparison/decimal_config.json`
   - 不同用户共享同一个配置文件
   - 如需多用户独立配置，需要扩展实现

2. **文件路径处理**
   - 上传文件模式：保存的是临时文件名
   - 服务器路径模式：保存的是完整路径
   - 自动加载时，如果文件不存在，需要重新上传

3. **自动加载行为**
   - 页面加载时自动加载配置
   - 不会覆盖用户已上传的文件
   - 只有点击"加载配置"按钮才会强制覆盖

4. **配置覆盖**
   - 每次保存会覆盖之前的配置
   - 目前只支持单个配置文件
   - 如需多配置管理，需要扩展实现
