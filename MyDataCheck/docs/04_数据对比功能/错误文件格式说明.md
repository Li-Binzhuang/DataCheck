# 错误文件格式说明

## 概述

当接口数据对比过程中出现接口请求失败时，系统会自动生成错误文件 `_errors.csv`，记录所有失败的请求信息。

## 文件格式

### 列定义

| 列名 | 说明 | 示例 |
|------|------|------|
| applyId | 申请ID（从CSV文件中提取） | `1073458287442403329` |
| baseTime | 基准时间（从CSV文件中提取） | `2025-11-27T14:15:48.503` |
| errors | 接口请求失败的详细错误信息 | `请求失败 - 错误: Expecting ':' delimiter` |

### 完整示例

```csv
applyId,baseTime,errors
1073458287442403329,2025-11-27T14:15:48.503,请求失败 - params={'applyId': '1073458287442403329'}, 错误: Expecting ':' delimiter: line 1 column 811011
1073488146055061505,2025-11-27T14:30:17.436,请求失败 - params={'applyId': '1073488146055061505'}, 错误: Response ended prematurely
1073488146055061506,2025-11-27T14:35:20.123,JSON解析失败 - params={'applyId': '1073488146055061506'}, 错误: Expecting value: line 1 column 1
1073488146055061507,2025-11-27T14:40:15.789,请求超时 - params={'applyId': '1073488146055061507'}, 超时时间: 60秒
1073488146055061508,2025-11-27T14:45:30.456,连接错误 - params={'applyId': '1073488146055061508'}, 错误: Connection refused
```

## 字段提取逻辑

### applyId 提取

系统会按以下优先级查找 applyId：

1. **从表头匹配**：查找以下字段名（不区分大小写）
   - `applyId`
   - `apply_id`
   - `custNo`
   - `cust_no`
   - `use_credit_apply_id`

2. **从配置的列索引**：如果表头匹配失败，使用配置的第一个参数列

**示例**：
```python
# 配置中的参数
"api_params": [
    {
        "param_name": "applyId",
        "column_index": 0,  # 从第0列读取
        "is_time_field": false
    }
]
```

### baseTime 提取

系统会按以下优先级查找 baseTime：

1. **从表头匹配**：查找以下字段名（不区分大小写）
   - `baseTime`
   - `base_time`
   - `use_create_time`
   - `create_time`

2. **从配置的列索引**：如果表头匹配失败，使用配置的时间字段列

**示例**：
```python
# 配置中的参数
"api_params": [
    {
        "param_name": "baseTime",
        "column_index": 1,  # 从第1列读取
        "is_time_field": true
    }
]
```

## 错误信息格式

### 常见错误类型

#### 1. JSON 解析错误
```
JSON解析失败 - params={'applyId': '1073458287442403329'}, 错误: Expecting ':' delimiter: line 1 column 811011
```

**含义**：接口返回的响应不是有效的 JSON 格式

#### 2. 响应中断错误
```
请求失败 - params={'applyId': '1073488146055061505'}, 错误: Response ended prematurely
```

**含义**：服务器在发送完整响应前关闭了连接

#### 3. 请求超时
```
请求超时 - params={'applyId': '1073488146055061507'}, 超时时间: 60秒
```

**含义**：接口响应时间超过配置的超时时间

#### 4. 连接错误
```
连接错误 - params={'applyId': '1073488146055061508'}, 错误: Connection refused
```

**含义**：无法连接到接口服务器

#### 5. HTTP 错误
```
请求失败 - params={'applyId': '1073488146055061509'}, 错误: 500 Internal Server Error
```

**含义**：接口返回 HTTP 错误状态码

## 使用场景

### 1. 快速定位问题

在 Excel 中打开错误文件：
- 按 applyId 排序，查看特定申请的错误
- 按 errors 筛选，查看特定类型的错误
- 统计错误数量和类型

### 2. 批量分析

使用 Python 分析错误模式：

```python
import pandas as pd

# 读取错误文件
df = pd.read_csv('bz_3_01272015_compare_errors.csv')

# 统计错误类型
error_types = df['errors'].str.extract(r'(.*?) -')[0].value_counts()
print("错误类型统计:")
print(error_types)

# 查看特定错误
json_errors = df[df['errors'].str.contains('JSON解析失败')]
print(f"\nJSON解析错误数量: {len(json_errors)}")

# 导出特定错误的 applyId
json_error_ids = json_errors['applyId'].tolist()
print(f"JSON解析错误的 applyId: {json_error_ids[:10]}")
```

### 3. 重试失败的请求

提取失败的 applyId，创建新的输入文件进行重试：

```python
import pandas as pd

# 读取错误文件
errors_df = pd.read_csv('bz_3_01272015_compare_errors.csv')

# 读取原始文件
original_df = pd.read_csv('cdc1_features_consultas.csv')

# 提取失败的 applyId
failed_ids = errors_df['applyId'].tolist()

# 从原始文件中筛选失败的行
retry_df = original_df[original_df['applyId'].isin(failed_ids)]

# 保存为新文件用于重试
retry_df.to_csv('retry_input.csv', index=False)
print(f"已生成重试文件，包含 {len(retry_df)} 行数据")
```

## 错误排查建议

### 按错误类型排查

| 错误类型 | 可能原因 | 解决方案 |
|---------|---------|---------|
| JSON解析失败 | 接口返回格式错误 | 联系接口提供方修复 |
| Response ended prematurely | 服务器中断连接 | 增加超时时间，检查服务器资源 |
| 请求超时 | 接口响应慢 | 增加 timeout 配置，减少并发数 |
| 连接错误 | 网络问题或服务未启动 | 检查网络和服务状态 |
| HTTP 错误 | 接口内部错误 | 查看接口日志，联系接口提供方 |

### 批量排查流程

1. **打开错误文件**：查看错误数量和类型
2. **分类统计**：统计每种错误类型的数量
3. **优先处理**：优先处理数量最多的错误类型
4. **测试验证**：修复后用少量数据测试
5. **批量重试**：确认修复后批量重试失败的请求

## 文件生成规则

### 何时生成

- ✅ 有接口请求失败时，自动生成错误文件
- ❌ 所有请求都成功时，不生成错误文件

### 文件命名

格式：`{前缀}_{时间戳}_errors.csv`

**示例**：
- `bz_3_01272015_errors.csv`
- `test_01271950_errors.csv`
- `cdc1_01280930_errors.csv`

### 文件位置

错误文件保存在输出目录中：
- Web 模式：`MyDataCheck/outputdata/api_comparison/`
- 命令行模式：执行脚本所在目录

## 注意事项

### 1. 编码问题

错误文件使用 UTF-8 编码，如果在 Excel 中打开出现乱码：
- 使用"数据" → "从文本/CSV"导入
- 选择 UTF-8 编码

### 2. 错误信息长度

某些错误信息可能很长，在 Excel 中可能需要：
- 调整列宽
- 启用自动换行
- 使用文本编辑器查看

### 3. 数据隐私

错误文件包含 applyId 等敏感信息，注意：
- 不要上传到公共位置
- 使用后及时删除
- 遵守数据安全规范

## 技术实现

### 错误收集

```python
# 收集错误信息
error_records = []

if "error" in comparison_result:
    error_records.append({
        "row_index": row_index + 2,
        "row_data": row,
        "error_message": error_msg
    })
```

### 字段提取

```python
# 提取 applyId
apply_id_fields = ['applyId', 'apply_id', 'custNo', 'cust_no', 'use_credit_apply_id']
for i, header in enumerate(headers):
    if header in apply_id_fields and i < len(row_data):
        apply_id = row_data[i].strip()
        break

# 提取 baseTime
time_fields = ['baseTime', 'base_time', 'use_create_time', 'create_time']
for i, header in enumerate(headers):
    if header in time_fields and i < len(row_data):
        base_time = row_data[i].strip()
        break
```

### 文件写入

```python
def _write_error_file(self, output_path, headers, error_records):
    with open(output_path, "w", encoding="utf-8", newline="") as f:
        writer = csv.writer(f)
        writer.writerow(["applyId", "baseTime", "errors"])
        
        for record in error_records:
            writer.writerow([apply_id, base_time, error_msg])
```
