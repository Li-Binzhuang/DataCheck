# CSV 数据对比 - 列名前后缀匹配功能说明

## 功能概述

在 CSV 数据对比模块中，当两个表格的列名存在前后缀差异时，可以通过指定列名前缀和后缀来自动匹配列名，无需手动修改文件。

## 使用场景

### 场景示例

**模型特征表（第一个文件）的列名：**
```
inst_telcel_scat_account_count_7d
inst_telcel_scat_account_count_30d
inst_telcel_scat_account_count_90d
```

**接口/灰度/从库特征表（第二个文件）的列名：**
```
local_all_sms_inst_telcel_scat_account_count_7d_v3
local_all_sms_inst_telcel_scat_account_count_30d_v3
local_all_sms_inst_telcel_scat_account_count_90d_v3
```

## 配置方法

### 步骤 1：上传文件

1. 在"文件输入方式"中选择"上传文件"或"服务器路径"
2. 上传或指定两个 CSV/XLSX 文件

### 步骤 2：配置列名匹配

在"列名匹配配置"部分填写：

- **列名前缀**：`local_all_sms_`
- **列名后缀**：`_v3`

### 步骤 3：执行对比

点击"执行对比"按钮，系统会自动：

1. 读取模型特征表的列名（如 `inst_telcel_scat_account_count_7d`）
2. 在接口表中查找对应的列名（通过添加前后缀：`local_all_sms_inst_telcel_scat_account_count_7d_v3`）
3. 对匹配的列进行数据对比

## 工作原理

### 列名匹配流程

```
模型表列名: inst_telcel_scat_account_count_7d
                    ↓
            应用前后缀转换
                    ↓
前缀 + 列名 + 后缀 = local_all_sms_inst_telcel_scat_account_count_7d_v3
                    ↓
在接口表中查找该列名
                    ↓
如果找到 → 对比该列的数据
如果未找到 → 该列标记为"匹配失败"
```

### 匹配结果

- **匹配成功**：在执行日志中会显示 `✓ 列名匹配: '原列名' <- '转换后的列名'`
- **匹配失败**：该列将被标记为在接口表中不存在，对比结果中会显示为差异

## 配置示例

### 示例 1：仅有前缀

**模型表列名：** `feature_a`, `feature_b`  
**接口表列名：** `prefix_feature_a`, `prefix_feature_b`

配置：
- 列名前缀：`prefix_`
- 列名后缀：（留空）

### 示例 2：仅有后缀

**模型表列名：** `feature_a`, `feature_b`  
**接口表列名：** `feature_a_v2`, `feature_b_v2`

配置：
- 列名前缀：（留空）
- 列名后缀：`_v2`

### 示例 3：前后缀都有

**模型表列名：** `feature_a`, `feature_b`  
**接口表列名：** `api_feature_a_v3`, `api_feature_b_v3`

配置：
- 列名前缀：`api_`
- 列名后缀：`_v3`

### 示例 4：不使用前后缀（默认行为）

**模型表列名：** `feature_a`, `feature_b`  
**接口表列名：** `feature_a`, `feature_b`

配置：
- 列名前缀：（留空）
- 列名后缀：（留空）

此时系统会使用原始列名进行精确匹配。

## 配置保存和加载

### 保存配置

点击"保存配置"按钮，当前的所有配置（包括列名前后缀）会被保存到 `data_comparison/config.json`

### 加载配置

点击"加载配置"按钮，会自动加载之前保存的配置，包括列名前后缀设置

## 常见问题

### Q1：如果列名前后缀配置不正确会怎样？

A：系统会尝试直接匹配列名。如果直接匹配也失败，该列会被标记为在接口表中不存在，对比结果中会显示为差异。

### Q2：前后缀是否区分大小写？

A：是的，前后缀匹配是区分大小写的。请确保前后缀的大小写与实际列名一致。

### Q3：如果模型表中的列在接口表中不存在怎样？

A：该列的所有数据都会被标记为差异，因为接口表中没有对应的列进行对比。

### Q4：可以同时使用前缀和后缀吗？

A：可以。系统会先应用前缀，再应用后缀，最终形成 `前缀 + 原列名 + 后缀` 的形式。

### Q5：如果不需要前后缀匹配怎样？

A：将前缀和后缀都留空即可。系统会使用原始列名进行精确匹配，这是默认行为。

## 技术细节

### 匹配优先级

1. **直接匹配**：首先尝试在接口表中查找与模型表列名完全相同的列
2. **前后缀匹配**：如果直接匹配失败且配置了前后缀，则尝试带前后缀的匹配

这样设计可以确保即使配置了前后缀，也不会影响那些列名完全相同的列的匹配。

### 性能影响

- 前后缀匹配使用字典查找，时间复杂度为 O(1)
- 对整体对比性能的影响可以忽略不计

## 输出文件说明

启用列名前后缀匹配后，输出文件中的列名仍然使用模型表的原始列名，便于对标准特征进行追踪。
