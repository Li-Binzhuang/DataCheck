# 线上灰度落数对比

## 功能说明

本系统用于对比线上落数数据和离线数据，主要功能包括：

1. **JSON解析**：解析线上落数CSV文件中指定列的JSON字符串，输出为特征CSV文件
2. **数据对比**：对比解析后的在线数据和离线数据，以离线文件为基准
3. **报告生成**：生成差异特征汇总和差异数据明细文件

## 配置文件说明

`执行配置.json` 文件包含以下配置项：

```json
{
  "output_prefix": "yx_online",          // 输出文件前缀（可选）
  "online_file": "线上落数.csv",           // 线上文件路径
  "offline_file": "离线文件.csv",         // 离线文件路径
  "json_column": "local_olduser_order_new", // JSON列名
  "online_key_column": 0,                 // 在线文件主键列索引（A列=0，B列=1）
  "offline_key_column": 1,                // 离线文件主键列索引（A列=0，B列=1）
  "feature_start_column": 3,             // 特征列起始索引（从0开始）
  "convert_string_to_number": false      // 是否将JSON中的字符串值转换为数值
}
```

### 配置项说明

- **output_prefix**: 输出文件前缀（可选，如果为空则不添加前缀）
- **online_file**: 线上落数CSV文件路径（相对于脚本目录）
- **offline_file**: 离线文件CSV文件路径（相对于脚本目录）
- **json_column**: 需要解析的JSON列名
- **online_key_column**: 在线文件的主键列索引（从0开始，A列=0，B列=1，C列=2，以此类推）
- **offline_key_column**: 离线文件的主键列索引（从0开始，A列=0，B列=1，C列=2，以此类推）
- **feature_start_column**: 从第几列开始对比特征（索引从0开始，例如3表示第4列）
- **convert_string_to_number**: 是否将JSON中的字符串值转换为数值类型（true/false）

**注意**：所有输出文件会自动添加时间戳后缀（格式：MMDDHHmm，例如：10081950表示10月8日19:50）

## 使用方法

### 1. 准备配置文件

编辑 `执行配置.json`，设置正确的文件路径和列名。

### 2. 准备数据文件

将以下文件放在脚本目录下：
- `线上落数.csv` - 包含JSON列的线上数据文件
- `离线文件.csv` - 离线数据文件

### 3. 执行对比流程

```bash
python 执行对比流程.py
```

## 输出文件

执行完成后会生成以下文件（文件名格式：`{前缀}_{原文件名}_{时间戳}_{类型}.csv`）：

1. **`{前缀}_{线上文件名}_{时间戳}_解析后.csv`** - 解析JSON后的特征CSV文件
   - 示例：`yx_online_yx_online_json_10081950_解析后.csv`
2. **`{前缀}_{线上文件名}_{时间戳}_对比结果_差异特征汇总.csv`** - 差异特征汇总文件
   - 包含：特征名、总数、差异数、差异占比
   - 只包含有差异的特征
   - 示例：`yx_online_yx_online_json_10081950_对比结果_差异特征汇总.csv`
3. **`{前缀}_{线上文件名}_{时间戳}_对比结果_差异数据明细.csv`** - 差异数据明细文件
   - 包含：主键值、cust_no、时间、特征名、离线值、在线值
   - 示例：`yx_online_yx_online_json_10081950_对比结果_差异数据明细.csv`
4. **`{前缀}_{线上文件名}_{时间戳}_对比结果_特征统计.csv`** - 特征统计文件
   - 汇总统计：无异常特征总数、有异常特征总数
   - 详细统计：特征名、是否有异常、比对数据、匹配数量、异常数量、匹配率(%)、异常率(%)
   - 包含所有特征（有异常和无异常）
   - 示例：`yx_online_yx_online_json_10081950_对比结果_特征统计.csv`

**文件名说明**：
- `{前缀}`：从配置文件的 `output_prefix` 字段读取（如果为空则不添加）
- `{时间戳}`：执行时的月日时分（格式：MMDDHHmm，例如：10081950）

## 功能特性

### JSON解析功能

- 自动打平嵌套字典结构
- 支持多种编码格式的CSV文件读取
- 每个apply_id为一行，每个特征值为一列
- 可选：将JSON中的字符串值转换为数值类型

### 数据对比功能

- 以离线文件为基准进行对比
- 支持自定义主键列（A列和B列）
- 支持自定义特征列起始位置
- 自动统计差异特征和差异数据

### 报告生成功能

- 差异特征汇总：统计每个特征的差异数量和占比
- 差异数据明细：详细的差异记录，包含主键值、cust_no、时间、特征名、离线值和在线值
- 输出格式：CSV格式（便于查看和处理）

## 注意事项

1. 确保CSV文件使用UTF-8编码（或GBK/GB2312编码，系统会自动检测）
2. 主键列的值必须唯一，用于匹配记录
3. 特征列起始索引从0开始计数（例如：第4列的索引是3）
4. 如果启用字符串转数值功能，系统会尝试将JSON中的字符串值转换为数值类型
