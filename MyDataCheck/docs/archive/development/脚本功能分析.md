# 场景1_接口数据对比 - 脚本功能分析

## 当前脚本列表

1. **执行对比流程.py** - 主执行脚本（推荐使用）
2. **获取接口数据.py** - 获取接口数据模块
3. **对比接口数据.py** - 对比数据模块
4. **检测列索引.py** - 检测列索引工具
5. **接口校验.py** - ⚠️ 旧版脚本（冗余）

## 脚本功能对比

### ✅ 推荐使用的脚本组合

**执行对比流程.py** (主脚本)
- 功能：整合所有功能，支持JSON配置和多场景
- 调用：`获取接口数据.py` + `对比接口数据.py` + `检测列索引.py`
- 优势：
  - 支持多场景配置
  - 统一的配置管理
  - 自动检测列索引
  - 输出文件自动添加前缀和时间戳

**获取接口数据.py** (功能模块)
- 功能：读取CSV文件，请求接口，输出接口全部数据
- 被调用：`执行对比流程.py`
- 可独立运行：是（有自己的main函数）

**对比接口数据.py** (功能模块)
- 功能：对比原始数据和接口返回的数据
- 被调用：`执行对比流程.py`
- 可独立运行：是（有自己的main函数）

**检测列索引.py** (工具模块)
- 功能：自动检测CSV文件中cust_no和时间字段的列索引
- 被调用：`执行对比流程.py`
- 可独立运行：是（有自己的main函数）

### ⚠️ 冗余脚本

**接口校验.py** (旧版脚本)
- 功能：将获取接口数据和对比数据的功能合并在一个脚本中
- 状态：**冗余，建议删除或移动到历史文件夹**
- 原因：
  1. 功能已被 `获取接口数据.py` + `对比接口数据.py` 替代
  2. 没有使用公共工具模块，代码重复
  3. 不支持JSON配置和多场景
  4. 输出格式较简单（仅CSV格式）
  5. 没有自动检测列索引功能

## 建议

### 推荐方案

**使用 `执行对比流程.py`** 作为主入口：
- 支持JSON配置，便于管理多个场景
- 自动检测列索引，适应不同的文件格式
- 输出文件自动添加前缀和时间戳
- 代码结构清晰，易于维护

### 清理建议

1. **可以删除**：`接口校验.py`（功能已被新脚本替代）
2. **保留但标记**：如果担心兼容性，可以移动到 `历史/` 文件夹

1. 删除或移动到 `历史/` 文件夹

3. 其他脚本作为功能模块保留（可独立运行，也可被主脚本调用）
