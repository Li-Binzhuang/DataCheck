# 流式对比批次大小配置修复

## 问题描述

在接口数据对比的流式对比模式中，"流式对比批次大小"配置存在以下问题：
1. 配置无法保存到配置文件
2. 配置无法从配置文件加载
3. 执行时未按配置的批次大小进行处理

## 问题原因

前端JS代码（`api-compare.js`）中：
- `collectConfig()` 函数没有收集 `global_batch_size` 的值
- `loadConfig()` 函数没有加载 `default_batch_size` 的值
- 导致配置无法保存和加载

## 修复内容

### 1. 修复 `collectConfig()` 函数

**文件**: `MyDataCheck/static/js/api-compare.js`

**修改位置1**: 添加批次大小的读取
```javascript
const globalThreadCount = getGlobalValue('global_thread_count', 150);
const globalTimeout = getGlobalValue('global_timeout', 60);
const globalBatchSize = getGlobalValue('global_batch_size', 50);  // 新增
```

**修改位置2**: 将批次大小添加到场景配置
```javascript
const scenario = {
    name: getValue('.scenario-name-input', '未命名场景'),

    column_config: {
```

**修改位置3**: 将批次大小添加到全局配置

## 配置说明

### 配置位置

在Web界面的"接口数据对比"页面，全局配置区域：
- **流式对比批次大小**: 控制流式对比模式下每批处理的行数

### 配置范围

- 最小值: 1
- 最大值: 1000
- 建议值: 50-200
- 默认值: 50

### 配置效果

- **较小的批次** (如50): 降低内存占用，适合大文件或内存受限环境
- **较大的批次** (如200): 提升处理速度，适合内存充足环境

### 配置文件格式

配置保存在 `MyDataCheck/api_comparison/config.json`:

```json
{
  "scenarios": [
    {
      "name": "场景1",
      "batch_size": 50,
      ...
    }
  ],
  "global_config": {
    "default_thread_count": 150,
    "default_timeout": 60,
    "default_batch_size": 50
  }
}
```

## 使用方法

### 1. 设置批次大小

1. 打开Web界面，进入"接口数据对比"页面
2. 在全局配置区域找到"流式对比批次大小"输入框
3. 输入期望的批次大小（建议50-200）
4. 点击"保存配置"按钮

### 2. 验证配置

保存后，配置会立即生效：
- 刷新页面，检查批次大小是否正确加载
- 执行对比任务，观察输出日志中的批次信息

### 3. 输出示例

执行时会显示实际使用的批次大小：

```
================================================================================
流式对比模式
================================================================================
  输入文件: test.csv
  批次大小: 50
  线程数: 150
================================================================================

处理批次 1/10: 50 行
处理批次 2/10: 50 行
处理批次 3/10: 50 行
...
```

## 注意事项

1. **配置生效时机**: 保存配置后立即生效，无需重启服务
2. **场景级配置**: 每个场景会使用全局配置的批次大小
3. **内存影响**: 批次大小越大，内存占用越高，但处理速度可能更快
4. **性能平衡**: 建议根据实际数据量和内存情况调整批次大小
