## 概述

CDC 项目从墨西哥征信局（Círculo de Crédito）的 JSON 数据中提取特征，分为四个板块：

| 板块 | 数据来源 | 主要内容 | 特征数量 |
|------|---------|---------|---------|
| 第一板块 | `response_body.consultas` | 征信查询记录 | 988 |
| 第二板块 | `response_body.creditos` | 信贷账户记录 | 16,647 |
| 第三板块 | `response_body.creditos` | 预防类型统计 | 120 |
| BOSS板块 | 全部（consultas + creditos + empleos + domicilios） | 综合特征 | 148 |

**核心思想**：
- 把 JSON 数据"平铺"成明细表（一条记录一行）
- 按时间窗口筛选数据
- 按不同维度分组统计
- 输出每个申请（apply_id）的特征值

## 第一板块：consultas（查询板块）

### 📊 数据来源

从 `response_body.consultas` 解析查询记录，平均每个申请有 **32.8 条查询记录**。

### 🎯 核心逻辑

**用大白话说**：统计"别人什么时候查过我的征信，查了多少次，查询金额是多少"。

### 📅 时间窗口

使用 **`fechaConsulta`（查询日期）** 作为窗口筛选字段。

```
窗口筛选条件：
request_time - fechaConsulta <= N 天
```

**7 个时间窗口**：30、60、90、120、180、360、720 天

### 🔢 计算维度

#### 1. 按机构类型分组（17 类）

把查询机构（`nombreOtorgante`）归类为 17 大类：

| 类别 | 说明 | 英文ID |
|------|------|--------|
| 商店信息 | 商店、超市、百货 | shop |
| 大众金融协会 | 金融合作社 | mass_fin_assn |
| 多用途 | 多用途金融公司 | multi_purpose |
| 非银行抵押 | 非银行抵押贷款 | nonbank_mortgage |
| 服务信息 | 医疗、服务 | service |
| 付费电视 | 电视服务 | paytv |
| 个人贷款 | 个人贷款公司 | personal_loan |
| 基金和信托 | 基金、信托 | fund_trust |
| 建筑 | 建材 | construction |
| 金融公司 | 汽车金融等 | finance_company |
| 通讯 | 电话、通讯 | telecom |
| 销售 | 目录销售 | catalog_sale |
| 小额贷款 | 小额信贷 | micro_loan |
| 银行 | 银行 | bank |
| 非银行 | 非银行金融 | nonbank |
| 政府 | 政府机构 | government |
| 个征信机构 | SIC | sic |

#### 2. 按查询类型分组（7 类）

根据 `tipoCredito`（查询的信贷类型）分组：

| 类型代码 | 说明 |
|---------|------|
| CC | 消费信贷 |
| PP | 个人贷款 |
| TC | 信用卡 |
| LC | 信用额度 |
| PB | 个人银行贷款 |
| AR | 租赁 |
| AF | 汽车金融 |

### 📈 输出特征

**每个窗口 N 天内，输出以下特征**：

#### A. 总量特征（1 个）
- `consultas_{N}d_total_cnt`：窗口内查询总次数

#### B. 机构类型特征（17 类 × 2 = 34 个）
- `consultas_{N}d_otorgante_{机构ID}_cnt`：该机构类型查询次数
- `consultas_{N}d_otorgante_{机构ID}_ratio`：该机构类型查询占比

#### C. 查询类型特征（7 类 × 2 = 14 个）
- `consultas_{N}d_tipo_{类型ID}_cnt`：该查询类型次数
- `consultas_{N}d_tipo_{类型ID}_ratio`：该查询类型占比

#### D. 金额统计特征（17 类 × 5 统计 = 85 个）

对每个机构类型，统计 `importeCredito`（合同金额）：
- `mean`：平均值
- `std`：标准差
- `min`：最小值
- `max`：最大值
- `notNull`：有值占比

**每个窗口特征数**：1 + 34 + 85 + 14 = **134 个**
**总特征数**：134 × 7 窗口 + 1（request_time）= **939 个**

（实际是 988 个，可能包含一些全量统计特征）

### 💡 举例说明

**场景**：张三在 2025-11-25 申请贷款

**近 30 天查询记录**：
- 2025-11-20：银行查询，金额 5000
- 2025-11-15：信用卡公司查询，金额 3000
- 2025-11-10：小额贷款公司查询，金额 1000

**输出特征**：
- `consultas_30d_total_cnt` = 3（总共 3 次查询）
- `consultas_30d_otorgante_bank_cnt` = 1（银行查询 1 次）
- `consultas_30d_otorgante_bank_ratio` = 0.33（银行查询占比 33%）
- `consultas_30d_otorgante_bank_importeCredito_mean` = 5000（银行查询平均金额）
- `consultas_30d_tipo_tc_cnt` = 1（信用卡查询 1 次）
- `consultas_30d_tipo_pp_cnt` = 1（个人贷款查询 1 次）

### ⚠️ 异常处理

**A/C/D 规则**：
- **A**：关键字段缺失（apply_id/request_time/response_body）→ 所有特征 = -999
- **C**：response_body 非法 JSON → 所有特征 = -999
- **D**：没有 consultas 字段 → 所有特征 = -999

**空列表**：
- consultas = [] → total_cnt = 0，其他特征 = 0 或 NaN

## 第二板块：creditos（信贷板块）

### 📊 数据来源

从 `response_body.creditos` 解析信贷账户记录，平均每个申请有 **55.6 个信贷账户**。

### 🎯 核心逻辑

**用大白话说**：统计"我有多少个信贷账户，每个账户的状态、余额、额度、逾期情况"。

### 📅 时间窗口

使用 **`fechaAperturaCuenta`（开户日期）** 作为窗口筛选字段。

```
窗口筛选条件：
request_time - fechaAperturaCuenta <= N 天
```

**7 个时间窗口**：30、60、90、120、180、360、720 天

### 🔢 计算维度

#### 1. 按机构类型分组（17 类）

与第一板块相同的 17 个机构类型。

#### 2. 按账户类型分组（7 类）

根据 `tipoCuenta`（账户类型）分组：

| 类型代码 | 说明 |
|---------|------|
| F | 固定付款 |
| H | 抵押贷款 |
| R | 循环信用 |
| Q | 无担保信用 |
| A | 授信或预支信用 |
| E | 分期付款信用 |
| P | 质押信用 |

#### 3. 按信贷类型分组（5 类）

根据 `tipoCredito`（信贷类型）分组：

| 类型代码 | 说明 |
|---------|------|
| CC | 消费信贷 |
| PP | 个人贷款 |
| TC | 信用卡 |
| LC | 信用额度 |
| PB | 个人银行贷款 |

### 📈 输出特征

**每个窗口 N 天内，输出以下特征**：

#### A. 总量特征（1 个）
- `creditos_{N}d_total_cnt`：窗口内信贷账户总数

#### B. 机构类型特征（17 类 × 2 = 34 个）
- `creditos_{N}d_otorgante_{机构ID}_cnt`：该机构类型账户数
- `creditos_{N}d_otorgante_{机构ID}_ratio`：该机构类型账户占比

#### C. 账户类型特征（7 类 × 2 = 14 个）
- `creditos_{N}d_tipo_cuenta_{类型ID}_cnt`：该账户类型数量
- `creditos_{N}d_tipo_cuenta_{类型ID}_ratio`：该账户类型占比

#### D. 信贷类型特征（5 类 × 2 = 10 个）
- `creditos_{N}d_tipo_credito_{类型ID}_cnt`：该信贷类型数量
- `creditos_{N}d_tipo_credito_{类型ID}_ratio`：该信贷类型占比

#### E. 机构类型数值画像（17 类 × 16 指标 × 5 统计 = 1360 个）

对每个机构类型，统计 16 个指标：

**金额/余额指标**：
1. `montoPagar`：应付金额
2. `saldoActual`：当前余额
3. `limiteCredito`：信用额度
4. `saldoVencido`：逾期余额
5. `valorActivoValuacion`：资产估值
6. `creditoMaximo`：最大授信
7. `numeroPagos`：还款次数
8. `peorAtraso`：最差逾期天数

**日期差指标（天）**：
9. `days_since_open`：距开户天数
10. `days_since_last_payment`：距最后还款天数
11. `days_since_close`：距关户天数
12. `days_pay_since_open`：开户到最后还款天数
13. `days_worst_arrears_since_open`：开户到最差逾期天数
14. `days_since_worst_arrears_date`：距最差逾期日期天数

**状态指标**：
15. `is_closed`：是否已关闭（0/1）
16. （其他状态指标）

**5 个统计值**：
- `mean`：平均值
- `std`：标准差
- `min`：最小值
- `max`：最大值
- `notNull`：有值占比

#### F. 账户类型数值画像（7 类 × 16 指标 × 5 统计 = 560 个）

与机构类型相同的统计逻辑。

#### G. 信贷类型数值画像（5 类 × 16 指标 × 5 统计 = 400 个）

与机构类型相同的统计逻辑。

**每个窗口特征数**：1 + 34 + 14 + 10 + 1360 + 560 + 400 = **2379 个**
**总特征数**：2379 × 7 窗口 + 1（request_time）= **16,654 个**

（实际是 16,647 个，可能有些特征合并或调整）

### 💡 举例说明

**场景**：张三在 2025-11-25 申请贷款

**近 30 天开户的账户**：
- 2025-11-20：银行信用卡，额度 10000，余额 5000，无逾期
- 2025-11-15：小额贷款，额度 3000，余额 2000，逾期 5 天

**输出特征**：
- `creditos_30d_total_cnt` = 2（总共 2 个账户）
- `creditos_30d_otorgante_bank_cnt` = 1（银行账户 1 个）
- `creditos_30d_otorgante_bank_ratio` = 0.5（银行账户占比 50%）
- `creditos_30d_tipo_cuenta_r_cnt` = 1（循环信用 1 个，信用卡）
- `creditos_30d_tipo_credito_tc_cnt` = 1（信用卡 1 个）
- `creditos_30d_otorgante_bank_limiteCredito_mean` = 10000（银行平均额度）
- `creditos_30d_otorgante_bank_saldoActual_mean` = 5000（银行平均余额）
- `creditos_30d_otorgante_micro_loan_saldoVencido_max` = 2000（小额贷款最大逾期余额）

### ⚠️ 异常处理

与第一板块相同的 A/C/D 规则。

## 第三板块：clavePrevencion（预防类型板块）

### 📊 数据来源

从 `response_body.creditos` 解析信贷账户记录（与第二板块相同数据源）。

### 🎯 核心逻辑

**用大白话说**：统计"我的信贷账户处于什么状态（正常关闭、催收、违约等）"。

### 📅 时间窗口

使用 **`fechaAperturaCuenta`（开户日期）** 作为窗口筛选字段。

```
窗口筛选条件：
request_time - fechaAperturaCuenta <= N 天
```

**7 个时间窗口**：30、60、90、120、180、360、720 天

### 🔢 计算维度

#### 1. 按预防类型分组（5 类）

根据 `clavePrevencion`（预防类型）分组：

| 类型代码 | 说明 |
|---------|------|
| CL | 账户已关闭，客户已全额支付且未造成违约，账户正常关闭 |
| IA | 账户已停用，账户余额为零，处于非活动状态 |
| LC | 与消费者达成减免或分期付款协议（协商还款安排） |
| PC | 账户进入催收 |
| UP | 导致违约事件的账户 |

#### 2. 按责任类型分组（3 类）

根据 `tipoResponsabilidad`（责任类型）分组：

| 类型代码 | 说明 |
|---------|------|
| I | 个人责任（Individual） |
| M | 联名责任（Mancomunado） |
| A | 担保责任（Aval） |

### 📈 输出特征

**每个窗口 N 天内，输出以下特征**：

#### A. 总量特征（1 个）
- `creditos_{N}d_total_cnt`：窗口内信贷账户总数

#### B. 预防类型特征（5 类 × 2 = 10 个）
- `creditos_{N}d_prev_{类型ID}_cnt`：该预防类型账户数
- `creditos_{N}d_prev_{类型ID}_ratio`：该预防类型账户占比

#### C. 责任类型特征（3 类 × 2 = 6 个）
- `creditos_{N}d_resp_{类型ID}_cnt`：该责任类型账户数
- `creditos_{N}d_resp_{类型ID}_ratio`：该责任类型账户占比

**每个窗口特征数**：1 + 10 + 6 = **17 个**
**总特征数**：17 × 7 窗口 + 1（request_time）= **120 个**

### 💡 举例说明

**场景**：张三在 2025-11-25 申请贷款

**近 30 天开户的账户**：
- 2025-11-20：银行信用卡，预防类型 = CL（正常关闭），责任类型 = I（个人）
- 2025-11-15：小额贷款，预防类型 = PC（催收），责任类型 = I（个人）
- 2025-11-10：汽车贷款，预防类型 = 空，责任类型 = M（联名）

**输出特征**：
- `creditos_30d_total_cnt` = 3（总共 3 个账户）
- `creditos_30d_prev_cl_cnt` = 1（正常关闭 1 个）
- `creditos_30d_prev_cl_ratio` = 0.33（正常关闭占比 33%）
- `creditos_30d_prev_pc_cnt` = 1（催收 1 个）
- `creditos_30d_prev_pc_ratio` = 0.33（催收占比 33%）
- `creditos_30d_resp_i_cnt` = 2（个人责任 2 个）
- `creditos_30d_resp_i_ratio` = 0.67（个人责任占比 67%）
- `creditos_30d_resp_m_cnt` = 1（联名责任 1 个）
- `creditos_30d_resp_m_ratio` = 0.33（联名责任占比 33%）

### 🎯 与第二板块的区别

| 维度 | 第二板块 | 第三板块 |
|------|---------|---------|
| **数据源** | creditos | creditos（相同） |
| **关注点** | 账户金额、余额、额度、逾期 | 账户状态、责任类型 |
| **分组维度** | 机构类型、账户类型、信贷类型 | 预防类型、责任类型 |
| **统计内容** | 金额统计（mean/std/min/max） | 计数统计（cnt/ratio） |
| **特征数量** | 16,647 | 120 |
| **复杂度** | 高（多维度、多指标） | 低（简单计数） |

### ⚠️ 异常处理

与第一、二板块相同的 A/C/D 规则。

## BOSS板块：综合特征板块

### 📊 数据来源

从 `response_body` 的**四个子板块**解析数据：

1. **consultas**（查询记录）：412,045 条
2. **creditos**（信贷账户）：697,815 条
3. **empleos**（工作记录）：5,188 条
4. **domicilios**（住址记录）：37,272 条

### 🎯 核心逻辑

**用大白话说**：综合所有数据，计算"业务上最关心的核心指标"，比如账户使用率、负债收入比、工作稳定性等。

### 📅 时间窗口

根据不同特征使用不同的窗口字段：

| 特征类型 | 窗口字段 | 窗口长度 |
|---------|---------|---------|
| 查询特征 | `fechaConsulta` | 30/60/90/180/360/720 天 |
| 信贷特征 | `fechaReporte` | 30/60/90/180/360/720 天 |
| 工作特征 | `fechaContratacion` | 全量（不分窗口） |
| 住址特征 | `fechaResidencia` | 全量（不分窗口） |

### 🔢 特征分类

#### 1. 查询板块特征（约 30-40 个）

**窗口内查询统计**：
- 查询金额：sum、max、mean
- 查询次数：total_cnt
- 查询类型：车贷查询次数、信用卡查询次数

**全量查询特征**：
- 查询类型数量
- 平均查询间隔（天）
- 查询频率（次/月）
- 查询激增标记

**示例特征**：
```
cdc_consultas_30d_importeCredito_sum_607      # 近30天查询金额总和
cdc_consultas_60d_importeCredito_max_607      # 近60天查询金额最大值
cdc_consultas_tipo_AF_cnt_607                 # 汽车金融查询次数
cdc_consultas_tipo_TC_cnt_607                 # 信用卡查询次数
cdc_consultas_avg_interval_days_607           # 平均查询间隔天数
```

#### 2. 信贷板块特征（约 80-100 个）

**账户数量/状态**：
- 总账户数
- 开放账户数
- 关闭账户数
- 零余额账户数
- 个人/联名/担保账户数

**账户使用率**：
- 账户平均使用率 = saldoActual / limiteCredito
- 信用卡使用率
- 高使用率账户数（>80%）
- 总使用率 = Σ saldoActual / Σ limiteCredito

**收入比指标**：
- 授信收入比 = Σ limiteCredito / 月收入
- 负债收入比 = Σ saldoActual / 月收入
- 逾期收入比 = Σ saldoVencido / 月收入
- 还款收入比 = Σ montoPagar / 月收入

**时效特征**：
- 距最后销户天数
- 距最后查询天数
- 距最后逾期天数
- 距最后还款天数
- 距最后消费天数
- 距最后报告天数
- 距最后更新天数
- 距最后零余额天数

**逾期特征**：
- 最差逾期状态（max peorAtraso）
- 还款期数均值/总和
- 已报告还款次数均值/总和
- 还款频率众数映射

**示例特征**：
```
cdc_creditos_total_accounts_607               # 总账户数
cdc_creditos_open_accounts_607                # 开放账户数
cdc_creditos_avg_utilization_607              # 平均使用率
cdc_creditos_credit_to_income_ratio_607       # 授信收入比
cdc_creditos_debt_to_income_ratio_607         # 负债收入比
cdc_creditos_days_since_last_payment_607      # 距最后还款天数
cdc_creditos_worst_arrears_status_607         # 最差逾期状态
```

#### 3. 工作板块特征（约 10-15 个）

**工作记录统计**：
- 工作记录数
- 平均工作月数
- 最长工作月数
- 当前工作月数

**薪资统计**：
- 平均月薪
- 当前月薪
- 历史最高月薪

**稳定性指标**：
- 工作变更次数
- 工作稳定性得分
- 薪资已验证标记

**示例特征**：
```
cdc_empleos_total_jobs_607                    # 工作记录数
cdc_empleos_avg_job_months_607                # 平均工作月数
cdc_empleos_current_job_months_607            # 当前工作月数
cdc_empleos_avg_salary_607                    # 平均月薪
```
