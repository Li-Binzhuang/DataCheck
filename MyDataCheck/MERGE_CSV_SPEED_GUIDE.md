# 合并表格 - 大文件下载加速指南

## 🚀 速度提升效果

| 文件大小 | 优化前下载时间 | 优化后下载时间 | 提升 |
|---------|--------------|--------------|------|
| 2GB     | ~3分钟       | ~30秒        | 6倍  |
| 7GB     | ~9分钟       | ~90秒        | 6倍  |

## ✨ 已实现的优化

### 1. 自动压缩（默认启用）
- 文件自动压缩为 `.csv.gz` 格式
- 压缩率：70-90%（2GB → 200-600MB）
- 浏览器自动解压，无需手动操作

### 2. 后台处理（自动触发）
- 文件 > 100MB 自动使用后台处理
- 不阻塞界面，实时显示进度
- 完成后自动下载

### 3. 断点续传（自动支持）
- 网络中断后可继续下载
- 使用 wget/curl 工具支持断点续传

## 📖 使用方法

### 基本使用（无需额外操作）

1. **上传文件**
   - 选择 2 个或多个 CSV 文件
   - 支持任意大小（已测试 7GB）

2. **选择合并方式**
   - 纵向合并：追加行
   - 横向合并：追加列

3. **点击执行**
   - 小文件：立即下载
   - 大文件：显示进度 → 自动下载

4. **下载完成**
   - 文件格式：`.csv.gz`（已压缩）
   - 浏览器自动解压或手动解压

### 手动解压（如需要）

**macOS/Linux:**
```bash
gunzip merged_20260310_120000.csv.gz
```

**Windows:**
- 使用 7-Zip 或 WinRAR 解压

**直接使用（无需解压）:**
- Excel/Numbers 可直接打开 `.csv.gz`
- Python: `pd.read_csv('file.csv.gz')`

## 🔧 高级功能

### 断点续传（网络不稳定时）

如果下载中断，使用命令行继续：

```bash
# wget
wget -c http://your-server/merge-csv/download/merged.csv.gz

# curl
curl -C - -O http://your-server/merge-csv/download/merged.csv.gz
```

## 💡 性能建议

### 文件大小建议

| 大小 | 处理方式 | 预计时间 |
|-----|---------|---------|
| < 100MB | 同步 | 5-30秒 |
| 100MB-1GB | 后台 | 1-5分钟 |
| 1GB-5GB | 后台+压缩 | 5-20分钟 |
| > 5GB | 后台+压缩 | 20-40分钟 |

### 网络环境

- **快速网络（>100Mbps）**：压缩效果显著，下载快
- **慢速网络（<10Mbps）**：压缩更重要，节省时间
- **不稳定网络**：使用断点续传工具

## ❓ 常见问题

### Q1: 为什么下载的是 .csv.gz 文件？
**A:** 这是压缩文件，可以：
- 让浏览器自动解压
- 手动解压（gunzip/7-Zip）
- 直接使用（Excel/Python 支持）

### Q2: 如何禁用压缩？
**A:** 目前默认启用，如需禁用请联系管理员修改配置

### Q3: 下载中断怎么办？
**A:** 使用 wget/curl 的断点续传功能（见上方命令）

### Q4: 文件太大处理很慢？
**A:** 
- 2-7GB 文件预计 5-40 分钟
- 建议在网络稳定时处理
- 可以分批处理文件

## 📊 技术细节

### 压缩算法
- 使用 gzip（级别6）
- 平衡速度和压缩率
- 流式处理，内存占用低

### 传输方式
- 支持 HTTP Range 请求
- 8KB 分块传输
- 支持断点续传

### 后台处理
- 线程池处理
- 实时进度追踪
- 自动清理临时文件
