# 接口异常处理优化

## 问题描述

接口数据对比时，如果接口请求失败，会导致：
1. 所有失败的请求都被计入失败统计（如：成功: 0, 失败: 12546）
2. 失败的行会写入分析报告，影响正常数据的对比结果
3. 无法单独查看哪些行请求失败，以及失败原因

## 修复方案

### 1. 异常处理逻辑优化

**修改前**：
- 接口请求失败的行会写入 `_analysis_report.csv`
- 失败行标记为"错误"，但与正常对比数据混在一起
- 无法单独查看失败记录

**修改后**：
- 接口请求失败的行**跳过对比**，不写入分析报告
- 失败记录单独收集，写入 `_errors.csv` 文件
- 如果没有失败记录，则不生成错误文件

### 2. 错误文件格式

生成的 `_errors.csv` 文件包含以下列：
- **applyId**：申请ID（从CSV文件中提取）
- **baseTime**：基准时间（从CSV文件中提取）
- **errors**：接口请求失败的详细错误信息

示例：
```csv
applyId,baseTime,errors
1073458287442403329,2025-11-27T14:15:48.503,请求失败 - params={'applyId': '1073458287442403329'}, 错误: Expecting ':' delimiter: line 1 column 811011
1073488146055061505,2025-11-27T14:30:17.436,请求失败 - params={'applyId': '1073488146055061505'}, 错误: Response ended prematurely
1073488146055061506,2025-11-27T14:35:20.123,JSON解析失败 - params={'applyId': '1073488146055061506'}, 错误: Expecting value: line 1 column 1
```

### 3. 统计信息优化

**修改前**：
```
成功: 0, 失败: 12546
```

**修改后**：
```
成功: 10000, 失败: 2546

⚠️  发现 2546 个接口请求异常，正在写入错误文件...
✅ 错误文件已生成: bz_new1_01271950_errors.csv
```

如果所有请求都失败：
```
成功: 0, 失败: 12546

❌ 所有接口请求均失败，无法生成对比报告
  请检查:
  1. 接口URL是否正确
  2. 接口是否可访问
  3. 接口参数配置是否正确
  4. 查看错误文件了解详细信息: bz_new1_01271950_errors.csv
```

### 4. 特征统计优化

- 特征统计**只统计成功的行**
- 失败的行不影响特征匹配率计算
- 确保统计结果的准确性

## 修复效果

✅ **异常隔离**：接口请求失败的行不影响正常数据对比  
✅ **单独记录**：失败记录单独输出到 `_errors.csv` 文件  
✅ **按需生成**：如果没有失败记录，则不生成错误文件  
✅ **详细信息**：错误文件包含完整的原始数据和错误信息  
✅ **准确统计**：特征统计只基于成功的行，结果更准确  
✅ **友好提示**：清晰的错误提示和排查建议  

## 输出文件说明

执行接口数据对比后，会生成以下文件：

### 1. 分析报告（必定生成）
- **文件名**：`{前缀}_{时间戳}_analysis_report.csv`
- **内容**：只包含成功请求且有差异的行
- **用途**：查看哪些数据存在差异

### 2. 特征统计（成功时生成）
- **文件名**：`{前缀}_{时间戳}_feature_stats.csv`
- **内容**：每个特征的匹配率统计
- **用途**：分析哪些特征容易出现差异

### 3. 错误记录（有异常时生成）
- **文件名**：`{前缀}_{时间戳}_errors.csv`
- **内容**：接口请求失败的记录
- **格式**：
  - applyId：申请ID
  - baseTime：基准时间
  - errors：错误信息
- **用途**：排查接口请求失败的原因

**示例**：
```csv
applyId,baseTime,errors
1073458287442403329,2025-11-27T14:15:48.503,请求失败 - 错误: Expecting ':' delimiter
1073488146055061505,2025-11-27T14:30:17.436,请求失败 - 错误: Response ended prematurely
```

## 常见错误及解决方案

### 1. JSON 解析错误
```
错误: Expecting ':' delimiter: line 1 column 811011
```
**原因**：接口返回的JSON格式错误或不完整  
**解决**：
- 检查接口是否正常
- 查看接口返回的响应体
- 联系接口提供方修复

### 2. 响应中断错误
```
错误: Response ended prematurely
```
**原因**：服务器在发送完整响应前关闭连接  
**解决**：
- 增加超时时间（timeout配置）
- 检查网络连接
- 检查服务器资源

### 3. 连接超时
```
错误: 请求超时 - 超时时间: 60秒
```
**原因**：接口响应时间过长  
**解决**：
- 增加 timeout 配置（如改为 120）
- 检查接口性能
- 减少并发线程数

### 4. 连接错误
```
错误: 连接错误 - Connection refused
```
**原因**：无法连接到接口服务器  
**解决**：
- 检查接口URL是否正确
- 检查网络连接
- 确认接口服务是否启动

## 使用建议

1. **执行前检查**：
   - 确认接口URL正确
   - 确认接口参数配置正确
   - 先用少量数据测试

2. **执行中监控**：
   - 观察日志输出
   - 注意错误提示
   - 每100个错误会打印一次

3. **执行后分析**：
   - 查看成功/失败统计
   - 如果有错误文件，优先排查错误原因
   - 分析特征统计，了解数据质量

4. **错误排查**：
   - 打开 `_errors.csv` 查看详细错误信息
   - 根据错误类型采取相应措施
   - 修复后重新执行对比
