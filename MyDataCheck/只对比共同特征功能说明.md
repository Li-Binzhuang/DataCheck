# 只对比共同特征功能说明

## 功能概述

在CSV数据对比模块中新增了"只对比共同特征"功能，允许用户选择只对比两个文件中都存在的特征列，自动忽略单边存在的列。

## 使用场景

当两个文件的特征列不完全一致时，可以选择只对比共同的列，避免因列缺失而导致的对比失败或错误报告。

### 示例场景

**场景1：file1有额外的特征列**
- file1的特征列：`age`, `income`, `score`, `pt`
- file2的特征列：`age`, `income`, `score`
- 不启用"只对比共同特征"：会报告`pt`列在file2中缺失
- 启用"只对比共同特征"：只对比`age`, `income`, `score`，自动忽略`pt`列

**场景2：结合特征名称映射使用**
- file1的特征列：`age`, `income`, `pt`, `score`（映射后为`local_all_sms_age_v3`, `local_all_sms_income_v3`, `local_all_sms_pt_v3`, `local_all_sms_score_v3`）
- file2的特征列：`local_all_sms_age_v3`, `local_all_sms_income_v3`, `local_all_sms_score_v3`
- 启用"只对比共同特征"：只对比3个共同列，自动排除`local_all_sms_pt_v3`

## 功能配置

### Web界面配置

1. 在"数据对比"页面，找到"只对比共同特征"选项（位于"启用特征名称映射"下方）
2. 勾选该选项后，系统会自动筛选共同特征列
3. 配置参数：
   - **只对比共同特征**：勾选后启用，默认不勾选

### 配置示例

```json
{
  "scenarios": [{
    "name": "当前配置",
    "enabled": true,
    "sql_file": "file1.csv",
    "api_file": "file2.csv",
    "sql_key_column": [0],
    "api_key_column": [0],
    "sql_feature_start": 1,
    "api_feature_start": 1,
    "convert_feature_to_number": true,
    "enable_column_mapping": true,
    "mapping_file": "file1",
    "mapping_prefix": "local_all_sms_",
    "mapping_suffix": "_v3",
    "compare_common_features_only": true
  }]
}
```

## 功能特点

### 1. 默认行为
- 默认不启用（保持现有逻辑）
- 不勾选时，会对比所有特征列，报告缺失列
- 勾选后，只对比共同列，不报告单边存在的列

### 2. 执行顺序
```
1. 读取文件
   ↓
2. 根据特征起始列索引提取特征列
   ↓
3. 应用特征名称映射（如果启用）
   ↓
4. 筛选共同特征（如果启用"只对比共同特征"）
   - 计算两个文件特征列的交集
   - 只保留共同的列
   ↓
5. 执行数据对比
```

### 3. 日志输出
启用后会输出详细的筛选信息：
```
[特征列筛选] 只对比共同特征: 启用
  file1特征列数: 50
  file2特征列数: 48
  共同特征列数: 47
  file1独有列 (1个): local_all_sms_pt_v3
  file2独有列: (无)
  将只对比47个共同特征列
```

## 使用步骤

### 步骤1：上传文件
上传两个需要对比的CSV文件

### 步骤2：配置基本参数
- 设置主键列索引
- 设置特征起始列索引
- 设置其他对比选项

### 步骤3：配置特征名称映射（可选）
如果需要统一列名格式，可以启用特征名称映射

### 步骤4：启用"只对比共同特征"（可选）
勾选"只对比共同特征"复选框

### 步骤5：执行对比
点击"执行对比"按钮，系统会：
1. 读取两个文件
2. 应用特征名称映射（如果启用）
3. 筛选共同特征列（如果启用）
4. 只对比共同列的值
5. 生成对比报告

## 注意事项

1. **特征起始列索引不变**：仍然按指定的索引提取特征列
2. **主键列不受影响**：主键列的处理完全不变
3. **映射后再筛选**：先应用特征名称映射，再筛选共同列
4. **配置可保存**：配置会保存到`data_comparison/config.json`
5. **向后兼容**：默认不启用，保持现有逻辑不变
6. **无共同列会报错**：如果两个文件没有共同特征列，会抛出错误

## 常见问题

### Q1：启用后会影响主键列吗？
A：不会。主键列的处理完全不变，只影响特征列的筛选。

### Q2：如果两个文件没有共同特征列会怎样？
A：系统会抛出错误提示："错误: 两个文件没有共同的特征列，无法进行对比！请检查特征名称映射配置。"

### Q3：特征起始列索引还有效吗？
A：有效。系统仍然按指定的索引提取特征列，只是在提取后会筛选共同列。

### Q4：可以和特征名称映射一起使用吗？
A：可以。推荐的使用顺序是：先启用特征名称映射统一列名，再启用只对比共同特征筛选列。

### Q5：如何查看哪些列被排除了？
A：执行对比时，日志会输出详细信息，包括file1独有列和file2独有列的列表。
