# 自动清理旧文件功能说明

## 功能概述

自动清理 `outputdata` 和 `inputdata` 目录下超过指定天数的 `.csv` 文件，避免磁盘空间被占满。

## 核心特性

- ✅ 自动清理旧CSV文件
- ✅ 可配置保留天数（默认5天）
- ✅ 支持试运行模式（查看但不删除）
- ✅ 递归扫描所有子目录
- ✅ 详细的统计信息
- ✅ 定时任务自动执行
- ✅ 日志记录

## 文件说明

### 1. cleanup_old_files.py

**核心清理脚本**

功能：
- 扫描指定目录下的所有CSV文件
- 根据文件修改时间判断是否过期
- 删除超过保留天数的文件
- 生成详细的统计报告

### 2. setup_auto_cleanup.sh

**自动化配置脚本**

功能：
- 配置 cron 定时任务
- 每天凌晨2点自动执行清理
- 自动创建日志目录
- 检查并替换已存在的任务

### 3. cleanup_now.sh

**手动清理脚本**

功能：
- 立即执行清理
- 交互式确认
- 支持自定义参数
- 方便测试和手动清理

## 使用方法

### 方法1: 设置自动清理（推荐）

```bash
cd MyDataCheck

# 设置自动清理任务
chmod +x setup_auto_cleanup.sh
./setup_auto_cleanup.sh
```

**效果**:
- 每天凌晨2点自动执行
- 删除5天前的CSV文件
- 日志保存到 `logs/cleanup.log`

### 方法2: 手动清理

```bash
cd MyDataCheck

# 试运行（查看将要删除的文件，不会真正删除）
chmod +x cleanup_now.sh
./cleanup_now.sh --dry-run

# 正式清理（会删除文件）
./cleanup_now.sh

# 自定义保留天数（删除7天前的文件）
./cleanup_now.sh --days 7

# 试运行 + 自定义天数
./cleanup_now.sh --days 3 --dry-run
```

### 方法3: 直接使用Python脚本

```bash
cd MyDataCheck

# 试运行
python cleanup_old_files.py --dry-run

# 正式运行
python cleanup_old_files.py

# 自定义保留天数
python cleanup_old_files.py --days 7

# 指定基础目录
python cleanup_old_files.py --base-dir /path/to/MyDataCheck
```

## 命令行参数

### cleanup_old_files.py

| 参数 | 说明 | 默认值 |
|-----|------|--------|
| --days N | 保留最近N天的文件 | 5 |
| --dry-run | 试运行模式（不删除） | False |
| --base-dir PATH | 指定基础目录 | 脚本所在目录 |

### cleanup_now.sh

| 参数 | 说明 | 默认值 |
|-----|------|--------|
| --days N | 保留最近N天的文件 | 5 |
| --dry-run | 试运行模式（不删除） | False |
| --help, -h | 显示帮助信息 | - |

## Cron 时间格式说明

```
* * * * * 命令
│ │ │ │ │
│ │ │ │ └─── 星期几 (0-7, 0和7都表示星期日)
│ │ │ └───── 月份 (1-12)
│ │ └─────── 日期 (1-31)
│ └───────── 小时 (0-23)
└─────────── 分钟 (0-59)
```

**常用示例**:

| Cron 表达式 | 说明 |
|-----------|------|
| `0 2 * * *` | 每天凌晨2点 |
| `0 */6 * * *` | 每6小时 |
| `0 0 * * 0` | 每周日凌晨 |
| `0 0 1 * *` | 每月1号凌晨 |
| `30 3 * * *` | 每天凌晨3:30 |

## 清理规则

### 判断标准

文件是否被删除取决于：
1. **文件类型**: 只删除 `.csv` 文件
2. **文件年龄**: 根据文件修改时间计算
3. **保留天数**: 超过指定天数的文件会被删除

### 计算方式

```python
文件年龄（天） = (当前时间 - 文件修改时间) / 86400秒

if 文件年龄 > 保留天数:
    删除文件
```

### 扫描范围

```
MyDataCheck/
├── outputdata/           ← 扫描此目录
│   ├── api_comparison/   ← 递归扫描子目录
│   ├── data_comparison/
│   └── online_comparison/
└── inputdata/            ← 扫描此目录
    ├── api_comparison/   ← 递归扫描子目录
    ├── data_comparison/
    └── online_comparison/
```

## 常见问题

### Q: 如何修改保留天数？

A: 有三种方法：

1. **修改定时任务**:
   ```bash
   crontab -e
   # 将 --days 5 改为 --days 7
   ```

2. **手动清理时指定**:
   ```bash
   ./cleanup_now.sh --days 7
   ```

3. **直接使用Python脚本**:
   ```bash
   python cleanup_old_files.py --days 7
   ```

### Q: 如何修改执行时间？

A: 编辑 crontab：

```bash
crontab -e

# 修改时间（例如改为每天凌晨3点）
0 3 * * * cd /path/to/MyDataCheck && python cleanup_old_files.py --days 5 >> logs/cleanup.log 2>&1
```

### Q: 如何查看清理历史？

A: 查看日志文件：

```bash
# 查看最新日志
tail -f MyDataCheck/logs/cleanup.log

# 搜索特定日期的清理记录
grep "2026-01-27" MyDataCheck/logs/cleanup.log
```

### Q: 误删文件怎么办？

A: 建议：

1. **使用试运行模式**: 先 `--dry-run` 查看
2. **定期备份**: 重要文件定期备份
3. **调整保留天数**: 如果担心误删，增加保留天数

### Q: 清理任务没有执行？

A: 检查以下几点：

1. **cron 服务是否运行**:
   ```bash
   # macOS
   sudo launchctl list | grep cron
   
   # Linux
   systemctl status cron
   ```

2. **crontab 是否正确**:
   ```bash
   crontab -l
   ```

3. **Python 路径是否正确**:
   ```bash
   which python3
   ```

4. **查看系统日志**:
   ```bash
   # macOS
   tail -f /var/log/system.log | grep cron
   
   # Linux
   tail -f /var/log/syslog | grep CRON
   ```

### Q: 如何临时禁用自动清理？

A: 注释掉 crontab 中的任务：

```bash
crontab -e

# 在任务前添加 # 注释
# 0 2 * * * cd /path/to/MyDataCheck && python cleanup_old_files.py --days 5 >> logs/cleanup.log 2>&1
```
