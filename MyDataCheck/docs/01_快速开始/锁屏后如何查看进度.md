# 锁屏后如何查看真实进度

## 问题现象

锁屏几分钟后，重新打开浏览器发现进度增长很少，怀疑任务暂停了。

## 原因分析

### ✅ 任务没有暂停
- 数据对比任务在**服务器端**执行，不受锁屏影响
- 后台线程持续运行，接口请求正常进行
- 文件正常写入

### ⚠️ 但浏览器显示可能不准确
- 浏览器在锁屏后会降低后台标签页优先级
- SSE（实时输出）连接可能被暂停或降频
- 进度更新消息堆积，解锁后才一次性显示
- **你看到的进度不是实时的**

## 验证方法

### 方法1: 监控脚本（推荐）

```bash
# 运行监控脚本
cd MyDataCheck
./scripts/maintenance/monitor_progress.sh

# 或者持续监控（每5秒刷新）
watch -n 5 './scripts/maintenance/monitor_progress.sh'
```

这个脚本会显示：
- Web服务运行状态
- 最近生成的文件
- 正在写入的文件大小和行数
- 最新的任务日志

### 方法2: 手动检查文件

```bash
# 1. 查看输出目录的文件（按时间排序）
ls -lht MyDataCheck/outputdata/api_comparison/ | head -10

# 2. 实时监控文件大小变化
watch -n 2 'du -h MyDataCheck/outputdata/api_comparison/*.csv'

# 3. 查看文件行数（判断写入进度）
wc -l MyDataCheck/outputdata/api_comparison/bz_cdc1_all_*.csv

# 4. 持续监控行数变化
watch -n 5 'wc -l MyDataCheck/outputdata/api_comparison/bz_cdc1_all_*.csv'
```

### 方法3: 查看进程状态

```bash
# 1. 确认Web服务在运行
ps aux | grep web_app.py

# 2. 查看CPU和内存使用
top -p $(pgrep -f web_app.py)

# 3. 查看网络连接（确认在请求接口）
netstat -an | grep ESTABLISHED | grep "8081\|8082"
```

### 方法4: 查看任务日志

```bash
# 查看最新的任务日志
ls -t MyDataCheck/logs/tasks/*.log | head -1 | xargs tail -f

# 或者查看最后50行
ls -t MyDataCheck/logs/tasks/*.log | head -1 | xargs tail -50
```

## 进度缓慢的真实原因

如果确认任务在运行，但进度确实很慢，可能是：

### 1. 接口响应慢
- 接口服务器性能不足
- 网络延迟高
- 接口有限流

**解决方案**：
```json
// 在 config.json 中调整
{
  "timeout": 15,        // 降低超时时间（从30秒到15秒）
  "thread_count": 100   // 降低并发数（从200到100）
}
```

### 2. 大量请求超时
- 每个超时请求会等待30秒
- 如果有很多失败请求，会严重拖慢进度

**查看失败记录**：
```bash
# 查看错误记录文件
cat MyDataCheck/outputdata/api_comparison/*_error_records.csv
```

### 3. 数据量太大
- 如果有几万条数据，即使200并发也需要较长时间
- 例如：10000条数据，200并发，每个请求2秒 = 约100秒

**计算预期时间**：
```
总时间 ≈ (总行数 / 并发数) × 平均响应时间
```

## 最佳实践

### 1. 使用后台运行（推荐）

不依赖浏览器，任务更稳定：

```bash
# 使用 screen
screen -S mydatacheck
cd MyDataCheck
python web_app.py

# 按 Ctrl+A, D 分离会话
# 锁屏后重新连接: screen -r mydatacheck
```

### 2. 定期检查进度

```bash
# 每10秒检查一次文件行数
while true; do
  clear
  echo "=== 当前时间: $(date) ==="
  wc -l MyDataCheck/outputdata/api_comparison/*.csv 2>/dev/null
  sleep 10
done
```

### 3. 优化配置

根据接口性能调整：

```json
{
  "scenarios": [
    {
      "thread_count": 100,  // 并发数：100-200
      "timeout": 15,        // 超时：15-30秒
      ...
    }
  ]
}
```

### 4. 分批处理大数据

如果数据量特别大（>10万行），建议分批：

```bash
# 将大CSV文件分割成多个小文件
split -l 10000 large_file.csv part_

# 分别处理每个文件
```

## 常见问题

### Q1: 锁屏后任务会暂停吗？
**A**: ❌ 不会。任务在服务器端运行，不受锁屏影响。

### Q2: 为什么浏览器显示的进度不动？
**A**: 浏览器在锁屏后降低了后台标签页的优先级，SSE连接可能被暂停。**实际任务仍在运行**。

### Q3: 如何确认任务真的在运行？
**A**: 
1. 运行监控脚本: `./scripts/maintenance/monitor_progress.sh`
2. 查看文件大小: `watch -n 5 'du -h MyDataCheck/outputdata/api_comparison/*.csv'`
3. 查看进程: `ps aux | grep web_app.py`

### Q4: 进度确实很慢怎么办？
**A**: 
1. 检查接口响应时间
2. 查看错误记录文件（*_error_records.csv）
3. 降低超时时间和并发数
4. 考虑分批处理

### Q5: 可以关闭浏览器吗？
**A**: ✅ 可以。任务会继续执行，完成后查看输出目录即可。

## 总结

| 操作 | 任务是否继续 | 进度显示 |
|------|------------|---------|
| 锁屏 | ✅ 继续执行 | ⚠️ 可能不准确 |
| 关闭浏览器 | ✅ 继续执行 | ❌ 无法查看 |
| 关闭Web服务 | ❌ 任务中断 | ❌ 无法查看 |

**建议**：
1. ✅ 使用监控脚本查看真实进度
2. ✅ 不要依赖浏览器显示
3. ✅ 使用 screen 后台运行
4. ✅ 定期检查输出文件
