# 数据对比功能优化说明 v1.2

## 📋 更新概述

本次更新优化了数据对比功能的界面文字和 cust_no 字段的获取逻辑。

## ✨ 更新内容

### 1. 界面文字优化

将界面中的"文件1"和"文件2"改为更具业务含义的名称：

| 原名称 | 新名称 | 说明 |
|--------|--------|------|
| 文件1 | Sql文件 | 表示来自SQL查询的数据 |
| 文件2 | 接口文件 | 表示来自接口返回的数据 |

**修改位置**：
- 文件上传标签
- 主键列索引标签
- 特征起始列索引标签
- 日志输出信息

**优化效果**：
- ✅ 更直观地表达数据来源
- ✅ 符合实际业务场景
- ✅ 减少用户理解成本

### 2. cust_no 字段获取逻辑优化

**问题描述**：
- 原逻辑只在离线文件（接口文件）中查找 cust_no
- 当 cust_no 在在线文件（Sql文件）中时，无法正确获取

**优化方案**：
- 同时在两个文件中查找 cust_no 列
- 优先从在线文件（Sql文件）获取 cust_no
- 如果在线文件中没有，则从离线文件（接口文件）获取
- 支持更多的列名变体：`cust_no`、`customer_no`、`custno`

**代码位置**：`MyDataCheck/online_comparison/job/data_comparator.py`

### 2. cust_no 获取逻辑优化

#### 查找逻辑
```python
# 新增：在在线文件中查找cust_no
cust_no_idx_online = None

# 在在线文件中查找
for i, header in enumerate(headers_online):
    header_lower = header.lower()
    if cust_no_idx_online is None and ("cust_no" in header_lower or 
                                        "customer_no" in header_lower or 
                                        "custno" in header_lower):
        cust_no_idx_online = i

# 在离线文件中查找
for i, header in enumerate(headers_offline):
    header_lower = header.lower()
    if cust_no_idx_offline is None and ("cust_no" in header_lower or 
                                         "customer_no" in header_lower or 
                                         "custno" in header_lower):
        cust_no_idx_offline = i
```

#### 获取值逻辑
```python
# 优先从在线文件（Sql文件）获取
cust_no = ""
if cust_no_idx_online is not None and online_row and cust_no_idx_online < len(online_row):
    cust_no = str(online_row[cust_no_idx_online]).strip() if online_row[cust_no_idx_online] is not None else ""

# 如果在线文件中没有找到，尝试从离线文件（接口文件）获取
if not cust_no and cust_no_idx_offline is not None and cust_no_idx_offline < len(row_offline):
    cust_no = str(row_offline[cust_no_idx_offline]).strip() if row_offline[cust_no_idx_offline] is not None else ""
```

#### 日志输出
```python
# 输出找到的列信息
if cust_no_idx_offline is not None:
    print(f"找到离线文件cust_no列: 索引{cust_no_idx_offline} ({headers_offline[cust_no_idx_offline]})")
else:
    print(f"提示: 离线文件中未找到cust_no列")

if cust_no_idx_online is not None:
    print(f"找到在线文件cust_no列: 索引{cust_no_idx_online} ({headers_online[cust_no_idx_online]})")
else:
    print(f"提示: 在线文件中未找到cust_no列")
```

## 📊 支持的 cust_no 列名

现在支持以下列名变体（不区分大小写）：

| 列名 | 说明 |
|------|------|
| `cust_no` | 标准列名 |
| `customer_no` | 完整列名 |
| `custno` | 简写列名 |
| `CUST_NO` | 大写形式 |
| `Cust_No` | 混合大小写 |

**匹配规则**：
- 不区分大小写
- 只要列名中包含上述关键字即可
- 例如：`cust_no_id`、`customer_no_code` 都会被识别

## 📝 使用示例

### 示例1：标准场景

**文件结构**：
```
Sql文件（file1）:
| cust_no | apply_id | feature1 | feature2 |
|---------|----------|----------|----------|
| 123456  | A001     | 100      | 200      |

接口文件（file2）:
| apply_id | feature1 | feature2 |
|----------|----------|----------|
| A001     | 100      | 201      |
```

**配置**：
- Sql文件主键列索引：1（apply_id）
- 接口文件主键列索引：0（apply_id）
- Sql文件特征起始列：2（feature1开始）
- 接口文件特征起始列：1（feature1开始）

**结果**：
- ✅ cust_no 从 Sql 文件中正确获取：123456
- ✅ 差异明细中显示：cust_no=123456, feature2差异（200 vs 201）

### 示例2：cust_no 在接口文件

**文件结构**：
```
Sql文件（file1）:
| apply_id | feature1 | feature2 |
|----------|----------|----------|
| A001     | 100      | 200      |

接口文件（file2）:
| apply_id | cust_no | feature1 | feature2 |
|----------|---------|----------|----------|
| A001     | 123456  | 100      | 201      |
```

**结果**：
- ✅ cust_no 从接口文件中获取：123456
- ✅ 差异明细中显示：cust_no=123456, feature2差异（200 vs 201）

### 示例3：两个文件都有 cust_no

**文件结构**：
```
Sql文件（file1）:
| cust_no | apply_id | feature1 |
|---------|----------|----------|
| 123456  | A001     | 100      |

接口文件（file2）:
| apply_id | customer_no | feature1 |
|----------|-------------|----------|
| A001     | 789012      | 101      |
```

**结果**：
- ✅ 优先使用 Sql 文件的 cust_no：123456
- ✅ 差异明细中显示：cust_no=123456, feature1差异（100 vs 101）
