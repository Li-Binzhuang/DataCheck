# 接口对比卡住问题修复

**修复时间**：2026-01-27  
**问题**：执行接口数据对比时，Web页面卡住不动  
**状态**：✅ 已修复

---

## 问题描述

用户执行接口数据对比时，Web页面显示"执行中"但一直卡住，没有任何输出。

---

## 问题原因

**`script_dir` 变量路径错误**

### 错误的配置
```python
# web/routes/api_routes.py
script_dir = SCRIPT_DIR  # 指向项目根目录 MyDataCheck/
```

### 问题分析
1. `script_dir` 被用于构建配置文件路径：
   ```python
   column_config_path = os.path.join(script_dir, "json", "column_index_config.json")
   ```

2. 实际路径应该是：
   ```
   MyDataCheck/api_comparison/json/column_index_config.json
   ```

3. 但错误配置导致路径变成：
   ```
   MyDataCheck/json/column_index_config.json  # ❌ 不存在
   ```

4. 程序尝试访问不存在的文件，导致卡住或报错

---

## 修复方案

### 修改文件
`web/routes/api_routes.py`

### 修改内容

#### 修复前
```python
api_bp = Blueprint('api_routes', __name__)

# 为了兼容性，创建小写别名
script_dir = SCRIPT_DIR  # ❌ 错误：指向项目根目录
api_input_dir = API_INPUT_DIR
api_output_dir = API_OUTPUT_DIR
```

#### 修复后
```python
api_bp = Blueprint('api_routes', __name__)

# 为了兼容性，创建小写别名
script_dir = os.path.join(SCRIPT_DIR, "api_comparison")  # ✅ 正确：指向api_comparison目录
api_input_dir = API_INPUT_DIR
api_output_dir = API_OUTPUT_DIR
```

## 其他可能的卡住原因

### 1. CSV文件不存在
**症状**：执行后立即卡住

**解决**：
```bash
# 检查文件
ls -la inputdata/api_comparison/

# 确保CSV文件已上传
```

### 2. API地址无法访问
**症状**：显示"正在请求API..."后卡住

**解决**：
- 检查API地址是否正确
- 检查网络连接
- 减少超时时间

### 3. 线程数过大
**症状**：CPU占用很高，页面卡住

**解决**：
- 减少线程数（建议50-150）
- 检查系统资源

### 4. 浏览器缓存
**症状**：页面显示异常

**解决**：
- 清除浏览器缓存
- 硬刷新（Cmd+Shift+R）
