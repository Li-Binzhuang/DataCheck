一、基础筛选条件说明
条件	含义	计算逻辑
order_asc_rank=1	最远一笔在贷订单	对未结清订单按创建时间升序排序，取rank=1
order_desc_rank=1	最近一笔在贷订单	对未结清订单按创建时间降序排序，取rank=1
order_desc_rank=2	最近第二笔在贷订单	对未结清订单按创建时间降序排序，取rank=2
order_asc_rank>1	续贷订单（非首笔）	所有非首笔的在贷订单
total_order_asc_rank=1	最远一笔订单（含已结清）	对所有订单按创建时间升序排序，取rank=1
total_order_desc_rank=1	最近一笔订单（含已结清）	对所有订单按创建时间降序排序，取rank=1
total_order_desc_rank=2	最近第二笔订单（含已结清）	对所有订单按创建时间降序排序，取rank=2
is_tqjq=1	提前结清	结清时间 < 用信窗口时间 且 结清日期 < 贷款结束日期
is_tqjq_15d=1	提前15天及以上结清	提前结清且提前天数 ≥ 15天
is_tqjq_7d=1	提前7天及以上结清	提前结清且提前天数 ≥ 7天
is_tqjq_3d=1	提前3天及以上结清	提前结清且提前天数 ≥ 3天
is_overdue=1	逾期	贷款结束日期 < 用信日期 且 (未结清 或 结清时间 > 贷款结束日期)
is_unclear_tag=1	在贷订单	未结清 或 结清时间 > 用信窗口时间

sql
-- 第1步：生成序号
row_number() over(partition by loan_no order by periods asc) as rn

-- 第2步：识别连续组
(periods - rn) as diff  -- 相同diff值的记录属于同一连续组

-- 第3步：统计每组连续长度
count(1) as cnt  -- 每组包含的期数

-- 第4步：取最大连续长度
max(cnt) as max_cnt  -- 该订单的最大连续期数
计算类型	通用公式	筛选条件
最大连续提前结清期数	max(cnt) where is_tqjq=1	筛选提前结清记录
最大连续逾期期数	max(cnt) where is_overdue=1	筛选逾期记录
最大连续提前15天结清期数	max(cnt) where is_tqjq_15d=1	筛选提前15天结清记录
最大连续提前7天结清期数	max(cnt) where is_tqjq_7d=1	筛选提前7天结清记录
最大连续提前3天结清期数	max(cnt) where is_tqjq_3d=1	筛选提前3天结清记录
2.4 比例计算
计算类型	通用公式	说明
占到期账单比例	最大连续期数 / 到期账单数	分母为已到期的账单数量
占所有期数比例	最大连续期数 / 总期数	分母为订单的总期数
占所有提前结清订单比例	最大连续期数 / 所有提前结清订单数	分母为满足条件的订单总数
占到期或已结清订单比例	最大连续期数 / (到期订单数 + 已结清订单数)	分母为已到期或已结清的订单数
2.5 本金与额度比例
计算类型	通用公式	说明
已结清本金占剩余额度比例	sum(已结清本金) / 最新可用额度	分子为符合条件的订单已还本金总和
2.6 时间间隔计算
计算类型	通用公式	说明
时间间隔（天数）	datediff(用信创建时间, 授信创建时间)	计算两个时间点之间的天数差
2.7 未来账单分布统计
计算类型	通用公式	说明
未来X-Y天未结清占未结清比例	未来X-Y天到期的未结清账单数 / 所有未结清账单数	衡量短期还款压力
未来X-Y天未结清占已结清比例	未来X-Y天到期的未结清账单数 / 所有已结清账单数	对比未来压力与历史表现
时间窗口定义：

未来0-15天：loan_end_date > use_create_time and loan_end_date <= use_create_time + 15

未来15-30天：loan_end_date > use_create_time + 15 and loan_end_date <= use_create_time + 30

未来30-60天：loan_end_date > use_create_time + 30 and loan_end_date <= use_create_time + 60

未来60-90天：loan_end_date > use_create_time + 60 and loan_end_date <= use_create_time + 90

2.8 多时间段最大值
计算类型	通用公式	说明
多时间段取最大值	greatest(值1, 值2, 值3)	从多个时间段的统计值中取最大值
