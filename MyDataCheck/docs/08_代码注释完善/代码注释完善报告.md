## 注释规范

### 模块文档字符串格式
```python
#!/usr/bin/env python
# -*- coding: utf-8 -*-
"""
模块名称

功能说明:
    - 功能点1
    - 功能点2
    - 功能点3

作者: MyDataCheck Team
创建时间: YYYY-MM
最后更新: YYYY-MM-DD
"""
```

### 函数文档字符串格式
```python
def function_name(param1, param2):
    """
    函数简短描述
    
    详细功能说明（可选）
    
    Args:
        param1 (type): 参数1说明
        param2 (type): 参数2说明
    
    Returns:
        type: 返回值说明
    
    Raises:
        ExceptionType: 异常说明（可选）
    
    示例:
        >>> function_name(value1, value2)
        expected_result
    
    Note:
        重要提示或注意事项（可选）
    """
```

### 类文档字符串格式
```python
class ClassName:
    """
    类简短描述
    
    详细功能说明
    
    属性:
        attr1 (type): 属性1说明
        attr2 (type): 属性2说明
    
    使用示例:
        >>> obj = ClassName()
        >>> obj.method()
    """
```

    sql_file_path: str,
    api_file_path: str,
    sql_key_column: int,
    api_key_column: int,

    api_feature_start: int = 1,
    convert_feature_to_number: bool = True

    """

        api_file_path (str): 接口文件路径

        api_key_column (int): 接口文件的主键列索引（从0开始）
        sql_feature_start (int): Sql文件特征列起始索引（默认1）
        api_feature_start (int): 接口文件特征列起始索引（默认1）

        dict: 包含对比结果的字典，包括：

## 最佳实践

### 1. 注释原则
- **简洁明了**：注释应该简洁，避免冗余
- **及时更新**：代码修改时同步更新注释
- **避免废话**：不要注释显而易见的代码
- **解释为什么**：注释应该解释为什么这样做，而不是做了什么

### 2. 文档字符串原则
- **完整性**：包含所有必要的信息（参数、返回值、异常）
- **准确性**：确保文档与代码实现一致
- **示例性**：提供实际使用示例
- **可读性**：使用清晰的语言和格式

### 3. 行内注释原则
- **关键逻辑**：为复杂的算法和逻辑添加注释
- **特殊处理**：为特殊情况和边界条件添加注释
- **TODO标记**：使用TODO标记待完成的工作
- **FIXME标记**：使用FIXME标记需要修复的问题
