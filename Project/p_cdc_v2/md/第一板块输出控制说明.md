# 第一板块输出控制说明

## 输出开关配置

第一板块现在有**两个独立的开关**来控制输出：

### 1. 总开关：`WRITE_OUTPUTS`

```python
WRITE_OUTPUTS = True  # True：写 outputs/*.csv；False：只在内存里生成 features/consultas_df 不落盘
```

- **True**：启用输出功能
- **False**：完全关闭输出，不生成任何 CSV 文件

### 2. 明细表开关：`WRITE_FLAT_CSV`

```python
WRITE_FLAT_CSV = False  # True：输出 consultas_flat.csv；False：跳过明细表输出（只输出特征表）
```

- **True**：输出 `consultas_flat.csv`（明细表）
- **False**：跳过明细表输出（默认）

---

## 输出文件说明

第一板块可以输出两个文件：

| 文件名 | 内容 | 控制开关 | 默认状态 |
|--------|------|---------|---------|
| `consultas_flat.csv` | consultas 原始明细数据（平铺） | `WRITE_FLAT_CSV` | ❌ 关闭 |
| `features_consultas.csv` | 第一板块衍生特征表 | `WRITE_OUTPUTS` | ✅ 开启 |

---

## 使用场景

### 场景1：只输出特征表（推荐，默认配置）

```python
WRITE_OUTPUTS = True
WRITE_FLAT_CSV = False  # 不输出明细表
```

**输出**：
- ✅ `features_consultas.csv`（特征表）
- ❌ `consultas_flat.csv`（跳过）

**适用于**：
- 只需要特征数据用于建模
- 节省磁盘空间
- 加快输出速度

---

### 场景2：同时输出明细表和特征表

```python
WRITE_OUTPUTS = True
WRITE_FLAT_CSV = True  # 输出明细表
```

**输出**：
- ✅ `features_consultas.csv`（特征表）
- ✅ `consultas_flat.csv`（明细表）

**适用于**：
- 需要核对原始数据
- 需要做数据质量检查
- 需要查看明细记录

---

### 场景3：完全关闭输出

```python
WRITE_OUTPUTS = False
WRITE_FLAT_CSV = False  # 此时这个开关无效
```

**输出**：
- ❌ 不输出任何文件

**适用于**：
- 只想在内存中计算特征
- 调试代码
- 测试运行

---

## 开关逻辑说明

### 逻辑关系

```
WRITE_OUTPUTS = True
    ├─ WRITE_FLAT_CSV = True  → 输出 consultas_flat.csv
    ├─ WRITE_FLAT_CSV = False → 跳过 consultas_flat.csv
    └─ 总是输出 features_consultas.csv

WRITE_OUTPUTS = False
    └─ 不输出任何文件（WRITE_FLAT_CSV 无效）
```

### 代码实现

```python
if WRITE_OUTPUTS:  # 总开关
    if WRITE_FLAT_CSV:  # 明细表开关
        consultas_df.to_csv(consultas_out_path, ...)  # 输出明细表
    
    # 总是输出特征表
    _features_to_write.to_csv(features_out_path, ...)
```

## 常见问题

### Q1：为什么默认关闭 `WRITE_FLAT_CSV`？

**A**：因为：
1. 明细表文件较大，占用磁盘空间
2. 大多数情况下只需要特征表
3. 明细表主要用于数据核对，不是必需的

### Q2：如果我想查看明细数据怎么办？

**A**：有两种方式：
1. 打开 `WRITE_FLAT_CSV = True`，输出到 CSV 文件
2. 在 notebook 中直接查看 `consultas_df` 变量（不需要输出文件）

```python
# 在 notebook 中查看
consultas_df.head(10)  # 查看前10行
consultas_df.shape     # 查看形状
```

### Q3：两个开关可以独立控制吗？

**A**：不完全独立：
- `WRITE_OUTPUTS = False` 时，`WRITE_FLAT_CSV` 无效（不输出任何文件）
- `WRITE_OUTPUTS = True` 时，`WRITE_FLAT_CSV` 可以独立控制明细表输出

### Q4：如果只想输出明细表，不输出特征表呢？

**A**：目前的设计不支持这种场景，因为：
- 特征表是主要产出，通常都需要
- 如果只需要明细表，可以在输出后删除特征表文件
