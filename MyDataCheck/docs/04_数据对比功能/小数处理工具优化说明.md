# 小数处理工具逻辑优化说明

## 更新日期
2026-03-05

## 优化内容

### 原有逻辑
1. 上传差异明细文件
2. 自动查找"模型特征值"和"接口值"列
3. 根据模型特征值的小数位数处理接口值
4. 在文件末尾添加处理后的列和差异值列
5. 输出有差异的记录

### 优化后的逻辑
1. **上传CSV文件**（任意CSV文件，不限于差异明细）
2. **手动指定列名**：
   - 源列（需要处理的列）
   - 参考列（提供小数位数标准的列）
3. **根据参考列的小数位数处理源列的值**
4. **处理后的值插入到源列后面作为新列**
   - 新列名 = 源列名 + 处理方式后缀
   - 例如：`API值-四舍五入`
5. **对比处理后的列和参考列**
6. **输出有差异的记录到新CSV文件**

## 主要改进

### 1. 更灵活的列选择
- **原来**：自动查找固定列名（"模型特征值"、"接口值"）
- **现在**：用户手动指定任意列名

### 2. 更清晰的列插入位置
- **原来**：在文件末尾添加新列
- **现在**：在源列后面插入新列，保持相关列的邻近性

### 3. 更明确的列命名
- **原来**：固定列名"接口/灰度/从库值-处理小数"
- **现在**：动态列名"源列名-处理方式"（如"API值-四舍五入"）

### 4. 更通用的适用场景
- **原来**：只能处理特定格式的差异明细文件
- **现在**：可以处理任意CSV文件，只要指定列名即可

## 处理逻辑说明

### 小数位数获取
根据参考列的值获取小数位数：
- `100.123` → 3位小数
- `200.45` → 2位小数
- `300` → 0位小数

### 处理方式

#### 1. 不处理
直接返回原值

#### 2. 四舍五入
使用Python的`round()`函数
```python
round(100.1234, 3) → 100.123
```

#### 3. 双精度四舍五入
先多保留一位小数四舍五入，再按目标位数四舍五入
```python
first_round = round(100.1234, 4)  # 100.1234
result = round(first_round, 3)     # 100.123
```

#### 4. 截取
直接截取，不进行四舍五入
```python
math.trunc(100.1234 * 1000) / 1000 → 100.123
```

#### 5. 向上取整
向上取整到指定小数位数
```python
math.ceil(100.1234 * 1000) / 1000 → 100.124
```

### 差异计算
```
差异值 = 参考列值 - 处理后的列值
```

### 对比方式

#### 精确对比
差异值 = 0 才不计为差异

#### 容差对比
|差异值| ≤ 容差值 时不计为差异

#### 最后一位差1不计异常
根据参考列的小数位数，最后一位允许±1误差
- 3位小数：允许±0.001
- 2位小数：允许±0.01
- 整数：允许±1

#### 最后一位差2不计异常
根据参考列的小数位数，最后一位允许±2误差
- 3位小数：允许±0.002
- 2位小数：允许±0.02
- 整数：允许±2

## 常见问题

### Q: 如何知道文件中有哪些列？
A: 可以先用Excel或文本编辑器打开CSV文件查看列名，或者查看执行日志中的"列名"输出。

### Q: 列名输入错误会怎样？
A: 系统会提示"源列 'XXX' 不存在于文件中"，请检查列名是否正确。

### Q: 可以处理多列吗？
A: 当前版本一次只能处理一对列（源列和参考列）。如需处理多列，请多次执行。

### Q: 处理后的列名太长怎么办？
A: 可以在输出文件中手动重命名列，或者在输入文件中使用较短的列名。
