# 忽略默认填充值配置说明

## 功能概述

在数据对比模块中新增了"忽略默认填充值"配置项，允许用户在对比时将 `-999` 和 `null` 视为一致。

## 使用场景

在某些业务场景中，数据缺失时会使用默认填充值（如 `-999`）或 `null` 来表示。当对比两个数据源时，如果一个使用 `-999`，另一个使用 `null`，实际上它们表达的含义是相同的（都表示数据缺失）。

启用此配置项后，对比时会将这两种表示方式视为一致，不会被标记为差异。

## 配置方式

### Web界面配置

1. 进入"数据对比"页面
2. 在对比配置区域找到"忽略默认填充值"复选框
3. 勾选该选项后，`-999` 和 `null` 会被视为一致
4. 默认状态：不勾选（不忽略默认填充值）

### 配置说明

- **默认状态**：不勾选（不忽略）
- **勾选后**：`-999` 和 `null` 被视为一致，不会被标记为差异
- **不勾选**：`-999` 和 `null` 被视为不同的值，会被标记为差异

## 对比逻辑

### 启用忽略默认填充值时

以下情况会被视为一致：
- 值1 = `-999`，值2 = `null` → 一致
- 值1 = `null`，值2 = `-999` → 一致
- 值1 = `-999`，值2 = `-999` → 一致
- 值1 = `null`，值2 = `null` → 一致

### 不启用时

- 值1 = `-999`，值2 = `null` → 不一致（差异）
- 值1 = `null`，值2 = `-999` → 不一致（差异）

## 示例

### 示例1：启用忽略默认填充值

**数据源1（模型特征表）**：
```
ID,特征A,特征B
001,-999,100
002,50,null
```

**数据源2（接口特征表）**：
```
ID,特征A,特征B
001,null,100
002,50,-999
```

**对比结果**：
- ID=001，特征A：`-999` vs `null` → 一致（因为启用了忽略默认填充值）
- ID=002，特征B：`null` vs `-999` → 一致（因为启用了忽略默认填充值）
- 无差异记录

### 示例2：不启用忽略默认填充值

使用相同的数据源：

**对比结果**：
- ID=001，特征A：`-999` vs `null` → 不一致（差异）
- ID=002，特征B：`null` vs `-999` → 不一致（差异）
- 2条差异记录

## 注意事项

1. **默认行为**：默认不勾选，保持原有的严格对比逻辑
2. **适用范围**：仅适用于数据对比模块，不影响其他模块
3. **数值类型**：只有 `-999`（数值）和 `null`（空值）会被视为默认填充值
4. **其他值**：其他数值（如 `-998`、`0`）不会被视为默认填充值
