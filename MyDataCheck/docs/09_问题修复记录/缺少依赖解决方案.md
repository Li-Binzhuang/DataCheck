# 缺少依赖解决方案

## 问题描述

启动时报错：
```
ModuleNotFoundError: No module named 'psutil'
```

## 原因

`psutil` 是新增的内存管理依赖，需要安装。

## 解决方案

### 方式一：快速安装（推荐）

```bash
cd MyDataCheck
./install_psutil.sh
```

### 方式二：手动安装

```bash
cd MyDataCheck
source .venv/bin/activate
pip install psutil
```

### 方式三：安装所有依赖

```bash
cd MyDataCheck
source .venv/bin/activate
pip install -r requirements.txt
```

## 验证安装

```bash
cd MyDataCheck
source .venv/bin/activate
python -c "import psutil; print('✅ psutil 已安装，版本:', psutil.__version__)"
```

## 完整的依赖列表

`requirements.txt` 已更新，包含：

```
Flask>=2.0.0
```

## 为什么需要 psutil？

`psutil` 用于内存监控和管理：

1. **监控内存使用** - 实时查看内存占用
2. **性能优化** - 帮助优化内存使用
3. **自动清理** - 任务完成后释放内存

这是内存优化功能的核心依赖。

## 常见问题

### Q1: pip install 很慢

**解决**: 使用国内镜像
```bash
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple psutil
```

### Q2: 权限错误

**解决**: 确保虚拟环境已激活
```bash
source .venv/bin/activate
pip install psutil
```

### Q3: 版本冲突

**解决**: 升级 pip
```bash
pip install --upgrade pip
pip install psutil
```

## 一键解决

如果遇到任何依赖问题，运行：

```bash
cd MyDataCheck
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

这会安装所有需要的依赖。
