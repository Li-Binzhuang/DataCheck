### 2. 核心模块文件

#### 2.1 数据对比器 (data_comparator.py)
- 独立的数据对比逻辑
- 支持CSV和XLSX文件
- 特征值转换功能（去除引号、字符串转数值）
- 智能识别cust_no和主键字段
- 完整的统计信息输出

**主要函数：**
- `compare_two_files()`: 对比两个文件
- `_convert_string_to_number()`: 转换字符串为数值

#### 2.2 报告生成器 (report_generator.py)
- 生成6种对比报告
- 使用UTF-8-BOM编码，Excel友好
- 清晰的数据分类和统计

**生成的报告：**
1. 差异特征汇总
2. 差异数据明细
3. 特征统计
4. 全量数据合并
5. 仅在Sql文件中的数据
6. 仅在接口文件中的数据

#### 2.3 配置管理器 (config_manager.py)
- 配置文件的保存和加载
- 场景管理（添加、更新、删除、查询）
- 支持全局默认配置

**主要函数：**
- `save_config()`: 保存配置
- `load_config()`: 加载配置
- `add_scenario()`: 添加场景
- `update_scenario()`: 更新场景
- `delete_scenario()`: 删除场景
- `get_scenario()`: 获取场景

### 3. 配置文件 (config.json)

支持多场景配置，每个场景包含：
- 场景名称和描述
- 启用/禁用状态
- Sql文件和接口文件路径
- 主键列配置
- 特征起始列配置
- 转换功能开关
- 输出前缀

**配置示例：**
```json
{
  "scenarios": [
    {
      "name": "示例场景",
      "enabled": true,
      "description": "这是一个示例配置",
      "sql_file": "file1.csv",
      "api_file": "file2.csv",
      "sql_key_column": 0,
      "api_key_column": 0,
      "sql_feature_start": 1,
      "api_feature_start": 1,
      "convert_feature_to_number": true,
      "output_prefix": "compare"
    }
  ],
  "global_config": {
    "default_convert_feature_to_number": true,
    "default_sql_key_column": 0,
    "default_api_key_column": 0,
    "default_sql_feature_start": 1,
    "default_api_feature_start": 1
  }
}
```

### 4. 执行脚本 (execute_data_comparison.py)

## 使用方法

### 方法1：通过Web界面
1. 访问Web界面的"数据对比"Tab
2. 上传Sql文件和接口文件
3. 配置主键列和特征起始列
4. 选择是否转换特征值为数值
5. 点击"执行对比"按钮
6. 配置会自动保存到config.json

### 方法2：通过命令行
```bash
cd MyDataCheck/data_comparison
python execute_data_comparison.py
```

### 方法3：通过Python代码
```python
from data_comparison.job.data_comparator import compare_two_files
from data_comparison.job.report_generator import generate_comparison_reports

# 执行对比
results = compare_two_files(
    sql_file_path="path/to/sql_file.csv",
    api_file_path="path/to/api_file.csv",
    sql_key_column=0,
    api_key_column=0,
    sql_feature_start=1,
    api_feature_start=1,
    convert_feature_to_number=True
)

# 生成报告
generate_comparison_reports("output/result", results)
```
