# CDC板块衍生脚本 - 分批输出功能说明

## 修改目标

将四个板块的输出从"全量文件+sample200"改为"分批输出多个文件"，每个文件500条数据。

---

## 修改内容

### 1. 配置部分修改

#### 修改前
```python
WRITE_OUTPUTS = True
WRITE_FLAT_CSV = True
WRITE_SAMPLE_200 = True  # ❌ 删除这行
```

#### 修改后
```python
WRITE_OUTPUTS = True
WRITE_FLAT_CSV = True
# zlf update: 改为分批输出，每个文件500条数据
BATCH_SIZE = 500  # ✅ 新增：每个文件的行数，可以根据需要调整
```

### 2. 输出逻辑修改

#### 第一板块（consultas）

**修改前的代码**（删除）：
```python
_features_to_write.to_csv(
    features_out_path,
    index=False,
    encoding="utf-8-sig",
)

if WRITE_SAMPLE_200:
    _features_sample200 = _features_to_write.head(200)
    _features_sample200.to_csv(
        features_sample200_path,
        index=False,
        encoding="utf-8-sig",
    )
```

**修改后的代码**（替换）：
```python
# zlf update: 分批输出功能 - 每个文件500条数据
total_rows = len(_features_to_write)
num_batches = (total_rows + BATCH_SIZE - 1) // BATCH_SIZE  # 向上取整

print("[INFO] 开始分批输出特征文件")
print(f"[INFO] 总数据量: {total_rows} 行")
print(f"[INFO] 批次大小: {BATCH_SIZE} 行/文件")
print(f"[INFO] 输出文件数: {num_batches} 个")
print()

batch_files = []  # 记录所有输出的文件路径

for batch_idx in range(num_batches):
    start_idx = batch_idx * BATCH_SIZE
    end_idx = min((batch_idx + 1) * BATCH_SIZE, total_rows)
    
    # 提取当前批次的数据
    batch_data = _features_to_write.iloc[start_idx:end_idx]
    
    # 生成文件名：cdc1_features_batch{批次号}_{起始行}-{结束行}.csv
    batch_filename = f"cdc1_features_batch{batch_idx + 1:03d}_{start_idx + 1}-{end_idx}.csv"
    batch_path = OUTPUT_DIR / batch_filename
    
    # 输出到CSV
    batch_data.to_csv(batch_path, index=False, encoding="utf-8-sig")
    batch_files.append(batch_path)
    
    print(f"[WRITE] 批次 {batch_idx + 1}/{num_batches}: {batch_filename}")
    print(f"        行范围: {start_idx + 1} - {end_idx} ({len(batch_data)} 行)")

print()
print("[SUCCESS] 分批输出完成！")
```

**打印输出部分修改**：
```python
print("written:")
if WRITE_FLAT_CSV:
    print("-", consultas_out_path)
for batch_file in batch_files:
    print("-", batch_file)
```

    total_rows = len(_features_to_write)
    num_batches = (total_rows + BATCH_SIZE - 1) // BATCH_SIZE
    
    print("[INFO] 开始分批输出特征文件")
    print(f"[INFO] 总数据量: {total_rows} 行")

    print()
    
    batch_files = []
    
    for batch_idx in range(num_batches):
        start_idx = batch_idx * BATCH_SIZE
        end_idx = min((batch_idx + 1) * BATCH_SIZE, total_rows)
        
        batch_data = _features_to_write.iloc[start_idx:end_idx]
        
        # 生成文件名：cdcboss_features_batch{批次号}_{起始行}-{结束行}.csv
        batch_filename = f"cdcboss_features_batch{batch_idx + 1:03d}_{start_idx + 1}-{end_idx}.csv"
        batch_path = output_dir / batch_filename
        
        batch_data.to_csv(batch_path, index=False, encoding="utf-8-sig")
        batch_files.append(batch_path)
        
        print(f"[WRITE] 批次 {batch_idx + 1}/{num_batches}: {batch_filename}")
        print(f"        行范围: {start_idx + 1} - {end_idx} ({len(batch_data)} 行)")
    
    print()
    print("[SUCCESS] 分批输出完成！")
    print("written:")
    for batch_file in batch_files:
        print("-", batch_file)
else:
    print("[INFO] WRITE_FEATURES_CSV = False，跳过特征文件输出")

## 输出文件命名规则

### 格式
```
{板块前缀}_batch{批次号:03d}_{起始行}-{结束行}.csv
```

### 示例

#### 第一板块（假设有1500行数据）

## 配置说明

### BATCH_SIZE 参数
```python
BATCH_SIZE = 500  # 每个文件的行数
```

**可以根据需要调整**：
- `BATCH_SIZE = 100`  → 每个文件100行（更小的文件）
- `BATCH_SIZE = 500`  → 每个文件500行（推荐）
- `BATCH_SIZE = 1000` → 每个文件1000行（较大的文件）

---

## 优点

### 1. 文件大小可控
- 每个文件固定行数，大小可预测
- 避免单个文件过大

### 2. 方便分批查看
- 可以只打开需要的批次文件
- 减少内存占用

### 3. 便于数据核对
- 文件名包含行范围，快速定位
- 可以按批次进行数据验证

### 4. 灵活性高
- 可以根据需要调整批次大小
- 可以选择性处理某些批次

## 注意事项

### 1. 文件命名
- 批次号使用3位数字（001, 002, ...），便于排序
- 文件名包含行范围，便于定位

### 2. 数据完整性
- 所有批次文件合并后应等于原始数据
- 每个批次的apply_id不应重复

### 3. 编码格式
- 统一使用 `utf-8-sig` 编码
- Excel可以直接打开

### 4. 精度保持
- 分批输出前已经应用了 `round(6)`
- 所有批次的数值精度一致
