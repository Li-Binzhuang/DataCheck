# 场景卡片收起功能修复报告

## 问题描述

**问题**：在"接口数据对比"和"线上灰度落数对比"页面中，点击场景卡片的收起按钮后，场景内容（描述、步骤等）仍然显示在页面中，没有被隐藏。

**影响范围**：
- 接口数据对比页面的场景卡片
- 线上灰度落数对比页面的场景卡片

**问题原因**：
缺少必要的CSS样式定义，特别是`.scenario-content`和`.scenario-content.collapsed`的样式规则。

## 修复方案

### 1. 添加场景内容区域CSS

```css
/* 场景内容区域 */
.scenario-content {
    max-height: 5000px;
    overflow: visible;
    transition: max-height 0.3s ease, opacity 0.3s ease, padding 0.3s ease;
    opacity: 1;
    padding-top: 0;
}

/* 场景内容收起状态 */
.scenario-content.collapsed {
    max-height: 0 !important;
    opacity: 0 !important;
    overflow: hidden !important;
    padding-top: 0 !important;
    padding-bottom: 0 !important;
    margin: 0 !important;
}
```

**关键点**：
- `max-height: 5000px`：展开时的最大高度，足够容纳所有内容
- `overflow: visible`：展开时允许内容正常显示
- `transition`：平滑的过渡动画
- `collapsed`状态使用`!important`确保样式优先级

## 技术细节

### CSS过渡动画
```css
transition: max-height 0.3s ease, opacity 0.3s ease, padding 0.3s ease;
```
- **max-height**：高度变化动画
- **opacity**：透明度变化动画
- **padding**：内边距变化动画
- **0.3s**：动画持续时间
- **ease**：缓动函数

### 使用!important的原因
由于可能存在其他CSS规则的干扰，使用`!important`确保收起状态的样式能够正确应用：
```css
max-height: 0 !important;
opacity: 0 !important;
overflow: hidden !important;
```

### JavaScript逻辑
现有的JavaScript逻辑已经正确实现：
```javascript
function toggleScenarioCollapse(scenarioId) {
    const content = document.getElementById(`scenario-content-${scenarioId}`);
    const toggleBtn = document.getElementById(`toggle-btn-${scenarioId}`);
    const card = document.getElementById(scenarioId);
    
    if (content.classList.contains('collapsed')) {
        // 展开
        content.classList.remove('collapsed');
        card.classList.remove('collapsed');
        // ...
    } else {
        // 收起
        content.classList.add('collapsed');
        card.classList.add('collapsed');
        // ...
    }
}
```

## 用户使用指南

### 如何使用

1. **收起场景**：
   - 点击场景标题右侧的▼按钮
   - 场景内容会平滑收起
   - 按钮变为▶

2. **展开场景**：
   - 点击场景标题右侧的▶按钮
   - 场景内容会平滑展开
   - 按钮变为▼

3. **点击标题区域**：
   - 也可以点击整个标题区域来切换收起/展开状态
   - 与点击按钮效果相同

### 使用场景

**适合收起场景的情况**：
- 场景配置已完成，不需要修改
- 有多个场景，需要查看其他场景
- 页面空间有限，需要更紧凑的显示
- 只关注某个场景的执行结果

**适合展开场景的情况**：
- 需要查看或修改场景配置
- 需要检查场景参数设置
- 初次配置场景
- 调试场景配置问题
