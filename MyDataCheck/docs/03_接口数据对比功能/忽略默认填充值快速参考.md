# 接口数据对比 - 忽略默认填充值快速参考

## 一句话说明

勾选"忽略默认填充值"后，CSV文件中的 `-999` 和接口返回的 `null` 会被视为一致，不会被标记为差异。

## 快速使用

1. 进入"接口数据对比"页面
2. 在"全局默认配置"区域找到"忽略默认填充值"复选框
3. 勾选 → `-999` 和 `null` 视为一致
4. 不勾选（默认）→ `-999` 和 `null` 视为不同

## 对比规则

| CSV值 | API值 | 不勾选（默认） | 勾选后 |
|-------|-------|---------------|--------|
| -999 | null | ❌ 不一致 | ✅ 一致 |
| null | -999 | ❌ 不一致 | ✅ 一致 |
| -999 | -999 | ✅ 一致 | ✅ 一致 |
| null | null | ✅ 一致 | ✅ 一致 |
| -999 | 0 | ❌ 不一致 | ❌ 不一致 |
| null | 0 | ❌ 不一致 | ❌ 不一致 |

## 典型场景

### 场景1：数据缺失的不同表示方式

- CSV文件：使用 `-999` 表示数据缺失
- 接口返回：使用 `null` 表示数据缺失
- 解决方案：勾选"忽略默认填充值"

### 场景2：严格对比

- 需要严格区分 `-999` 和 `null`
- 解决方案：不勾选"忽略默认填充值"（默认）

## 配置位置

```
接口数据对比页面
  └─ 全局默认配置
      └─ 默认线程数
      └─ 默认超时时间(秒)
      └─ 流式对比批次大小
      └─ 默认转换特征值为数值 ✓
      └─ 忽略默认填充值 ☐  ← 这里
      └─ 输出文件控制
```

## 配置文件

在 `api_comparison/config.json` 中：

```json
{
  "global_config": {
    "default_ignore_default_fill": false
  }
}
```

## 注意事项

- ⚠️ 默认不勾选（保持原有严格对比行为）
- ⚠️ 只有 `-999` 和 `null` 会被视为默认填充值
- ⚠️ 其他值（如 `-998`、`0`、空字符串）不受影响
- ⚠️ 场景级别配置优先于全局配置
