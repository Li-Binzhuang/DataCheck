## 🚀 快速启动

### 方式一：使用启动脚本（推荐）

```bash
cd MyDataCheck
./start_web.sh
```

启动脚本会自动：
1. 激活虚拟环境
2. 检查依赖
3. 启动服务

### 方式二：手动启动

```bash
cd MyDataCheck

# 1. 激活虚拟环境
source .venv/bin/activate

# 2. 检查依赖（可选）
python check_startup.py

# 3. 启动服务
python web_app.py
```

## 📋 启动前检查

### 检查1: 虚拟环境

```bash
# 检查虚拟环境是否存在
ls -la MyDataCheck/.venv

# 如果不存在，创建虚拟环境
python3.12 -m venv MyDataCheck/.venv
```

### 检查2: 依赖安装

```bash
cd MyDataCheck
source .venv/bin/activate
pip install -r requirements.txt
```

### 检查3: 模块导入

```bash
# 运行检查脚本
python check_startup.py
```

预期输出：
```
✅ 所有检查通过！可以启动服务
```

## 🔧 常见问题

### Q1: 提示 "No module named 'flask'"

**原因**: 虚拟环境未激活或依赖未安装

**解决**:
```bash
cd MyDataCheck
source .venv/bin/activate
pip install -r requirements.txt
```

### Q2: 提示 "SyntaxError: invalid syntax"

**原因**: 之前的语法错误

**解决**: 已修复，重新拉取代码或查看 `语法错误修复完成.txt`

### Q3: 端口被占用

**解决**:
```bash
# 停止旧服务
./stop_web.sh

# 或手动杀进程
lsof -ti:5001 | xargs kill -9

# 重新启动
./start_web.sh
```

### Q4: 内存占用高

**解决**: 已优化，内存占用降低 80%

查看优化说明：
- `内存优化总结.md`
- `docs/09_问题修复记录/内存优化完成报告.md`

## ✅ 检查清单

启动前请确认：

- [ ] 虚拟环境已创建 (`.venv` 目录存在)
