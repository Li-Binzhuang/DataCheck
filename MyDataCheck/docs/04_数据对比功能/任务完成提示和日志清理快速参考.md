## 优化2：自动清理日志

### 清理策略

✅ **自动清理**（任务完成后）:
- 保留任务信息
- 保留最后10条日志
- 删除完整日志文件
- 节省99%空间

### 保留的摘要示例


## 手动清理

### 清理指定任务

```bash
# 保留摘要
curl -X POST http://localhost:5001/api/tasks/{task_id}/cleanup \
  -H "Content-Type: application/json" \
  -d '{"keep_summary": true}'

# 完全清理
curl -X POST http://localhost:5001/api/tasks/{task_id}/cleanup \
  -H "Content-Type: application/json" \
  -d '{"keep_summary": false}'
```

### 清理旧任务

```bash
# 清理7天前的任务
curl -X POST http://localhost:5001/api/tasks/cleanup-old \
  -H "Content-Type: application/json" \
  -d '{"days": 7}'

# 清理30天前的任务
curl -X POST http://localhost:5001/api/tasks/cleanup-old \
  -H "Content-Type: application/json" \
  -d '{"days": 30}'
```

## API接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/tasks/{task_id}/cleanup` | POST | 清理指定任务日志 |
| `/api/tasks/cleanup-old` | POST | 清理旧任务 |

## 注意事项

⚠️ **清理后无法恢复**
- 清理前确认不再需要完整日志
- 建议保留摘要
- 重要任务手动备份

✅ **建议清理时机**
- 任务完成后（自动）
- 每周清理一次旧任务
- 磁盘空间不足时
