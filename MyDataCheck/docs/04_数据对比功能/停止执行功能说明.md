# 停止执行功能说明

## 功能概述

在数据对比过程中，如果接口报错或其他原因需要停止执行，可以使用停止按钮中断当前任务。

## 功能特点

1. ✅ **独立模块设计** - 不影响现有代码逻辑
2. ✅ **线程安全** - 支持多任务并发控制
3. ✅ **优雅停止** - 在检查点停止，不会中断正在执行的操作
4. ✅ **前后端分离** - 前端按钮 + 后端API + 执行控制
5. ✅ **实时反馈** - 停止信号发送后立即提示

---

## 使用方法

### 1. 启动任务

点击"▶️ 执行对比"按钮启动任务

### 2. 停止任务

任务执行过程中，会出现"⏹️ 停止执行"按钮

点击该按钮即可发送停止信号

### 3. 等待停止

停止信号发送后，任务会在下一个检查点停止

- 场景切换时检查
- 批次处理时检查
- 文件写入时检查

## 技术实现

### 架构设计

```
前端 (HTML/JS)
    ↓
停止按钮 → 发送停止请求
    ↓
后端 API (/api/stop/task/<task_id>)
    ↓
停止控制器 (StopController)
    ↓
执行流程 (检查停止信号)
```

### 核心组件

#### 1. 停止控制器 (StopController)

**文件**: `MyDataCheck/common/stop_controller.py`

**功能**:
- 管理任务注册和注销
- 提供停止信号检查
- 线程安全的单例模式

**API**:
```python
# 注册任务
task_id = StopController.register_task("任务名称")

# 检查是否应该停止
if StopController.should_stop(task_id):
    print("任务被停止")
    break

# 停止任务
StopController.stop_task(task_id)

# 注销任务
StopController.unregister_task(task_id)
```

#### 2. 停止路由 (stop_routes)

**文件**: `MyDataCheck/web/routes/stop_routes.py`

**端点**:
- `POST /api/stop/task/<task_id>` - 停止指定任务
- `GET /api/stop/tasks` - 获取所有任务状态
- `POST /api/stop/clear` - 清理已完成的任务
- `GET /api/stop/task/<task_id>/status` - 获取任务状态

#### 3. 前端控制 (stop-control.js)

**文件**: `MyDataCheck/static/js/stop-control.js`

**功能**:
- 管理停止按钮显示/隐藏
- 发送停止请求
- 更新按钮状态

**API**:
```javascript
// 设置当前任务ID（显示停止按钮）
setCurrentTaskId(taskId);

// 清除任务ID（隐藏停止按钮）
clearCurrentTaskId();

// 停止当前任务
stopCurrentTask();
```

## 停止检查点

任务会在以下位置检查停止信号：

### 接口数据对比

```python
# 场景循环中
for i, scenario in enumerate(enabled_scenarios, 1):
    # 检查是否应该停止
    if task_id and StopController.should_stop(task_id):
        print(f"\n⚠️  任务被用户停止 (场景 {i}/{len(enabled_scenarios)})")
        break
    
    # 执行场景
    execute_single_scenario(...)
```

### 未来扩展点

可以在以下位置添加更多检查点：

1. **接口请求循环** - 每N个请求检查一次
2. **数据写入循环** - 每N行检查一次
3. **批次处理** - 每个批次检查一次

## 使用示例

### 场景1: 接口频繁报错

```
1. 用户点击"执行对比"
2. 任务开始执行，出现"停止执行"按钮
3. 接口频繁报错，用户点击"停止执行"
4. 系统提示"停止信号已发送"
5. 当前场景执行完后，任务停止
6. 输出: "⚠️ 任务被用户停止 (场景 2/5)"
```

### 场景2: 配置错误

```
1. 用户点击"执行对比"
2. 发现配置错误，立即点击"停止执行"
3. 任务在下一个检查点停止
4. 用户修改配置后重新执行
```

## 注意事项

### 1. 停止时机

- ⚠️ 停止不是立即生效的
- ✅ 会在下一个检查点停止
- ✅ 正在执行的操作会完成

### 2. 数据完整性

- ✅ 已完成的场景数据完整
- ✅ 已写入的文件完整
- ⚠️ 当前场景可能未完成

### 3. 资源清理

- ✅ 任务停止后自动清理
- ✅ 任务ID自动注销
- ✅ 停止按钮自动隐藏

## API文档

### POST /api/stop/task/<task_id>

停止指定任务

**请求**:
```
POST /api/stop/task/a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

**响应**:
```json
{
    "success": true,
    "message": "停止信号已发送，任务将在下一个检查点停止",
    "task_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

### GET /api/stop/tasks

获取所有任务状态

**响应**:
```json
{
    "success": true,
    "tasks": [
        {
            "task_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
            "status": "running",
            "short_id": "a1b2c3d4"
        }
    ],
    "count": 1
}
```

### POST /api/stop/clear

清理已完成的任务

**响应**:
```json
{
    "success": true,
    "message": "已清理2个已完成的任务",
    "cleared_count": 2
}
```

可以扩展为支持暂停和恢复：

```python

while StopController.is_paused(task_id):
    time.sleep(1)
```

停止时保存进度，下次可以继续：

```python

    "current_scenario": i,
    "completed_rows": processed_rows
}
save_progress(task_id, progress)

# 恢复进度
progress = load_progress(task_id)
start_from = progress["current_scenario"]
```
