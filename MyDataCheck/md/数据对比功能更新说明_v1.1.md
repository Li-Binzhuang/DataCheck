# 数据对比功能更新说明 v1.1

## 📋 更新概述

为数据对比功能添加了"转换特征值为数值"配置选项，允许用户选择是否将字符串类型的特征值转换为数值类型进行对比。

---

## ✨ 新增功能

### 1. 转换特征值为数值配置

**位置**：数据对比页面 → 输出文件前缀下方

**功能说明**：
- 勾选后，会尝试将字符串类型的特征值转换为数值类型进行对比
- 例如："8" 会被转换为数字 8，"8.0" 会被转换为数字 8.0
- 这样可以避免因为数据类型不同导致的误判

**默认值**：勾选（启用转换）

**使用场景**：
- ✅ 两个文件中的数据类型不一致（一个是字符串"8"，另一个是数字8）
- ✅ 想要按数值大小进行对比（而不是字符串对比）
- ✅ 数据中包含整数和浮点数的混合（如 8 和 8.0）

**不适用场景**：
- ❌ 需要严格按字符串进行对比
- ❌ 数据中包含非数值的字符串（如 "N/A"、"未知"等）

```python
def _convert_string_to_number(value: Any) -> Any:
    """
    尝试将字符串转换为数值类型
    
    Args:
        value: 输入值
    
    Returns:
        如果可以转换为数值，返回数值；否则返回原值
    """
    if value is None or value == "":
        return value
    
    # 如果已经是数值类型，直接返回
    if isinstance(value, (int, float)):
        return value
    
    # 转换为字符串并去除空格
    str_value = str(value).strip()
    
    # 处理空字符串
    if not str_value:
        return value
    
    # 尝试转换为整数
    try:
        # 先尝试转换为浮点数
        float_value = float(str_value)
        # 如果浮点数等于其整数形式，返回整数
        if float_value == int(float_value):
            return int(float_value)
        # 否则返回浮点数
        return float_value
    except (ValueError, TypeError):
        # 无法转换，返回原值
        return value
```

**应用转换**：
```python
# 如果启用了特征值转换，尝试将字符串转换为数值
if convert_feature_to_number:
    offline_value = _convert_string_to_number(offline_value)
    online_value = _convert_string_to_number(online_value)
```

## 📊 转换逻辑说明

### 转换规则

1. **空值处理**：
   - `None`、空字符串 → 保持不变

2. **已是数值**：
   - 整数、浮点数 → 保持不变

3. **字符串转换**：
   - `"8"` → `8`（整数）
   - `"8.0"` → `8`（整数，因为等于其整数形式）
   - `"8.5"` → `8.5`（浮点数）
   - `"abc"` → `"abc"`（无法转换，保持原值）

### 对比示例

#### 示例1：启用转换（默认）

| 文件1值 | 文件2值 | 转换后文件1 | 转换后文件2 | 对比结果 |
|---------|---------|-------------|-------------|----------|
| "8"     | 8       | 8           | 8           | ✅ 相同  |
| "8.0"   | 8       | 8           | 8           | ✅ 相同  |
| "8.5"   | 8.5     | 8.5         | 8.5         | ✅ 相同  |
| "abc"   | "abc"   | "abc"       | "abc"       | ✅ 相同  |
| "8"     | "9"     | 8           | 9           | ❌ 不同  |

#### 示例2：不启用转换

| 文件1值 | 文件2值 | 对比结果 |
|---------|---------|----------|
| "8"     | 8       | ❌ 不同  |
| "8.0"   | 8       | ❌ 不同  |
| "8.5"   | 8.5     | ❌ 不同  |
| "abc"   | "abc"   | ✅ 相同  |
| "8"     | "9"     | ❌ 不同  |

## 🎯 使用建议

### 建议启用转换的场景

1. **数据来源不同**：
   - 一个文件来自数据库（数值类型）
   - 另一个文件来自CSV导出（字符串类型）

2. **数据格式不统一**：
   - 同一个字段在不同文件中的类型不一致
   - 例如：订单金额在一个文件中是字符串"100.00"，在另一个文件中是数字100

3. **浮点数精度问题**：
   - 需要将 8.0 和 8 视为相同
   - 需要将 "8" 和 8 视为相同

### 建议不启用转换的场景

1. **严格字符串对比**：
   - 需要区分 "8" 和 8
   - 需要区分 "08" 和 "8"

2. **包含特殊字符串**：
   - 数据中包含 "N/A"、"未知"、"无" 等非数值字符串
   - 这些字符串不应该被转换

3. **ID字段对比**：
   - 对比的是ID字段（如订单号、用户ID）
   - 这些字段虽然是数字，但应该作为字符串对比
