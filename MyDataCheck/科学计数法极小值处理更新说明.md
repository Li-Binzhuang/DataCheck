# 科学计数法极小值处理功能 - 更新说明

## 🎯 更新内容

在接口数据对比模块中，新增了科学计数法极小值的自动处理功能。

## ✨ 核心功能

当 CSV 文件中的特征值是科学计数法表示的极小值（如 `0E-8`）时，系统会自动将其视为 `0` 与接口数据进行对比。

## 📋 处理规则

### 会被转换为 0 的情况

- `0E-8` → `0`
- `0.0E-10` → `0`
- `1E-20` → `0` (极小值，绝对值 < 1e-15)
- `0e-8` → `0` (小写也支持)

### 不会被转换的情况

- `0.00000001` → 保持原值 (小数格式，不是科学计数法)
- `1E-5` → 保持原值 (0.00001，不够小)
- `123.456` → 保持原值 (正常数值)

## 📊 对比示例

| CSV 值 | 接口值 | 结果 | 说明 |
|--------|--------|------|------|
| `0E-8` | `0` | ✅ 一致 | 科学计数法的 0 |
| `1E-20` | `0` | ✅ 一致 | 极小值视为 0 |
| `1E-5` | `0.00001` | ✅ 一致 | 正常科学计数法 |
| `1E-5` | `0` | ❌ 不一致 | 不够小，不视为 0 |

3. 转换阈值固定为 `1e-15`
