# 授信产品表分析报告

## 一、表概述

**表名**: `ods_mx_aprv_approve_credit_apply_product_df` (授信产品表)

**业务定位**: 在业务流程中，授信产品表位于**授信申请**之后，用于存储每个授信申请对应的产品配置方案。

**业务流程位置**:
```
授信申请 → 授信产品 → 用信申请 → 资产申请(借款申请) → 额度 → 借据(贷款)
```

## 四、业务流程关联分析

### 4.1 与授信申请的关系

- 一个授信申请可以对应**多个产品配置**
- 产品配置的`max_amount`总和通常等于或接近授信申请的`credit_limit_amount`（授信额度金额）
- 匹配率: 100%（在可比较的数据中）

### 4.2 与用信申请的关系

- **产品使用率**: 83.33%
- 用信申请通过`credit_apply_id`关联到授信申请，进而关联到产品配置
- 用信申请使用的产品配置:
  - 4期产品: 7次
  - 8期产品: 3次
  - 借款天数: 全部为15天

### 4.3 业务流程中的价值

授信产品表在业务流程中起到**产品配置中心**的作用：

1. **产品方案定义**: 在授信申请通过后，系统根据客户资质和授信额度，生成多个可选的产品方案
2. **灵活配置**: 通过不同的期数、利率、金额组合，为客户提供多样化的借款选择
3. **用信指导**: 客户在用信时，可以选择授信产品表中已配置的产品方案
4. **风险控制**: 通过`max_amount`限制每个产品的最大借款金额，控制单笔借款风险

## 七、关键发现与建议

### 7.1 关键发现

1. **产品配置标准化**: 所有产品的借款天数统一为15天，说明产品配置高度标准化
2. **多产品策略**: 每个授信申请通常配置2个产品（4期和8期），为客户提供选择
3. **利率差异化**: 8期产品利率更低，但4期产品使用率更高，说明客户更偏好短期产品
4. **高使用率**: 83.33%的产品被实际使用，说明产品配置策略有效

### 7.2 业务建议

1. **产品优化**: 考虑增加更多期数选择（如6期、12期），满足不同客户需求
2. **金额梯度**: 可以设置更细化的金额梯度，提高产品匹配度
3. **利率策略**: 分析4期产品使用率高的原因，优化8期产品的吸引力
4. **状态管理**: 明确状态300和400的含义，建立状态流转规则
