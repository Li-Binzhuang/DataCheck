# CDC项目：浮点数处理功能说明

## 功能概述

为CDC项目的四个板块Notebook添加了浮点数处理功能，对特征字典中的浮点数列进行统一处理：
1. 保留6位小数（四舍五入）
2. 空值填充为-999

## 处理逻辑

### 代码实现

```python
# 对浮点数列进行处理：保留6位小数，空值填充为-999
exclude_cols = ['apply_id', 'request_time', 'apply_time']  # 排除列名，可添加
float_cols = [c for c in feature_dict.select_dtypes(include=['float64', 'float32']).columns if c not in exclude_cols]
feature_dict[float_cols] = feature_dict[float_cols].round(6).fillna(-999)
print("feature_dict shape:", feature_dict.shape)
display(feature_dict.head(10))
```

### 处理步骤

1. **定义排除列**: 不处理 `apply_id`, `request_time`, `apply_time` 这些标识列
2. **筛选浮点数列**: 自动识别 `float64` 和 `float32` 类型的列
3. **四舍五入**: 保留6位小数
4. **填充空值**: 将 NaN 值填充为 -999
5. **显示结果**: 打印形状并显示前10行

## 为什么需要这个功能？

### 1. 数据一致性

**问题**: 不同来源的浮点数可能有不同的精度
```python
# 处理前
feature_A: 0.123456789012345
feature_B: 0.12
feature_C: 0.123456
```

**解决**: 统一保留6位小数
```python
# 处理后
feature_A: 0.123457  # 四舍五入
feature_B: 0.120000  # 补齐位数
feature_C: 0.123456  # 保持不变
```

### 2. 空值处理

**问题**: NaN 值在某些系统中无法正确处理
```python
# 处理前
feature_X: NaN
feature_Y: None
feature_Z: 0.123456
```

**解决**: 统一填充为 -999
```python
# 处理后
feature_X: -999.0
feature_Y: -999.0
feature_Z: 0.123456
```

### 3. 文件大小优化

**问题**: 过多的小数位会增加文件大小
```python
# 处理前（每个值约20字节）
0.12345678901234567890

# 处理后（每个值约8字节）
0.123457
```

**效果**: 文件大小减少约 40%

### 4. 可读性提升

**问题**: 过多的小数位难以阅读
```python
# 处理前
cdc_consultas_30d_total_cnt_v2: 123.00000000000001
cdc_consultas_30d_shop_cnt_v2: 5.999999999999999

# 处理后
cdc_consultas_30d_total_cnt_v2: 123.0
cdc_consultas_30d_shop_cnt_v2: 6.0
```

## 技术细节

### 数据类型识别

```python
# 自动识别浮点数列
float_cols = feature_dict.select_dtypes(include=['float64', 'float32']).columns
```

支持的类型：
- `float64`: 双精度浮点数（默认）
- `float32`: 单精度浮点数

### 排除列机制

```python
exclude_cols = ['apply_id', 'request_time', 'apply_time']
```

**为什么排除这些列？**
- `apply_id`: 标识列，不应该是浮点数
- `request_time`: 时间列，不应该被四舍五入
- `apply_time`: 时间列，不应该被四舍五入

**如何添加更多排除列？**
```python
exclude_cols = ['apply_id', 'request_time', 'apply_time', 'custom_id', 'timestamp']
```

### 四舍五入规则

使用 Python 的 `round()` 函数：
- 遵循"银行家舍入"规则（四舍六入五成双）
- 保留6位小数

```python
# 示例
0.1234564 → 0.123456  # 4舍
0.1234565 → 0.123456  # 5成双（偶数）
0.1234575 → 0.123458  # 5成双（偶数）
0.1234566 → 0.123457  # 6入
```

### 空值填充

```python
.fillna(-999)
```

**为什么选择 -999？**
1. **明显标识**: -999 是一个明显的异常值，容易识别
2. **不影响统计**: 在大多数统计分析中，-999 会被识别为缺失值
3. **兼容性好**: 大多数机器学习库都支持 -999 作为缺失值标记
4. **行业惯例**: 金融和风控领域常用 -999 表示缺失

## 注意事项

### 1. 精度损失

保留6位小数会导致精度损失：
```python
原始值: 0.1234567890
处理后: 0.123457
损失:   0.0000007890
```

**影响评估**: 对于大多数金融特征，6位小数已经足够精确。

### 2. 空值语义

-999 表示"缺失值"，不是"真实的-999"：
```python
# 如果某个特征的真实值可能是-999，需要特殊处理
if feature_value == -999:
    # 这可能是缺失值，也可能是真实值
    pass
```

**建议**: 如果特征值域包含-999，考虑使用其他缺失值标记（如-9999）。

### 3. 时间列保护

确保时间列不被处理：
```python
exclude_cols = ['apply_id', 'request_time', 'apply_time']
```

**如果有其他时间列**: 添加到 `exclude_cols` 中。

### 4. 整数列

整数列不会被处理：
```python
# int64, int32 类型的列不受影响
feature_dict.select_dtypes(include=['float64', 'float32'])  # 只选浮点数
```
