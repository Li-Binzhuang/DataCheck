# 配置字段名修复说明

## 问题描述

在实现内存对比模式后，发现 `config.json` 中的输出控制配置字段名与代码不一致：

**旧配置（错误）**:
```json
"output_config": {
  "output_merged_data": false,
  "output_api_data": false
}
```

**新配置（正确）**:
```json
"output_config": {
  "output_intermediate_files": false
}
```

## 问题原因

在 TASK 10 中实现内存对比模式时，将两个独立的输出控制开关合并为一个统一的开关 `output_intermediate_files`，但 `config.json` 文件中仍使用旧的字段名。

## 修复内容

### 修改文件
- `MyDataCheck/api_comparison/config.json`

### 修改前
```json
"output_config": {
  "output_merged_data": false,
  "output_api_data": false
}
```

### 修改后
```json
"output_config": {
  "output_intermediate_files": false
}
```

## 字段说明

### `output_intermediate_files` (boolean)
- **默认值**: `true`
- **作用**: 控制是否输出中间文件（接口数据和全量合并）
- **取值**:
  - `true`: 完整输出模式 - 输出所有中间文件（4个文件）
    - `_api_data.csv` - 接口返回的原始数据
    - `_全量数据合并.csv` - 原始数据与接口数据的全量合并
    - `_analysis_report.csv` - 对比分析报告
    - `_feature_stats.csv` - 特征统计报告
  - `false`: 仅报告模式 - 只输出对比报告（2个文件）
    - `_analysis_report.csv` - 对比分析报告
    - `_feature_stats.csv` - 特征统计报告

## 代码引用位置

### 1. 前端配置收集
**文件**: `MyDataCheck/static/js/api-compare.js`
```javascript
output_config: {
    output_intermediate_files: getGlobalChecked('output_intermediate_files', true)
}
```

### 2. 前端配置加载
**文件**: `MyDataCheck/static/js/api-compare.js`
```javascript
const outputIntermediateFilesElem = document.getElementById('output_intermediate_files');
if (outputIntermediateFilesElem) {
    outputIntermediateFilesElem.checked = config.output_config?.output_intermediate_files !== false;
}
```

### 3. 后端执行流程
**文件**: `MyDataCheck/api_comparison/job/process_executor.py`
```python
output_config = global_config.get('output_config', {})
output_intermediate_files = output_config.get('output_intermediate_files', True)

if output_intermediate_files:
    # 完整输出模式
    fetch_api_data_step(...)
    compare_data_step(...)
else:
    # 仅报告模式（内存对比）
    compare_data_in_memory_step(...)
```
