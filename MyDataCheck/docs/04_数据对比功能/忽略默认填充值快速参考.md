# 忽略默认填充值 - 快速参考

## 一句话说明

勾选"忽略默认填充值"后，`-999` 和 `null` 会被视为一致，不会被标记为差异。

## 快速使用

1. 进入"数据对比"页面
2. 找到"忽略默认填充值"复选框
3. 勾选 → `-999` 和 `null` 视为一致
4. 不勾选（默认）→ `-999` 和 `null` 视为不同

## 对比规则

| 值1 | 值2 | 不勾选（默认） | 勾选后 |
|-----|-----|---------------|--------|
| -999 | null | ❌ 不一致 | ✅ 一致 |
| null | -999 | ❌ 不一致 | ✅ 一致 |
| -999 | -999 | ✅ 一致 | ✅ 一致 |
| null | null | ✅ 一致 | ✅ 一致 |
| -999 | 0 | ❌ 不一致 | ❌ 不一致 |
| null | 0 | ❌ 不一致 | ❌ 不一致 |

## 典型场景

### 场景1：数据缺失的不同表示方式

- 模型特征表：使用 `-999` 表示数据缺失
- 接口特征表：使用 `null` 表示数据缺失
- 解决方案：勾选"忽略默认填充值"

### 场景2：严格对比

- 需要严格区分 `-999` 和 `null`
- 解决方案：不勾选"忽略默认填充值"（默认）

## 配置位置

```
数据对比页面
  └─ 对比配置
      └─ 转换特征值为数值 ✓
      └─ 忽略默认填充值 ☐  ← 这里
      └─ 输出全量数据合并文件 ☐
```

## 注意事项

- ⚠️ 默认不勾选（保持原有严格对比行为）
- ⚠️ 只有 `-999` 和 `null` 会被视为默认填充值
- ⚠️ 其他值（如 `-998`、`0`、空字符串）不受影响
