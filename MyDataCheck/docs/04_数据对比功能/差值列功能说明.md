# 差值列功能说明

## 📋 功能概述

在接口数据对比的 `_compare_analysis_report.csv` 输出文件中增加"差值"列。

## 🎯 修改内容

### 1. 新增"差值"列

在 analysis_report.csv 文件中增加一列"差值"：
- **计算公式**：差值 = CSV值 - API值
- **精度**：保留6位小数
- **数据类型**：数值型（无法转换为数字时为空字符串）

### 2. 流式写入

采用流式写入方式，边处理边写入：
- ✅ 不在内存中累积所有记录
- ✅ 内存占用极低
- ✅ 适合处理超大数据集

## 📊 输出格式

### 修改前
```csv
特征名,applyId,request_time,CSV值,API值
feature1,123456,2025-11-25 01:53:44.943,100.5,99.8
feature2,123456,2025-11-25 01:53:44.943,200,200.1
```

### 修改后
```csv
特征名,applyId,request_time,CSV值,API值,差值
feature1,123456,2025-11-25 01:53:44.943,100.5,99.8,0.7
feature2,123456,2025-11-25 01:53:44.943,200,200.1,-0.1
```

## 🔧 修改的文件

### 1. streaming_comparator.py
**文件路径**：`MyDataCheck/api_comparison/job/streaming_comparator.py`

**修改内容**：
- 在流式对比模式中增加差值计算
- 边处理边写入，不累积记录
- 保持低内存占用

**关键代码**：
```python
# 计算差值（CSV值 - API值），保留6位小数
try:
    csv_num = float(csv_value) if csv_value else 0.0
    api_num = float(api_value) if api_value else 0.0
    difference = round(csv_num - api_num, 6)
except (ValueError, TypeError):
    # 如果无法转换为数字，差值设为空字符串
    difference = ""

# 直接写入（流式处理）
writer.writerow([
    feature_name,
    apply_id,
    use_create_time,
    csv_value,
    api_value,
    difference  # 差值
])
```

## 💡 使用场景

### 1. 快速分析差异
通过差值列可以快速分析：
- 哪些特征有差异
- 差异是正向还是负向
- 差异的大小

### 2. 数据质量评估
- 差值接近0：数据一致性好
- 差值较大：需要重点关注
- 差值为空：非数值型特征，需要单独分析

### 3. Excel 中进一步分析
在 Excel 中打开文件后，可以：
- 按差值列排序（找出差异最大的记录）
- 按差值列筛选（只看差异大于某个阈值的记录）
- 使用条件格式高亮显示差异较大的行

## ⚠️ 注意事项

### 1. 内存占用
采用流式写入方式：
- ✅ 不在内存中累积记录
- ✅ 内存占用极低（与修改前相同）
- ✅ 适合处理百万级甚至更大的数据集

### 2. 非数值型特征
对于非数值型特征（如字符串、日期等）：
- 差值列显示为空字符串
- 仍然会显示在报告中，只是无法计算差值

### 3. 记录顺序
记录按处理顺序写入：
- 不进行排序（避免内存占用）
- 如需排序，可在 Excel 中手动排序
- 或使用 Excel 的筛选功能查看特定范围的差值

## 💡 Excel 中的使用技巧

### 1. 按差值排序
在 Excel 中打开文件后：
1. 选中"差值"列
2. 点击"数据" → "排序"
3. 选择"降序"或"升序"

### 2. 筛选大差异
1. 选中表头行
2. 点击"数据" → "筛选"
3. 点击"差值"列的筛选按钮
4. 设置数值筛选条件（如：大于 10 或小于 -10）

### 3. 条件格式高亮
1. 选中"差值"列
2. 点击"开始" → "条件格式"
3. 选择"色阶"或"数据条"
4. 差异大的记录会自动高亮显示
