# 接口请求失败记录输出功能说明

## 功能概述

在接口数据对比时，如果选中输出中间文件，系统会自动将接口请求失败的记录输出到单独的CSV文件中，方便排查问题。

## 输出文件格式

### 文件命名规则
```
{输出前缀}_{时间戳}_api_data_error_records.csv
```

例如：
```
bz_200_boss_20260128_143025_api_data_error_records.csv
```

### 文件字段

| 字段名 | 说明 | 示例 |
|--------|------|------|
| applyid | 申请ID（从输入CSV中提取） | 1234567890 |
| baseTime | 基准时间（从输入CSV中提取） | 2024-01-15 10:30:00 |
| 错误code | 错误信息 | 接口请求失败: timeout |

## 配置说明

### 1. 全局配置

在 `config.json` 的 `output_config` 中配置：

```json
{
  "output_config": {
    "output_intermediate_files": true,
    "output_error_records": true
  }
}
```

**配置项说明**：
- `output_intermediate_files`: 是否输出中间文件（必须为true才会输出错误记录）
- `output_error_records`: 是否输出错误记录文件（默认true）

### 2. 场景配置示例

```json
{
  "scenarios": [
    {
      "name": "测试场景",
      "enabled": true,
      "input_csv_file": "test_data.csv",
      "output_file_prefix": "test",
      "api_url": "http://example.com/api",
      "api_params": [
        {
          "param_name": "applyId",
          "column_index": 0,
          "is_time_field": false
        },
        {
          "param_name": "baseTime",
          "column_index": 1,
          "is_time_field": true
        }
      ]
    }
  ],
  "output_config": {
    "output_intermediate_files": true,
    "output_error_records": true
  }
}
```

## 错误类型

系统会记录以下类型的错误：

### 1. 参数错误
- **列索引无效**: 参数配置的列索引超出CSV文件范围
- **参数为空**: 必填参数在CSV中为空值

示例：
```
参数 'applyId' 的列索引无效: column_index=5, 行长度=3
参数 'baseTime' 为空: 行 10, 列 1
```

### 2. 请求错误
- **接口请求失败**: 网络超时、连接失败等
- **JSON解析失败**: 接口返回的数据不是有效的JSON格式

示例：
```
接口请求失败: params={'applyId': '123', 'baseTime': '2024-01-15T10:30:00.000'}
JSON解析失败: params={'applyId': '456', 'baseTime': '2024-01-15T11:00:00.000'}
```

### 3. 处理异常
- **处理异常**: 数据处理过程中发生的其他异常

示例：
```
处理异常: division by zero
```

## 使用场景

### 1. 排查接口问题
当接口请求失败率较高时，可以通过错误记录文件快速定位问题：
- 查看哪些applyId请求失败
- 分析失败的时间分布
- 根据错误信息判断失败原因

### 2. 数据质量检查
通过错误记录可以发现输入数据的问题：
- 参数为空的记录
- 时间格式不正确的记录
- 列索引配置错误

### 3. 重试机制
可以使用错误记录文件中的applyId和baseTime，重新构造输入文件进行重试。

## 输出示例

### 输入CSV（test_data.csv）
```csv
applyId,baseTime,feature1,feature2
123,2024-01-15 10:30:00,100,200
456,2024-01-15 11:00:00,150,250
789,,200,300
```

### 错误记录输出（test_20260128_143025_api_data_error_records.csv）
```csv
applyid,baseTime,错误code
789,,参数 'baseTime' 为空: 行 3, 列 1
```

## 注意事项

1. **必须启用中间文件输出**
   - 只有当 `output_intermediate_files=true` 时，才会输出错误记录
   - 如果使用"仅报告模式"，不会生成错误记录文件

2. **字段提取逻辑**
   - applyId字段：优先从api_params配置中查找custNo/applyId相关字段
   - baseTime字段：从api_params配置中查找第一个时间字段
   - 如果配置中找不到，会尝试从CSV表头中匹配

3. **文件编码**
   - 输出文件使用UTF-8编码
   - 可以用Excel、记事本等工具直接打开

4. **性能影响**
   - 输出错误记录对性能影响很小
   - 只在有错误时才写入文件
   - 不影响正常请求的处理速度

## 常见问题

### Q1: 为什么没有生成错误记录文件？
A: 可能的原因：
1. 所有请求都成功了，没有错误记录
2. `output_intermediate_files` 配置为 false
3. `output_error_records` 配置为 false

### Q2: 错误记录文件中的applyId为空怎么办？
A: 检查以下配置：
1. api_params中是否正确配置了applyId字段
2. column_index是否指向正确的列
3. 输入CSV中该列是否有值

### Q3: 如何关闭错误记录输出？
A: 在config.json中设置：
```json
{
  "output_config": {
    "output_error_records": false
  }
}
```
