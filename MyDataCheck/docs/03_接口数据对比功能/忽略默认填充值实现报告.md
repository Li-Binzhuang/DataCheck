## 功能特性

### 默认行为
- 默认不勾选（`ignore_default_fill=False`）
- 保持原有的严格对比逻辑
- `-999` 和 `null` 被视为不同的值

### 启用后行为
- 勾选复选框（`ignore_default_fill=True`）
- `-999` 和 `null` 被视为一致
- 不会被标记为差异

### 对比规则

| CSV值 | API值 | 不勾选（默认） | 勾选后 |
|-------|-------|---------------|--------|
| -999 | null | ❌ 不一致 | ✅ 一致 |
| null | -999 | ❌ 不一致 | ✅ 一致 |
| -999 | -999 | ✅ 一致 | ✅ 一致 |
| null | null | ✅ 一致 | ✅ 一致 |
| -999 | 0 | ❌ 不一致 | ❌ 不一致 |
| null | 0 | ❌ 不一致 | ❌ 不一致 |

## 技术实现细节

### 1. 配置传递链路

```
前端复选框 (global_ignore_default_fill)
  ↓
JavaScript收集配置 (collectConfig)
  ↓
POST请求发送到后端 (global_config.default_ignore_default_fill)
  ↓
Flask路由接收配置 (execute_comparison_flow)
  ↓
场景执行器获取配置 (execute_single_scenario)
  ↓
传递给流式对比步骤 (streaming_compare_step)
  ↓
传递给StreamingComparator
  ↓
传递给compare_values()
  ↓
执行对比逻辑
```

### 2. 配置优先级

1. 场景级别配置：`scenarios[].ignore_default_fill`
2. 全局默认配置：`global_config.default_ignore_default_fill`
3. 代码默认值：`False`

## 使用建议

### 适用场景
1. CSV文件和接口返回使用不同方式表示缺失值（-999 vs null）
2. 需要将默认填充值视为等价的业务场景
3. 减少因默认填充值表示方式不同导致的误报差异

### 不适用场景
1. 需要严格区分 -999 和 null 的场景
2. -999 有实际业务含义的场景（不是默认填充值）

## 与数据对比模块的区别

| 特性 | 数据对比模块 | 接口数据对比模块 |
|------|-------------|-----------------|
| 配置位置 | 数据对比页面 | 接口数据对比页面 |
| 配置ID | `compare-ignore-default-fill` | `global_ignore_default_fill` |
| 配置级别 | 单次对比 | 全局默认 + 场景级别 |
| 对比对象 | CSV vs CSV | CSV vs 接口返回 |
| 核心函数 | `data_comparator.py` | `streaming_comparator.py` |
