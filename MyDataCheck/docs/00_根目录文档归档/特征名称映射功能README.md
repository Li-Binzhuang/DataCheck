# 特征名称映射功能 - README

## 📌 功能简介

在MyDataCheck项目的CSV数据对比模块中，新增"特征名称映射"功能，允许在对比前为指定文件的特征列名添加前缀和/或后缀。

## 🎯 核心特性

- ✅ 支持为指定文件（file1或file2）的特征列添加前缀和/或后缀
- ✅ 可以只添加前缀、只添加后缀，或同时添加
- ✅ 默认不启用，保持现有逻辑不变
- ✅ 支持配置保存和加载
- ✅ 映射只影响特征列，不影响主键列
- ✅ 映射在内存中进行，不修改原始文件

## 🚀 快速开始

### 1. 启用功能
在Web界面的"数据对比"页面，勾选"启用特征名称映射"

### 2. 配置参数
- 选择映射文件（file1或file2）
- 输入前缀（可选）
- 输入后缀（可选）

### 3. 执行对比
点击"执行对比"按钮，系统会自动应用映射并执行对比

## 💡 使用示例

### 示例：为file1添加前缀
```
原始列名：age, income, score
配置：
  - 映射文件：file1
  - 前缀：model_
  - 后缀：（留空）
映射后：model_age, model_income, model_score
```

## 📊 配置格式

配置保存在 `data_comparison/config.json`：
```json
{
  "scenarios": [{
    "enable_column_mapping": true,
    "mapping_file": "file1",
    "mapping_prefix": "model_",
    "mapping_suffix": "_v1"
  }]
}
```

## ⚠️ 注意事项

1. 映射只影响特征列（从`feature_start`开始的列）
2. 主键列不受映射影响
3. 映射不修改原始文件
4. 默认不启用，需要手动勾选

## 📞 常见问题

**Q: 可以为两个文件都添加映射吗？**
A: 目前只支持为一个文件添加映射。

**Q: 映射会影响主键列吗？**
A: 不会，映射只影响特征列。

**Q: 如何取消映射？**
A: 取消勾选"启用特征名称映射"即可。
