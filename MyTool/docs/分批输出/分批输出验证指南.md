## ✅ 验证项目

详细验证脚本会检查以下内容：

### 1. 文件存在性
- ✅ 是否生成了批次文件
- ✅ 文件数量是否合理

### 2. 文件命名
- ✅ 命名格式: `{前缀}_batch{批次号:03d}_{起始行}-{结束行}.csv`
- ✅ 批次号连续性
- ✅ 行范围正确性

### 3. 文件内容
- ✅ 每个文件的行数（除最后一个外应为500行）
- ✅ 最后一个文件行数 ≤ 500
- ✅ CSV格式正确性

### 4. 数据完整性
- ✅ 总行数统计
- ✅ apply_id 唯一性检查
- ✅ 无重复数据

## 🔍 常见问题排查

### 问题1: 未找到批次文件

**现象**:
```
⏳ 未找到批次文件
```

**原因**:
- 尚未运行板块衍生脚本
- 脚本运行出错

**解决**:
1. 在 Jupyter 中运行对应的板块脚本
2. 检查是否有报错信息
3. 确认 `outputs/` 目录存在

---

### 问题2: 文件行数不正确

**现象**:
```
⚠️ 批次 001: cdc1_features_batch001_1-500.csv
   行数: 450 (预期: 500)
```

**原因**:
- 数据量不足500行
- 这是正常的（如果是最后一个批次）

**解决**:
- 如果是最后一个批次，这是正常的
- 如果不是最后一个批次，检查脚本逻辑

---

### 问题3: 存在重复的 apply_id

**现象**:
```
⚠️ 警告: 存在重复的 apply_id！
   重复数量: 5
```

**原因**:
- 分批逻辑有问题
- 数据源本身有重复

**解决**:
1. 检查分批输出代码
2. 检查原始数据是否有重复
3. 重新运行脚本

---

### 问题4: 文件命名不符合预期

**现象**:
```
⚠️ 文件命名不符合预期: cdc1_features_batch1_1-500.csv
```

**原因**:
- 批次号格式不正确（应为3位数字）

**解决**:
- 检查代码中的文件命名格式
- 应使用 `{batch_idx + 1:03d}` 格式化批次号

## 📊 手动验证方法

如果验证脚本不可用，可以手动验证：

### 1. 查看文件列表
```bash
cd CDC
ls -lh outputs/*_batch*.csv
```

### 2. 统计文件数量
```bash
# 第一板块
ls outputs/cdc1_features_batch*.csv | wc -l

# 第二板块
ls outputs/cdc2_features_batch*.csv | wc -l

# 第三板块
ls outputs/cdc3_features_batch*.csv | wc -l

# BOSS板块
ls outputs/cdcboss_features_batch*.csv | wc -l
```

### 3. 检查文件行数
```bash
# 检查所有批次文件的行数（包含表头）
wc -l outputs/cdc1_features_batch*.csv
```

### 4. 使用 Python 验证
```python
import pandas as pd
from pathlib import Path

# 读取所有批次
batch_files = sorted(Path("outputs").glob("cdc1_features_batch*.csv"))
dfs = [pd.read_csv(f) for f in batch_files]

# 统计信息
print(f"批次数: {len(batch_files)}")
print(f"总行数: {sum(len(df) for df in dfs)}")

# 检查 apply_id 唯一性
all_apply_ids = []
for df in dfs:
    all_apply_ids.extend(df['apply_id'].tolist())

print(f"唯一 apply_id: {len(set(all_apply_ids))}")
print(f"总 apply_id: {len(all_apply_ids)}")
```
