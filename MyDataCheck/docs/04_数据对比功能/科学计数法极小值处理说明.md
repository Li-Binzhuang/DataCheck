# 科学计数法极小值处理说明

## 功能概述

在接口数据对比过程中，自动将 CSV 文件中科学计数法表示的极小值（如 `0E-8`）视为 `0` 进行比较。

## 背景

CSV 文件中经常会出现科学计数法表示的数值，特别是极小值，如：
- `0E-8`
- `0.0E-10`
- `1E-20`

这些值在数学上等于或非常接近 0，但在字符串比较时会被认为不相等。为了正确对比这些值，系统会自动将它们标准化为 0。

## 处理规则

### 会被转换为 0 的情况

1. **科学计数法格式的 0**
   - `0E-8` → `0`
   - `0.0E-10` → `0`
   - `0e-8` → `0` (小写也支持)
   - `0.0e-10` → `0`

2. **科学计数法表示的极小值**
   - `1E-20` → `0` (绝对值 < 1e-15)
   - `5E-16` → `0`

### 不会被转换的情况

1. **小数格式的小数值**
   - `0.00000001` → 保持原值 (1E-8)
   - `0.0000000001` → 保持原值 (1E-10)
   - 这些是小数格式，不是科学计数法格式

2. **科学计数法表示的正常小数值**
   - `1E-5` → 保持原值 (0.00001)
   - `5E-6` → 保持原值 (0.000005)
   - 这些值不够小，不会被视为 0

3. **正常数值**
   - `123.456` → 保持原值
   - `0.5` → 保持原值

## 技术实现

### 核心函数

```python
def _normalize_scientific_notation(value: Any) -> Any:
    """
    标准化科学计数法表示的极小值
    将类似 0E-8, 0.0E-10 等形式的 0 统一转换为 0
    """
```

### 判断逻辑

1. 检查值是否包含 `E` 或 `e`（科学计数法标记）
2. 如果是科学计数法格式，转换为浮点数
3. 如果绝对值 < 1e-15，则视为 0
4. 否则保持原值

### 阈值说明

- **转换阈值**: `1e-15`
- 小于此阈值的科学计数法数值会被转换为 0
- 这个阈值足够小，不会影响正常的小数值

## 注意事项

1. **只处理科学计数法格式**
   - 只有包含 `E` 或 `e` 的科学计数法格式才会被处理
   - 小数格式（如 `0.00000001`）不会被转换

2. **阈值固定**
   - 转换阈值固定为 `1e-15`
   - 如需调整，需修改 `_normalize_scientific_notation` 函数

3. **向后兼容**
   - 此功能不影响现有的正常数值比较
   - 只是增强了对科学计数法极小值的处理
