# 合并分析报告说明

## 📋 处理内容

将 Mytest 项目下的 8 个 `*_analysis_report.csv` 文件合并，提取异常特征。

## 🎯 处理策略

1. **读取所有文件**: 读取 8 个分析报告文件
2. **统计异常数量**: 统计每个客户+时间组合的异常特征数量
3. **选择最优客户**: 选择异常特征最多的客户+时间组合
4. **按特征去重**: 提取该客户的所有异常特征，按特征名去重
5. **生成结果文件**: 保存到 `merged_analysis_report_top_customer.csv`

## 🛠️ 使用方法

### 运行脚本

```bash
python Mytest/merge_analysis_reports.py
```

### 脚本功能

1. 自动查找所有 `*_analysis_report.csv` 文件
2. 读取并合并所有数据
3. 统计每个客户+时间组合的异常数量
4. 选择异常最多的客户
5. 提取该客户的所有异常特征（按特征名去重）
6. 保存到 `merged_analysis_report_top_customer.csv`

### 输出信息

脚本会显示：
- 找到的文件列表
- 每个文件的行数
- 合并后的总行数
- 客户异常统计（前10名）
- 最终选中的客户信息
- 去重后的异常特征数量

## ✨ 优势

1. **客户量最少**: 只选择1个客户（异常最多的）
2. **特征完整**: 包含该客户的所有51个异常特征
3. **数据去重**: 按特征名去重，每个特征只保留一条记录
4. **易于分析**: 所有异常特征集中在一个客户，便于分析和调试

## 📝 注意事项

1. 脚本会自动选择异常特征最多的客户
2. 如果需要选择其他客户，可以修改脚本中的选择逻辑
3. 输出文件会覆盖已存在的同名文件
4. 所有数据都来自同一个客户+时间组合，保证了数据的一致性
