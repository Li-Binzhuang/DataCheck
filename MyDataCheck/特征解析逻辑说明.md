# 特征解析逻辑说明

## 功能概述

跑数模块现在支持智能解析API响应中的特征数据，能够自动识别并处理两种不同的数据结构：

1. **data.features 结构**（优先）：特征字段在 `data.features` 中
2. **data 直接结构**：特征字段直接在 `data` 中

## 解析规则

### 规则说明

当API返回的 `data` 层级下：
- **如果存在 `features` 字段**：优先解析 `features` 内的特征
- **如果不存在 `features` 字段**：直接解析 `data` 中的特征

### 支持的数据结构示例

#### 结构1：data.features（新结构，优先）

```json
{
  "retCode": "0000",
  "retMsg": "成功",
  "data": {
    "features": {
      "age": "30",
      "income": "50000",
      "credit_score": "750"
    },
    "apply_id": "12345",
    "timestamp": "2026-03-06"
  }
}
```

解析结果：提取 `features` 中的 `age`, `income`, `credit_score`

#### 结构2：data直接包含特征（旧结构，兼容）

```json
{
  "retCode": "0000",
  "retMsg": "成功",
  "data": {
    "age": "30",
    "income": "50000",
    "credit_score": "750",
    "apply_id": "12345",
    "timestamp": "2026-03-06"
  }
}
```

解析结果：直接提取 `data` 中的所有字段

```python
def extract_features_from_data(d: Dict[str, Any]) -> Dict[str, Any]:
    """
    从data字典中提取特征数据
    
    如果data中包含features字段，则解析features；否则直接返回data
    """
    if d is None or not isinstance(d, dict):
        return {}
    
    # 检查是否存在features字段
    if 'features' in d:
        features = d['features']
        if isinstance(features, dict):
            return features
        # ... 其他处理逻辑
    
    # 没有features字段，直接返回data
    return d
```

### 3. 批量跑数模块（batch_run）

**文件**：`MyDataCheck/batch_run/job/batch_runner.py`

**核心函数**：`extract_features()`

```python
def extract_features(self, api_response: Dict[str, Any]) -> Dict[str, Any]:
    """
    从接口响应中提取特征值
    
    支持两种结构：
    1. data.features 包含特征字段（优先）
    2. data 直接包含特征字段
    """
    if not api_response or "_error" in api_response:
        return {}
    
    data = api_response.get("data", {})
    if not isinstance(data, dict):
        return {}
    
    # 优先检查 data.features 结构
    if "features" in data and isinstance(data["features"], dict):
        return data["features"]
    
    # 如果没有 features 字段，直接返回 data
    return data
```

## 使用场景

### 场景1：新接口使用 data.features 结构

适用于新开发的接口，将特征数据和元数据分离：

```json
{
  "data": {
    "features": {
      "feature1": "value1",
      "feature2": "value2"
    },
    "metadata": {
      "version": "1.0",
      "timestamp": "2026-03-06"
    }
  }
}
```

### 场景2：旧接口使用 data 直接结构

兼容现有接口，特征数据直接在 data 中：

```json
{
  "data": {
    "feature1": "value1",
    "feature2": "value2",
    "version": "1.0",
    "timestamp": "2026-03-06"
  }
}
```

### 场景3：混合结构

同时支持 features 和其他字段：

```json
{
  "data": {
    "features": {
      "feature1": "value1",
      "feature2": "value2"
    },
    "apply_id": "12345",
    "request_time": "2026-03-06"
  }
}
```

解析结果：提取 `features` 中的特征，同时保留 `apply_id` 和 `request_time`

## 优势

1. **向后兼容**：自动兼容旧的数据结构，无需修改现有配置
2. **灵活性**：支持新的 features 结构，便于数据组织
3. **智能识别**：自动判断数据结构，无需手动配置
4. **统一处理**：所有跑数模块使用相同的解析逻辑

## 注意事项

1. 如果 `data.features` 存在但为空，会尝试解析 `data` 中的其他字段
2. 系统字段（如 `retCode`, `retMsg`, `success`, `timestamp`）会被自动过滤
3. 字符串类型的数值会自动转换为数字类型（如果启用了转换功能）
4. 支持嵌套结构的递归解析
