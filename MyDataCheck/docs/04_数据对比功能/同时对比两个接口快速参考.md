# 同时对比两个接口 - 快速参考

## ✅ 一句话总结

通过在config.json中配置多个场景，可以使用同一份输入数据依次对比接口A和接口B。

---

## 🔧 快速配置

### 1. 编辑配置文件

编辑 `api_comparison/config.json`：

```json
{
  "scenarios": [
    {
      "name": "接口A",
      "enabled": true,
      "input_csv_file": "test_data.csv",
      "output_file_prefix": "api_a",
      "api_url": "http://server-a:8081/api/features",
      ...
    },
    {
      "name": "接口B",
      "enabled": true,
      "input_csv_file": "test_data.csv",
      "output_file_prefix": "api_b",
      "api_url": "http://server-b:8081/api/features",
      ...
    }
  ]
}
```

### 2. 执行对比

```bash
cd MyDataCheck/api_comparison
python execute_comparison_flow.py
```

---

## 📊 关键配置

| 配置项 | 接口A | 接口B | 说明 |
|--------|-------|-------|------|
| input_csv_file | 相同 | 相同 | 使用同一份输入 |
| output_file_prefix | api_a | api_b | 不同的输出前缀 |
| api_url | URL-A | URL-B | 不同的接口地址 |

---

## 📁 输出文件

```
outputdata/api_comparison/
├── api_a_01301530_analysis_report.csv  # 接口A分析报告
├── api_a_01301530_feature_stats.csv    # 接口A特征统计
├── api_b_01301530_analysis_report.csv  # 接口B分析报告
└── api_b_01301530_feature_stats.csv    # 接口B特征统计
```

---

## ⏱️ 执行时间

- 单个接口: T
- 两个接口: 约 2T（顺序执行）

---

## 💡 常用操作

### 临时禁用某个场景

```json
{
  "enabled": false  // 设置为false即可禁用
}
```

### 只输出对比报告

```json
{
  "output_config": {
    "output_intermediate_files": false
  }
}
```

### 调整线程数

```json
{
  "thread_count": 100  // 降低线程数减少服务器压力
}
```
