# 特征名称映射功能实现总结

## 📋 需求概述

在MyDataCheck项目的CSV数据对比模块中，新增"特征名称映射"功能：
- 对比前可为指定文件的特征列名添加前缀和/或后缀
- 默认不启用，保持现有逻辑
- 可以只添加前缀、只添加后缀，或同时添加
- 支持配置保存和加载

## ✅ 实现内容

### 1. 后端实现

#### 1.1 数据对比核心模块 (`data_comparison/job/data_comparator.py`)

**修改的函数：** `compare_two_files()`

**新增参数：**
```python
def compare_two_files(
    sql_file_path: str,
    api_file_path: str,
    sql_key_column,
    api_key_column,
    sql_feature_start: int = 1,
    api_feature_start: int = 1,

    enable_column_mapping: bool = False,      # 新增：是否启用映射
    mapping_file: str = None,                 # 新增：映射文件
    mapping_prefix: str = "",                 # 新增：前缀
    mapping_suffix: str = ""                  # 新增：后缀

```

**核心逻辑：**
```python
# 应用特征名称映射（如果启用）
if enable_column_mapping and mapping_file:
    print(f"\n[特征名称映射] 正在应用映射...")
    if mapping_file == 'file1':
        # 为file1（Sql文件）的特征列添加前缀/后缀
        original_feature_cols_sql = feature_cols_sql.copy()
        feature_cols_sql = [f"{mapping_prefix}{col}{mapping_suffix}" for col in feature_cols_sql]
        print(f"  已为file1的 {len(feature_cols_sql)} 个特征列添加映射")
        print(f"  示例: '{original_feature_cols_sql[0]}' -> '{feature_cols_sql[0]}'")
    elif mapping_file == 'file2':
        # 为file2（接口文件）的特征列添加前缀/后缀
        original_feature_cols_api = feature_cols_api.copy()
        feature_cols_api = [f"{mapping_prefix}{col}{mapping_suffix}" for col in feature_cols_api]
        print(f"  已为file2的 {len(feature_cols_api)} 个特征列添加映射")
        print(f"  示例: '{original_feature_cols_api[0]}' -> '{feature_cols_api[0]}'")
```

## 🎯 功能特点

1. **灵活配置**
   - 可以只添加前缀
   - 可以只添加后缀
   - 可以同时添加前缀和后缀
   - 可以选择为哪个文件添加映射

2. **默认行为**
   - 默认不启用映射功能（保持现有逻辑）
   - 只有勾选"启用特征名称映射"后才会应用映射

3. **映射逻辑**
   - 映射只影响特征列（从`feature_start`开始的列）
   - 主键列不受影响
   - 映射在对比前执行，不会修改原始文件

4. **配置持久化**
   - 支持配置保存到JSON文件
   - 支持从JSON文件加载配置
   - 配置包含映射参数

## 🔄 使用流程

```
1. 上传两个CSV文件
   ↓
2. 配置基本参数（主键列、特征起始列等）
   ↓
3. 勾选"启用特征名称映射"（可选）
   ↓
4. 选择映射文件（file1或file2）
   ↓
5. 输入前缀和/或后缀
   ↓
6. 点击"执行对比"
   ↓
7. 系统应用映射并执行对比
   ↓
8. 生成对比报告
```
