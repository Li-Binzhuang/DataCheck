# 数据对比功能说明

## 📋 功能概述

数据对比功能允许用户上传两个文件（CSV或XLSX格式），对比两个表格的数据差异，并生成详细的对比报告。

---

## ✨ 功能特点

1. **支持多种格式**：支持 CSV 和 XLSX 文件
2. **灵活配置**：可自定义主键列和特征起始列
3. **详细报告**：生成多个对比报告文件
4. **实时输出**：执行过程实时显示在界面上
5. **易于使用**：简洁的Web界面，无需命令行操作

---

## 🚀 使用步骤

### 步骤1：访问数据对比页面

1. 启动 Web 服务：`./start_web.sh`
2. 在浏览器中打开：`http://localhost:5001`
3. 点击顶部的 **"📊 数据对比"** Tab

### 步骤2：上传文件

1. **上传文件1**：
   - 点击"文件1 (CSV/XLSX)"的文件选择按钮
   - 选择第一个要对比的文件
   - 等待上传完成（显示绿色✓）

2. **上传文件2**：
   - 点击"文件2 (CSV/XLSX)"的文件选择按钮
   - 选择第二个要对比的文件
   - 等待上传完成（显示绿色✓）

### 步骤3：配置对比参数

1. **文件1主键列索引**：
   - 设置文件1中用作主键的列索引（A列=0，B列=1，以此类推）
   - 默认值：0（第一列）

2. **文件2主键列索引**：
   - 设置文件2中用作主键的列索引
   - 默认值：0（第一列）

3. **文件1特征起始列索引**：
   - 设置文件1中特征数据的起始列索引
   - 默认值：1（第二列）

4. **文件2特征起始列索引**：
   - 设置文件2中特征数据的起始列索引
   - 默认值：1（第二列）

5. **输出文件前缀**：
   - 设置生成报告文件的前缀
   - 默认值：compare

6. **转换特征值为数值**：
   - 勾选后，会尝试将字符串类型的特征值转换为数值类型进行对比
   - 例如："8" 会被转换为数字 8，"8.0" 会被转换为数字 8.0
   - 这样可以避免因为数据类型不同导致的误判（如 "8" 和 8 被认为不同）
   - 默认值：勾选（启用转换）

### 步骤4：执行对比

1. 点击 **"▶️ 执行对比"** 按钮
2. 等待执行完成（右侧输出面板会实时显示进度）
3. 执行完成后，查看生成的报告文件

## 📊 输出文件说明

执行完成后，会在 `MyDataCheck/outputdata/compare/` 目录下生成以下文件：

| 文件名 | 说明 | 用途 |
|--------|------|------|
| `{前缀}_{时间戳}_对比结果_差异特征汇总.csv` | 差异特征汇总表 | 查看哪些特征有差异，差异数量 |
| `{前缀}_{时间戳}_对比结果_差异数据明细.csv` | 差异数据明细表 | 查看具体哪些记录的哪些字段有差异 |
| `{前缀}_{时间戳}_对比结果_特征统计.csv` | 特征统计表 | 查看每个特征的匹配率、差异率等统计信息 |
| `{前缀}_{时间戳}_对比结果_全量数据合并.csv` | 全量数据合并表 | 查看所有数据的对比结果（包含两个文件的数据） |
| `{前缀}_{时间戳}_对比结果_仅在离线表中的数据.csv` | 仅在文件2中的数据 | 查看只在文件2中存在的记录 |
| `{前缀}_{时间戳}_对比结果_仅在线上文件中的数据.csv` | 仅在文件1中的数据 | 查看只在文件1中存在的记录 |

**时间戳格式**：`月日时分`，例如 `01261530` 表示 1月26日15点30分

## 🔍 常见问题

### Q1：支持哪些文件格式？

**A**：目前支持 CSV 和 XLSX 两种格式。

### Q2：文件大小有限制吗？

**A**：单个文件最大支持 1GB。如果文件过大，建议先进行数据筛选或分批处理。

### Q3：主键列索引是什么意思？

**A**：主键列索引是指用于匹配两个文件中记录的列。例如，如果两个文件都有 `apply_id` 列作为唯一标识，且该列在第一列（A列），则主键列索引为 0。

**列索引对应关系**：
- A列 = 0
- B列 = 1
- C列 = 2
- D列 = 3
- ...以此类推

### Q4：特征起始列索引是什么意思？

**A**：特征起始列索引是指从哪一列开始进行数据对比。通常前几列是ID、时间等标识列，不需要对比，从某一列开始才是需要对比的特征数据。

**示例**：
```
| apply_id | apply_time | feature1 | feature2 | feature3 |
|----------|------------|----------|----------|----------|
| 主键列   | 时间列     | 特征列   | 特征列   | 特征列   |
| 索引=0   | 索引=1     | 索引=2   | 索引=3   | 索引=4   |
```

如果要从 `feature1` 开始对比，则特征起始列索引为 2。

### Q5：如何查看对比结果？

**A**：
1. 执行完成后，在输出面板中会显示生成的文件路径
2. 打开 `MyDataCheck/outputdata/compare/` 目录
3. 找到对应时间戳的文件
4. 使用 Excel 或其他工具打开查看

### Q6：两个文件的列数不一样可以对比吗？

**A**：可以。系统会自动处理列数不同的情况，只对比两个文件中都存在的列。

### Q7：如果两个文件的主键列不在同一位置怎么办？

**A**：可以分别设置"文件1主键列索引"和"文件2主键列索引"，系统会根据设置的索引来匹配记录。

### Q8：执行失败怎么办？

**A**：
1. 检查文件格式是否正确（CSV或XLSX）
2. 检查文件是否损坏
3. 检查主键列索引和特征起始列索引是否设置正确
4. 查看输出面板中的错误信息
5. 如果问题仍然存在，请联系技术支持

### Q9：什么时候需要勾选"转换特征值为数值"？

**A**：
- **需要勾选的情况**：
  - 两个文件中的数据类型不一致（一个是字符串"8"，另一个是数字8）
  - 想要按数值大小进行对比（而不是字符串对比）
  - 数据中包含整数和浮点数的混合（如 8 和 8.0）

- **不需要勾选的情况**：
  - 两个文件的数据类型完全一致
  - 需要严格按字符串进行对比
  - 数据中包含非数值的字符串（如 "N/A"、"未知"等）

**示例**：
```
不勾选时：
  "8" ≠ 8      （被认为不同）
  "8.0" ≠ 8    （被认为不同）

勾选后：
  "8" = 8      （被认为相同）
  "8.0" = 8    （被认为相同）
  8.0 = 8      （被认为相同）
```

## 🛠️ 技术说明

### 对比逻辑

1. **文件读取**：
   - CSV文件：自动检测编码（UTF-8、GBK等）
   - XLSX文件：使用 pandas 读取

2. **数据匹配**：
   - 根据主键列索引匹配两个文件中的记录
   - 支持主键列在不同位置

3. **特征对比**：
   - 从特征起始列开始，逐列对比数据
   - 自动处理数值类型转换
   - 记录差异的具体值

4. **报告生成**：
   - 差异特征汇总：统计每个特征的差异数量
   - 差异数据明细：记录每条差异记录的详细信息
   - 特征统计：计算匹配率、差异率等指标
   - 全量数据合并：合并两个文件的所有数据

### 性能优化

- 使用流式处理，支持大文件
- 多线程执行，不阻塞界面
- 实时输出进度，用户体验友好
