# 批量跑数模块特征解析修复说明

## 问题描述

在使用批量跑数模块（batch_run）时，发现输出的CSV文件中 `features` 列显示的是一个完整的字典字符串，而不是将每个特征展开为单独的列。

### 问题示例

输出CSV中的 `features` 列内容：
```
{'multi_loan_order_info_multiloanrangefuture0dto90d_futurebillingunclearinstalcnt': '41', 'multi_loan_order_info_multiloanrangefuture90dto180d_futurebillingunclearinstalcnt': '3', ...}
```

### 期望结果

每个特征应该是单独的一列：
```
multi_loan_order_info_multiloanrangefuture0dto90d_futurebillingunclearinstalcnt | multi_loan_order_info_multiloanrangefuture90dto180d_futurebillingunclearinstalcnt | ...
41                                                                                | 3                                                                                 | ...
```

## 根本原因

`batch_runner.py` 中的 `extract_features()` 方法没有正确处理 API 响应中的 `data.features` 结构。

### 原代码

```python
def extract_features(self, api_response: Dict[str, Any]) -> Dict[str, Any]:
    """从接口响应中提取特征值（data字段中的所有字段）"""
    if not api_response or "_error" in api_response:
        return {}
    
    data = api_response.get("data", {})
    if isinstance(data, dict):
        return data
    return {}
```

这个实现直接返回整个 `data` 字典，如果 API 返回的是 `data.features` 结构，那么 `features` 字段本身会被当作一个特征，其值是一个字典。

## 解决方案

更新 `extract_features()` 方法，使其能够识别并优先处理 `data.features` 结构：

### 修复后的代码

```python
def extract_features(self, api_response: Dict[str, Any]) -> Dict[str, Any]:
    """
    从接口响应中提取特征值
    
    支持两种结构：
    1. data.features 包含特征字段（优先）
    2. data 直接包含特征字段
    """
    if not api_response or "_error" in api_response:
        return {}
    
    data = api_response.get("data", {})
    if not isinstance(data, dict):
        return {}
    
    # 优先检查 data.features 结构
    if "features" in data and isinstance(data["features"], dict):
        return data["features"]
    
    # 如果没有 features 字段，直接返回 data
    return data
```

## 修复效果

### 修复前

API 响应：
```json
{
  "retCode": "0000",
  "data": {
    "features": {
      "age": "30",
      "income": "50000"
    },
    "apply_id": "12345"
  }
}
```

输出CSV：
```
apply_id,features
12345,"{'age': '30', 'income': '50000'}"
```

### 修复后

输出CSV：
```
apply_id,age,income
12345,30,50000
```

## 兼容性

此修复保持了向后兼容性：

1. **新结构（data.features）**：优先提取 `features` 中的字段
2. **旧结构（data直接包含特征）**：直接提取 `data` 中的所有字段
3. **空数据**：返回空字典
4. **错误响应**：返回空字典

## 测试验证

创建了测试脚本 `test_features_extraction.py` 验证以下场景：

1. ✅ data.features 结构 - 正确提取 features 中的字段
2. ✅ data 直接包含特征 - 正确提取 data 中的所有字段
3. ✅ data 为空 - 返回空字典
4. ✅ 响应有错误 - 返回空字典
5. ✅ features 为空字典 - 返回空字典
