# MyDataCheck 内存优化方案

## 问题分析

当前 MyDataCheck 项目在处理大文件时存在以下内存占用问题：

### 1. 一次性加载所有数据到内存
- `read_csv_with_encoding()` 函数会将整个CSV文件读入内存
- 对于几万行、上百列的数据，内存占用可达数GB
- 多个对比场景同时运行时，内存占用成倍增加

### 2. 重复存储数据
- 原始数据、对比结果、输出数据都在内存中
- `write_merged_data_csv()` 等函数会创建完整的数据副本

### 3. 索引构建占用大量内存
- 为了快速查找，会构建完整的索引字典
- 索引本身也占用大量内存

## 优化方案

### 方案一：流式处理（推荐）

#### 优点
- 内存占用降低 90% 以上
- 可处理任意大小的文件
- 处理速度基本不变

### 方案二：分批处理

如果不想大改代码，可以采用分批处理：

```python
def process_in_batches(file_path: str, batch_size: int = 5000):
    """将大文件分批处理"""
    headers, all_rows = read_csv_with_encoding(file_path)
    
    for i in range(0, len(all_rows), batch_size):
        batch = all_rows[i:i+batch_size]
        yield headers, batch
        # 处理完一批后，Python会自动回收内存
```

### 方案三：使用 Pandas（如果已安装）

Pandas 提供了更高效的内存管理：

import pandas as pd

## 监控内存使用

添加内存监控代码：

import psutil
import os

def print_memory_usage(label=""):
    process = psutil.Process(os.getpid())
    mem_mb = process.memory_info().rss / 1024 / 1024
    print(f"[{label}] 内存使用: {mem_mb:.2f} MB")

# 在关键位置调用
print_memory_usage("开始读取文件")
