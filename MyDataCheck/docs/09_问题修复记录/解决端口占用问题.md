# 解决端口5000占用问题

## 问题说明

macOS的AirPlay Receiver默认占用5000端口，导致Web服务无法启动。

```
Address already in use
Port 5000 is in use by another program.
```

---

## 解决方案

### 方案1：关闭AirPlay Receiver（推荐）

#### 步骤：
1. 打开"系统设置"（System Settings）
2. 点击"通用"（General）
3. 点击"隔空播放与接力"（AirDrop & Handoff）
4. 找到"隔空播放接收器"（AirPlay Receiver）
5. 将其设置为"关闭"（Off）

#### 命令行方式（快速）：
```bash
# 临时关闭AirPlay Receiver
sudo launchctl unload -w /System/Library/LaunchDaemons/com.apple.AirPlayXPCHelper.plist 2>/dev/null
```

---

### 方案2：修改Web服务端口

如果不想关闭AirPlay，可以修改Web服务使用其他端口（如5001）。

#### 修改步骤：

1. **修改 web/app.py**
```python
# 找到这一行（约第145行）
app.run(debug=True, host='0.0.0.0', port=5000)

# 改为
app.run(debug=True, host='0.0.0.0', port=5001)
```

2. **修改 start_web.sh**
```bash
# 找到这一行
echo "访问地址: http://127.0.0.1:5000"

# 改为
echo "访问地址: http://127.0.0.1:5001"
```

3. **重新启动服务**
```bash
./start_web.sh
```

4. **访问新地址**
```
http://localhost:5001
```

---

## 快速解决（推荐）

执行以下命令，自动切换到5001端口：

```bash
cd MyDataCheck

# 修改端口配置
sed -i '' 's/port=5000/port=5001/g' web/app.py
sed -i '' 's/5000/5001/g' start_web.sh

# 重新启动
./start_web.sh
```

## 常见问题

### Q1: 修改后仍然无法启动？
```bash
# 停止所有web_app进程
pkill -9 -f web_app.py

# 重新启动
./start_web.sh
```

### Q2: 如何恢复使用5000端口？
1. 关闭AirPlay Receiver（见方案1）
2. 将端口改回5000
3. 重新启动服务

### Q3: 如何查看服务是否正常运行？
```bash
# 查看进程
ps aux | grep web_app.py

# 查看端口
lsof -i:5001

# 查看日志
tail -f nohup.out
```
