# 可重用 Shell 脚本模板

本文档提供了 MyDataCheck 项目中高度可重用的脚本模板，可以直接用于其他项目。

---

## 模板 1: 通用清理脚本

**适用场景**: 定期清理旧文件（CSV、日志、临时文件等）

**可重复使用性**: ⭐⭐⭐⭐⭐

### 使用方法

```bash
# 1. 复制脚本到你的项目
cp scripts/cleanup/cleanup_now.sh your_project/

# 2. 修改配置（在脚本中）
# - CLEANUP_SCRIPT: 指向你的清理 Python 脚本
# - 默认保留天数

# 3. 使用
./cleanup_now.sh --dry-run  # 试运行
./cleanup_now.sh            # 正式执行
./cleanup_now.sh --days 7   # 自定义天数
```

### 需要修改的部分

```bash
# 修改清理脚本路径
CLEANUP_SCRIPT="$SCRIPT_DIR/cleanup_old_files.py"
# 改为你的脚本路径
CLEANUP_SCRIPT="$SCRIPT_DIR/your_cleanup_script.py"

# 修改默认保留天数
DAYS_TO_KEEP=5
# 改为你需要的天数
DAYS_TO_KEEP=7
```

### 特点

- ✅ 支持试运行模式
- ✅ 可自定义保留天数
- ✅ 交互式确认
- ✅ 详细的帮助信息
- ✅ 参数验证

## 模板 2: 自动清理定时任务

**适用场景**: 设置 cron 定时任务自动清理

**可重复使用性**: ⭐⭐⭐⭐⭐

### 使用方法

```bash
# 1. 复制脚本
cp scripts/cleanup/setup_auto_cleanup.sh your_project/

# 2. 修改配置
# - 清理脚本路径
# - 执行时间
# - 保留天数

# 3. 执行
./setup_auto_cleanup.sh
```

### 需要修改的部分

```bash
# 1. 修改清理脚本路径
CLEANUP_SCRIPT="$SCRIPT_DIR/cleanup_old_files.py"

# 2. 修改 cron 时间和参数
# 格式: 分 时 日 月 周
CRON_CMD="0 2 * * * cd $SCRIPT_DIR && $PYTHON_CMD $CLEANUP_SCRIPT --days 5 >> $LOG_FILE 2>&1"
#        ↑ ↑
#        | 凌晨2点
#        0分

# 示例：
# 每天凌晨3点: "0 3 * * *"
# 每周一凌晨2点: "0 2 * * 1"
# 每月1号凌晨2点: "0 2 1 * *"
```

### 特点

- ✅ 自动配置 cron 任务
- ✅ 检测并替换已存在的任务
- ✅ 自动创建日志目录
- ✅ 详细的配置说明

## 模板 3: 服务停止脚本

**适用场景**: 停止占用特定端口的服务

**可重复使用性**: ⭐⭐⭐⭐⭐

### 使用方法

```bash
# 1. 复制脚本
cp scripts/startup/stop_web.sh your_project/stop_service.sh

# 2. 修改端口号
# 将所有 5001 改为你的端口

# 3. 使用
./stop_service.sh
```

### 需要修改的部分

```bash
# 修改端口号（所有出现的地方）
PIDS=$(lsof -ti:5001 2>/dev/null)
# 改为
PIDS=$(lsof -ti:8080 2>/dev/null)

# 修改进程名（如果需要）
PIDS=$(pgrep -f "web_app.py" 2>/dev/null)
# 改为
PIDS=$(pgrep -f "your_app.py" 2>/dev/null)
```

### 完整模板

```bash
#!/bin/bash
# 服务停止脚本（通用模板）

PORT=5001  # 修改为你的端口
PROCESS_NAME="web_app.py"  # 修改为你的进程名

echo "=========================================="
echo "停止服务（端口 $PORT）"
echo "=========================================="
echo ""

# 查找并停止进程
PIDS=$(lsof -ti:$PORT 2>/dev/null)

if [ -z "$PIDS" ]; then
    echo "未找到运行在端口 $PORT 的进程"
    PIDS=$(pgrep -f "$PROCESS_NAME" 2>/dev/null)
    if [ -z "$PIDS" ]; then
        echo "未找到 $PROCESS_NAME 进程"
        exit 0
    fi
fi

echo "找到以下进程:"
echo "$PIDS" | while read PID; do
    if [ ! -z "$PID" ]; then
        echo "  PID: $PID"
        ps -p $PID -o command= 2>/dev/null | head -1
    fi
done

echo ""
echo "正在强制停止..."

lsof -ti:$PORT 2>/dev/null | xargs kill -9 2>/dev/null
pkill -9 -f "$PROCESS_NAME" 2>/dev/null

sleep 1

REMAINING=$(lsof -ti:$PORT 2>/dev/null)
if [ -z "$REMAINING" ]; then
    echo "✅ 服务已成功停止"
else
    echo "⚠️  警告: 仍有进程占用端口 $PORT"
    echo "剩余进程PID: $REMAINING"
    echo "$REMAINING" | xargs kill -9 2>/dev/null
fi
```

### 特点

- ✅ 自动查找进程
- ✅ 强制停止
- ✅ 二次确认
- ✅ 友好提示

## 模板 4: Python 环境配置脚本

**适用场景**: 自动配置 Python 虚拟环境

**可重复使用性**: ⭐⭐⭐⭐

### 使用方法

```bash
# 1. 复制脚本
cp scripts/setup/setup_python312.sh your_project/setup_env.sh

# 2. 修改配置
# - Python 版本
# - 依赖文件

# 3. 执行
./setup_env.sh
```

### 需要修改的部分

```bash
# 1. 修改 Python 版本
if command -v python3.12 &> /dev/null; then
# 改为
if command -v python3.11 &> /dev/null; then

# 2. 修改安装命令
brew install python@3.12
# 改为
brew install python@3.11

# 3. 修改虚拟环境创建命令
python3.12 -m venv .venv
# 改为
python3.11 -m venv .venv

# 4. 修改依赖文件（如果不同）
pip install -r requirements.txt
# 改为
pip install -r your_requirements.txt
```

### 通用模板

```bash
#!/bin/bash
# Python 环境配置脚本（通用模板）

set -e

PYTHON_VERSION="3.12"  # 修改为你需要的版本
PYTHON_CMD="python${PYTHON_VERSION}"
REQUIREMENTS_FILE="requirements.txt"  # 修改为你的依赖文件

echo "=========================================="
echo "Python ${PYTHON_VERSION} 环境设置"
echo "=========================================="
echo ""

# 检查并安装 Python
if command -v $PYTHON_CMD &> /dev/null; then
    echo "✅ Python ${PYTHON_VERSION} 已安装"
else
    echo "正在安装 Python ${PYTHON_VERSION}..."
    brew install python@${PYTHON_VERSION}
fi

# 删除旧虚拟环境
if [ -d ".venv" ]; then
    rm -rf .venv
    echo "✅ 已删除旧虚拟环境"
fi

# 创建新虚拟环境
echo "创建虚拟环境..."
$PYTHON_CMD -m venv .venv

# 激活虚拟环境
source .venv/bin/activate

# 升级 pip
pip install --upgrade pip

# 安装依赖
pip install -r $REQUIREMENTS_FILE

echo "✅ 环境配置完成！"
```

## 模板 5: 服务启动脚本

**适用场景**: 启动 Python Web 服务

**可重复使用性**: ⭐⭐⭐

### 通用模板

```bash
#!/bin/bash
# Web 服务启动脚本（通用模板）

APP_NAME="web_app.py"  # 修改为你的应用入口
PORT=5001              # 修改为你的端口
VENV_PATH=".venv"      # 修改为你的虚拟环境路径

echo "=========================================="
echo "启动 Web 服务"
echo "=========================================="
echo ""

# 检查虚拟环境
if [ ! -d "$VENV_PATH" ]; then
    echo "❌ 错误: 未找到虚拟环境 $VENV_PATH"
    exit 1
fi

# 激活虚拟环境
source $VENV_PATH/bin/activate

# 检查依赖（可选）
# python -c "import flask" 2>/dev/null || pip install flask

# 启动服务
echo "启动服务..."
echo "访问地址: http://127.0.0.1:$PORT"
echo "按 Ctrl+C 停止"
echo ""

python $APP_NAME
```

## 模板 6: 依赖安装脚本

**适用场景**: 快速安装特定依赖

**可重复使用性**: ⭐⭐⭐⭐

### 通用模板

```bash
#!/bin/bash
# 依赖安装脚本（通用模板）

PACKAGE_NAME="psutil"     # 修改为你的包名
PACKAGE_VERSION=">=5.9.0" # 修改为版本要求
VENV_PATH=".venv"         # 修改为虚拟环境路径

echo "=========================================="
echo "安装 $PACKAGE_NAME"
echo "=========================================="
echo ""

# 检查虚拟环境
if [ ! -d "$VENV_PATH" ]; then
    echo "❌ 错误: 未找到虚拟环境"
    exit 1
fi

# 激活虚拟环境
source $VENV_PATH/bin/activate

# 安装包
pip install "$PACKAGE_NAME$PACKAGE_VERSION"

# 验证安装
python -c "import ${PACKAGE_NAME}; print('✅ ${PACKAGE_NAME} 已安装')"

echo "✅ 安装完成"
```

## 使用建议

### 1. 选择合适的模板

| 需求 | 推荐模板 | 可重复使用性 |
|------|---------|-------------|
| 定期清理文件 | 模板 1 + 模板 2 | ⭐⭐⭐⭐⭐ |
| 停止服务 | 模板 3 | ⭐⭐⭐⭐⭐ |
| 环境配置 | 模板 4 | ⭐⭐⭐⭐ |
| 启动服务 | 模板 5 | ⭐⭐⭐ |
| 安装依赖 | 模板 6 | ⭐⭐⭐⭐ |

### 2. 修改步骤

1. **复制模板** - 复制到你的项目
2. **修改配置** - 根据注释修改参数
3. **测试运行** - 先在测试环境运行
4. **正式使用** - 确认无误后使用

### 3. 最佳实践

- ✅ 保留原始注释
- ✅ 添加项目特定的说明
- ✅ 使用有意义的变量名
- ✅ 添加错误处理
- ✅ 提供帮助信息

### 4. 注意事项

- ⚠️ 修改路径时使用绝对路径或相对路径
- ⚠️ 测试前先备份重要数据
- ⚠️ 检查脚本权限（chmod +x）
- ⚠️ 验证 Python 版本兼容性
