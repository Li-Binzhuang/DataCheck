# 接口数据对比 - 忽略默认填充值配置说明

## 功能概述

在接口数据对比模块中新增了"忽略默认填充值"配置项，允许用户在对比时将 `-999` 和 `null` 视为一致。

## 使用场景

在某些业务场景中，数据缺失时会使用默认填充值（如 `-999`）或 `null` 来表示。当对比CSV文件和接口返回数据时，如果一个使用 `-999`，另一个使用 `null`，实际上它们表达的含义是相同的（都表示数据缺失）。

启用此配置项后，对比时会将这两种表示方式视为一致，不会被标记为差异。

## 配置方式

### Web界面配置

1. 进入"接口数据对比"页面
2. 在"全局默认配置"区域找到"忽略默认填充值"复选框
3. 勾选该选项后，`-999` 和 `null` 会被视为一致
4. 默认状态：不勾选（不忽略默认填充值）

### 配置说明

- **默认状态**：不勾选（不忽略）
- **勾选后**：`-999` 和 `null` 被视为一致，不会被标记为差异
- **不勾选**：`-999` 和 `null` 被视为不同的值，会被标记为差异

## 对比逻辑

### 启用忽略默认填充值时

以下情况会被视为一致：
- CSV值 = `-999`，API值 = `null` → 一致
- CSV值 = `null`，API值 = `-999` → 一致
- CSV值 = `-999`，API值 = `-999` → 一致
- CSV值 = `null`，API值 = `null` → 一致

### 不启用时

- CSV值 = `-999`，API值 = `null` → 不一致（差异）
- CSV值 = `null`，API值 = `-999` → 不一致（差异）

## 示例

### 示例1：启用忽略默认填充值

**CSV文件**：
```
custNo,baseTime,特征A,特征B
001,2024-01-01,-999,100
002,2024-01-02,50,null
```

**接口返回**：
```json
{
  "custNo": "001",
  "特征A": null,
  "特征B": 100
}
{
  "custNo": "002",
  "特征A": 50,
  "特征B": -999
}
```

**对比结果**：
- custNo=001，特征A：`-999` vs `null` → 一致（因为启用了忽略默认填充值）
- custNo=002，特征B：`null` vs `-999` → 一致（因为启用了忽略默认填充值）
- 无差异记录

### 示例2：不启用忽略默认填充值

使用相同的数据：

**对比结果**：
- custNo=001，特征A：`-999` vs `null` → 不一致（差异）
- custNo=002，特征B：`null` vs `-999` → 不一致（差异）
- 2条差异记录

## 配置文件格式

在 `api_comparison/config.json` 中：

```json
{
  "scenarios": [
    {
      "name": "场景1",
      "enabled": true,
      "ignore_default_fill": false,
      ...
    }
  ],
  "global_config": {
    "default_ignore_default_fill": false,
    ...
  }
}
```

- `scenarios[].ignore_default_fill`：场景级别配置（优先级高）
- `global_config.default_ignore_default_fill`：全局默认配置

## 技术实现

### 后端实现

在 `api_comparison/job/streaming_comparator.py` 中：

```python
class StreamingComparator:
    def __init__(self, ..., ignore_default_fill: bool = False):
        self.ignore_default_fill = ignore_default_fill
    
    def _compare_single_row(self, ...):
        # 比较值时传递ignore_default_fill参数
        is_match = compare_values(csv_value, api_value, header, self.ignore_default_fill)
```

在 `api_comparison/job/process_executor.py` 中：

```python
def execute_single_scenario(...):
    # 从配置中获取ignore_default_fill
    ignore_default_fill = scenario_config.get('ignore_default_fill',
                                               global_config.get('default_ignore_default_fill', False))
    
    # 传递给StreamingComparator
    comparator = StreamingComparator(..., ignore_default_fill=ignore_default_fill)
```

### 前端实现

在 `templates/index.html` 中添加复选框：

```html
<div class="form-group">
    <div class="checkbox-group">

        <label for="global_ignore_default_fill">忽略默认填充值</label>

    <p>勾选后，-999 和 null 会被视为一致（默认不勾选）</p>

```

在 `static/js/api-compare.js` 中获取配置值：

```javascript

    ...
    default_ignore_default_fill: getGlobalChecked('global_ignore_default_fill', false)
}
```

## 注意事项

1. **默认行为**：默认不勾选，保持原有的严格对比逻辑
2. **适用范围**：仅适用于接口数据对比模块
3. **数值类型**：只有 `-999`（数值）和 `null`（空值）会被视为默认填充值
4. **其他值**：其他数值（如 `-998`、`0`）不会被视为默认填充值
5. **配置优先级**：场景级别配置 > 全局默认配置
