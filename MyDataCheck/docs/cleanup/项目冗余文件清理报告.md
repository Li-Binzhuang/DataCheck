## 清理方案

### 自动清理脚本

创建了 `cleanup_redundant_files.py` 脚本，功能包括：

1. **删除 .DS_Store 文件**
   - 递归扫描所有目录
   - 自动删除所有 .DS_Store 文件

2. **删除 Python 缓存**
   - 删除所有 `__pycache__/` 目录
   - 删除所有 `.pyc` 文件

3. **归档测试脚本**
   - 移动到 `tests/archived/` 目录
   - 创建归档说明文档

4. **清理测试输出**
   - 删除所有测试生成的 CSV 文件
   - 显示释放的磁盘空间

5. **删除测试模板**
   - 删除不再使用的测试 HTML 文件

### 使用方法

```bash
cd MyDataCheck
python cleanup_redundant_files.py
```

脚本会提示确认后执行清理操作。

## 后续建议

### 1. 添加到 .gitignore

确保以下内容在 `.gitignore` 中：

```gitignore
# macOS
.DS_Store

# Python
__pycache__/
*.py[cod]
*$py.class
*.so

# 测试输出
outputdata/test_*.csv
outputdata/*/test_*.csv
```

### 2. 定期清理

建议定期运行清理脚本：

```bash
# 每周或每月运行一次
python cleanup_redundant_files.py
```

### 3. 自动化清理

可以添加到 Git hooks 或 CI/CD 流程中：

```bash
# pre-commit hook
#!/bin/bash
find . -name ".DS_Store" -delete
find . -name "__pycache__" -type d -exec rm -rf {} +
```
