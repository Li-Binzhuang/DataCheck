# CDC JSON 数据结构说明文档

## 一、数据概述

这个 JSON 文件包含的是**墨西哥征信局（Círculo de Crédito）**返回的用户信贷数据，用于风控模型的特征工程。

## 二、数据结构

### 2.1 顶层字段（申请基本信息）

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `apply_id` | string | 申请ID，唯一标识一次借款申请 | "1065991091661283329" |
| `apply_time` | string | 申请时间 | "2025-11-25 01:53:44.943" |
| `approve_state` | string | 审批状态 | "CYCLE_PASS"（循环通过） |
| `blind_lend` | null/int | 盲贷标记 | null |
| `credit_apply_cnt` | int | 授信申请次数 | 1 |
| `credit_limit_amount` | string | 授信额度 | "800.00000000" |
| `fpd7` | int | First Payment Default 7天（首次还款违约标记） | 0=正常, 1=违约 |
| `spd7` | int | Second Payment Default 7天（第二次还款违约标记） | 0=正常, 1=违约 |
| `principal_amount_borrowed` | string | 实际借款本金 | "700.00000000" |
| `use_amount` | string | 用信金额 | "700.00000000" |

### 2.2 response_body（征信局返回的详细数据）

这是一个**嵌套的 JSON 字符串**，需要二次解析。包含以下主要部分：

#### 2.2.1 consultas（征信查询记录数组）

记录了该用户在各个金融机构的征信查询历史，共 **55 条记录**。

**字段说明：**

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `fechaConsulta` | 查询日期 | "2025-11-25" |
| `nombreOtorgante` | 查询机构名称 | "COMPANIA DE PRESTAMO PERSONAL" |
| `importeCredito` | 查询时的信贷金额 | 1000 |
| `tipoCredito` | 信贷类型 | "M", "F", "Q", "TC" 等 |
| `claveUnidadMonetaria` | 货币单位代码 | "MX"（墨西哥比索） |
| `tipoResponsabilidad` | 责任类型 | "I"（个人） |

**信贷类型代码：**
- `M` = Mensual（月付）
- `F` = Fijo（固定）
- `Q` = Quincenal（双周付）
- `TC` = Tarjeta de Crédito（信用卡）
- `PP` = Préstamo Personal（个人贷款）
- `NC` = No Clasificado（未分类）
- `LC` = Línea de Crédito（信用额度）
- `OT` = Otros（其他）
- `HV` = Hipoteca Vivienda（住房抵押贷款）
- `CC` = Crédito al Consumo（消费信贷）

#### 2.2.2 creditos（信贷账户详情数组）

记录了该用户在各个金融机构的实际信贷账户信息，共 **142 条记录**。

**核心字段说明：**

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `nombreOtorgante` | 贷款机构名称 | "MERCANCIA PARA HOGAR Y OFICINA" |
| `creditoMaximo` | 该账户的历史最大信贷额度 | 7774 |
| `limiteCredito` | 当前信用额度 | 0 |
| `saldoActual` | 当前余额（欠款） | 0 |
| `saldoVencido` | 逾期金额 | 0 |
| `montoPagar` | 应付金额 | 0 |
| `peorAtraso` | 最差逾期等级（0-9+） | 1 |
| `numeroPagos` | 总期数 | 26 |
| `numeroPagosVencidos` | 逾期期数 | 1 |
| `fechaAperturaCuenta` | 账户开户日期 | "2023-01-26" |
| `fechaCierreCuenta` | 账户关闭日期 | "2023-09-14" |
| `fechaPeorAtraso` | 最差逾期发生日期 | "2023-04-06" |
| `fechaUltimoPago` | 最后还款日期 | "2023-09-14" |
| `historicoPagos` | 还款历史记录字符串 | " V--01 V" |
| `frecuenciaPagos` | 还款频率 | "S"=周, "M"=月, "Q"=双周, "U"=一次性 |
| `tipoCredito` | 信贷类型 | "PP", "CC", "LC" 等 |
| `tipoCuenta` | 账户类型 | "F"=固定, "R"=循环, "H"=抵押 |
| `clavePrevencion` | 预防标记 | "CC", "CL", "PC", "CD" |

**逾期等级说明（peorAtraso）：**
- `0` = 正常，无逾期
- `1` = 逾期 1-29 天
- `2` = 逾期 30-59 天
- `3` = 逾期 60-89 天
- `4` = 逾期 90-119 天
- `5` = 逾期 120-149 天
- `6` = 逾期 150-179 天
- `7` = 逾期 180+ 天
- `8` = 严重逾期
- `9` = 坏账

**还款历史记录（historicoPagos）：**
- `V` = Al corriente（正常还款）
- `01-09` = 逾期等级
- `-` = 无数据或未到期
- 从右到左表示从最近到最早的还款记录

示例：`" V--01 V"` 表示：
- 最近一期：正常（V）
- 往前两期：无数据（--）
- 再往前一期：逾期1级（01）
- 最早一期：正常（V）

#### 2.2.3 其他字段

- `domicilios`：地址信息数组
- `persona`：个人基本信息（姓名、出生日期、CURP、RFC等）
- `scores`：信用评分（如 FICO 分数）
- `folioConsulta`：查询流水号
- `mensajes`：响应消息

## 四、特征工程用途

从这些数据可以提取的风控特征包括：

### 4.1 征信查询特征
- 近 N 天/月的查询次数
- 查询机构类型分布
- 查询频率趋势
- 多头借贷指标（同时在多个机构查询）

### 4.2 信贷账户特征
- 总账户数
- 活跃账户数
- 已关闭账户数
- 平均账户年龄
- 最老/最新账户开户时间

### 4.3 额度使用特征
- 总授信额度
- 总欠款金额
- 信贷使用率（欠款/额度）
- 平均单账户额度

### 4.4 逾期行为特征
- 逾期账户数量
- 逾期账户占比
- 最差逾期等级
- 当前逾期金额
- 历史逾期金额
- 逾期频率
- 最近一次逾期距今天数

### 4.5 还款历史特征
- 从 `historicoPagos` 字段提取：
  - 正常还款次数
  - 逾期还款次数
  - 连续正常还款期数
  - 连续逾期期数
  - 还款稳定性指标

### 4.6 机构类型特征
- 不同类型机构的账户数
- 在银行/小贷公司/消费金融的借款情况
- 机构多样性指标

### 4.7 时间序列特征
- 近 3/6/12 个月的新增账户数
- 近期查询频率变化
- 逾期趋势
- 还款行为变化

## 五、数据质量说明

1. **数据完整性**：部分字段可能为空（null）或缺失
2. **数据时效性**：`fechaReporte` 表示数据报告日期
3. **数据准确性**：依赖于各金融机构的上报质量
4. **隐私保护**：实际使用时需要脱敏处理个人信息

## 六、使用建议

1. **数据预处理**：
   - 解析嵌套的 `response_body` JSON 字符串
   - 处理缺失值和异常值
   - 日期字段转换为标准格式

2. **特征提取**：
   - 使用 pandas 处理数组数据
   - 计算聚合统计特征
   - 构建时间窗口特征

3. **特征选择**：
   - 根据业务场景选择相关特征
   - 进行特征重要性分析
   - 避免特征冗余

4. **模型应用**：
   - 结合其他数据源（如设备信息、行为数据）
   - 进行特征工程和模型训练
   - 定期更新和验证模型效果
