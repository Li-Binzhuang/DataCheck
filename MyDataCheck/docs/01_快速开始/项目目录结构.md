## 快速访问

### 启动服务

根目录保留了常用脚本的软链接，可以直接使用：

```bash
# 在根目录直接启动（推荐）
cd MyDataCheck
./start_web.sh

# 停止服务
./stop_web.sh
```

### 使用脚本

```bash
# 清理脚本
./scripts/cleanup/cleanup_now.sh

# 维护脚本
python scripts/maintenance/check_startup.py

# 查看脚本说明
cat scripts/cleanup/README.md
cat scripts/startup/README.md
cat scripts/maintenance/README.md
```

## 软链接说明

根目录的 `start_web.sh` 和 `stop_web.sh` 是软链接：

```bash
start_web.sh -> scripts/startup/start_web.sh
stop_web.sh -> scripts/startup/stop_web.sh
```

**优势**:
- 保持根目录整洁
- 便于快速访问常用脚本
- 实际文件统一管理

**验证软链接**:
```bash
ls -la start_web.sh stop_web.sh
```

## 目录设计原则

1. **根目录精简**: 只保留核心文件和常用脚本的软链接
2. **分类管理**: 脚本按功能分类到子目录
3. **易于访问**: 常用脚本提供软链接
4. **文档完善**: 每个目录都有 README 说明
5. **便于维护**: 结构清晰，职责明确
