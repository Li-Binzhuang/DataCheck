- 特征值显示不完整（如 `cdc_boss_num_pagos_mean_607`）

### 修改后代码
```python
if WRITE_FEATURES_CSV:
    csv_path = Path("outputs") / csv_filename
    csv_path.parent.mkdir(parents=True, exist_ok=True)
    
    # zlf update: 对数值特征列保留6位小数
    _features_to_write = features_df.copy()
    _round_cols = [c for c in _features_to_write.columns if c not in {"apply_id", "request_time"}]
    _features_to_write[_round_cols] = _features_to_write[_round_cols].apply(pd.to_numeric, errors="coerce").round(6)
    
    # 输出处理后的数据
    _features_to_write.to_csv(csv_path, index=False, encoding="utf-8-sig")
    
    if WRITE_SAMPLE_200:
        features_sample200 = _features_to_write.head(200)  # 使用处理后的数据
        features_sample200.to_csv(csv_sample200_path, index=False, encoding="utf-8-sig")
```

### 关键改进
1. ✅ 添加 `zlf update` 注释标识
2. ✅ 创建数据副本避免修改原始数据
3. ✅ 识别需要处理的数值列（排除 apply_id 和 request_time）
4. ✅ 使用 `pd.to_numeric()` 确保数据类型正确
5. ✅ 应用 `.round(6)` 统一保留6位小数
6. ✅ sample200样本也使用处理后的数据

## 处理逻辑说明

### 列识别
```python
_round_cols = [c for c in _features_to_write.columns if c not in {"apply_id", "request_time"}]
```
- 排除 `apply_id`：申请ID，不需要精度处理
- 排除 `request_time`：请求时间，不需要精度处理
- 其他列：所有数值特征列

### 数值转换和精度控制
```python
_features_to_write[_round_cols] = _features_to_write[_round_cols].apply(pd.to_numeric, errors="coerce").round(6)
```
- `pd.to_numeric(errors="coerce")`：将列转换为数值类型，无法转换的设为NaN
- `.round(6)`：保留6位小数
