# Web 界面功能测试指南

## 测试目标

验证 Web 界面的动态接口参数配置功能是否正常工作。

## 测试前准备

1. 启动 Web 服务:
```bash
cd MyDataCheck
python web_app.py
```

2. 在浏览器中打开: http://localhost:5000

## 测试步骤

### 测试 1: 添加参数功能

1. 在"接口数据对比"标签页中，找到任意场景卡片
2. 在"接口参数配置"区域，点击"➕ 添加参数"按钮
3. **预期结果**: 应该出现一个新的参数配置行，包含:
   - 参数名称输入框
   - 列索引输入框
   - 时间字段复选框
   - 删除按钮 (🗑️)

### 测试 2: 删除参数功能

1. 在有多个参数的场景中，点击某个参数的删除按钮 (🗑️)
2. **预期结果**: 该参数配置行应该被删除
3. **注意**: 如果只剩一个参数，应该提示"至少需要保留一个接口参数"

### 测试 3: 参数配置输入

1. 在参数名称输入框中输入: `applyId`
2. 在列索引输入框中输入: `1`
3. 勾选"时间字段"复选框
4. **预期结果**: 所有输入应该正常显示和保存

### 测试 4: 列索引为空（不传参）

1. 添加一个新参数
2. 在参数名称输入框中输入: `optionalParam`
3. **将列索引输入框留空**
4. 点击"💾 保存配置"按钮
5. **预期结果**: 
   - 配置应该保存成功
   - 该参数不会被添加到实际的请求参数中（因为列索引为空）

### 测试 5: 保存和加载配置

1. 配置多个参数（包括一些列索引为空的参数）
2. 点击"💾 保存配置"按钮
3. 刷新页面
4. **预期结果**: 
   - 所有参数配置应该被正确加载
   - 包括参数名称、列索引、时间字段标记

### 测试 6: 执行对比（完整流程）

**前提条件**: 
- 已上传测试 CSV 文件
- 已配置 API 地址
- 已配置接口参数

**步骤**:
1. 配置接口参数:
   - 参数1: custNo, 列索引: 0, 非时间字段
   - 参数2: baseTime, 列索引: 2, 时间字段
2. 点击"▶️ 执行对比"按钮
3. **预期结果**:
   - 输出面板应该显示执行日志
   - 应该看到"第一条请求调试信息"
   - 应该显示请求入参和响应数据
   - 执行完成后应该显示成功消息

### 测试 7: 列索引为空时不传参（实际请求验证）

**步骤**:
1. 配置接口参数:
   - 参数1: custNo, 列索引: 0, 非时间字段
   - 参数2: baseTime, 列索引: (留空), 时间字段
2. 点击"▶️ 执行对比"按钮
3. **预期结果**:
   - 在"第一条请求调试信息"中，应该只看到 custNo 参数
   - baseTime 参数应该被跳过（显示"参数 'baseTime' 的列索引为空，跳过该参数"）
   - 实际请求只包含 custNo 参数

## 验证要点

### UI 元素检查

- [ ] "接口参数配置"区域正常显示
- [ ] "➕ 添加参数"按钮可点击
- [ ] 参数配置行包含所有必要字段
- [ ] 删除按钮 (🗑️) 正常工作
- [ ] 提示文本正确显示: "💡 提示: 参数名称对应接口入参字段名，列索引指定从CSV文件的哪一列读取该参数的值。**列索引留空则该参数不作为入参传递。**"

### 功能检查

- [ ] 可以添加任意数量的参数
- [ ] 可以删除参数（至少保留一个）
- [ ] 参数名称可以自定义
- [ ] 列索引可以留空
- [ ] 时间字段复选框正常工作
- [ ] 配置可以保存和加载
- [ ] 列索引为空时，该参数不会被传递给接口

### 数据验证

- [ ] 保存的配置包含 `api_params` 数组
- [ ] 每个参数包含 `param_name`, `column_index`, `is_time_field`
- [ ] 列索引为空的参数不会出现在 `api_params` 中
- [ ] 执行对比时，只传递有效的参数

## 常见问题排查

### 问题 1: 添加参数按钮不工作

**检查**:
- 浏览器控制台是否有 JavaScript 错误
- `addApiParam()` 函数是否正确定义

### 问题 2: 删除参数后编号错乱

**检查**:
- `removeApiParam()` 函数是否正确重新编号
- `data-param-index` 属性是否正确更新

### 问题 3: 配置保存后丢失

**检查**:
- `collectConfig()` 函数是否正确收集参数
- 列索引为空的参数是否被正确过滤

### 问题 4: 执行时参数未传递

**检查**:
- `fetch_api_data.py` 中的 `process_row()` 方法是否正确处理空列索引
- 调试日志中是否显示"跳过参数"消息
