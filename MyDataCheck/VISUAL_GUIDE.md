## 🎨 新增的 UI 元素

### 1. 接口参数配置区域

**位置**: 在场景卡片中，API地址下方

**外观**: 
- 黄色背景 (#fff3cd)
- 黄色边框 (#ffc107)
- 标题: "接口参数配置"

**包含元素**:
- "➕ 添加参数" 按钮（右上角）
- 参数配置列表
- 提示文本

---

### 2. 参数配置行

每个参数包含:

```
┌─────────────────────────────────────────────┐
│ 参数名称: [custNo        ]                  │
│ 列索引:   [0             ]                  │
│ □ 时间字段                          [🗑️]   │
└─────────────────────────────────────────────┘
```

**字段说明**:
- **参数名称**: 接口入参的字段名（如 custNo, applyId, baseTime）
- **列索引**: 从 CSV 文件的哪一列读取该参数的值（从 0 开始）
- **时间字段**: 勾选后会对该参数进行时间格式标准化
- **删除按钮**: 点击删除该参数（至少保留一个）

---

### 3. 按钮样式

#### 添加参数按钮
```
[➕ 添加参数]
```
- 颜色: 绿色 (#28a745)
- 位置: 参数配置区域右上角
- 功能: 添加新的参数配置行

#### 删除参数按钮
```
[🗑️]
```
- 颜色: 红色 (#dc3545)
- 位置: 每个参数配置行的右侧
- 功能: 删除该参数
- 限制: 至少保留一个参数

## 🎯 常见配置场景

### 场景 1: 标准双参数接口

**需求**: custNo + baseTime

**配置**:
```
参数1: custNo, 列索引: 0, 非时间字段
参数2: baseTime, 列索引: 2, 时间字段
```

**请求示例**:
```json
{
  "custNo": "123456",
  "baseTime": "2024-01-21T10:30:00.000"
}
```

---

### 场景 2: 单参数接口

**需求**: 只有 applyId

**配置**:
```
参数1: applyId, 列索引: 1, 非时间字段
```

**操作**:
1. 删除 baseTime 参数（点击 🗑️）
2. 修改 custNo 为 applyId
3. 修改列索引为 1

**请求示例**:
```json
{
  "applyId": "APPLY001"
}
```

---

### 场景 3: 三参数接口

**需求**: custNo + applyId + baseTime

**配置**:
```
参数1: custNo, 列索引: 0, 非时间字段
参数2: applyId, 列索引: 1, 非时间字段
参数3: baseTime, 列索引: 3, 时间字段
```

**操作**:
1. 保留默认的 custNo 和 baseTime
2. 点击"➕ 添加参数"
3. 配置新参数: applyId, 列索引: 1

**请求示例**:
```json
{
  "custNo": "123456",
  "applyId": "APPLY001",
  "baseTime": "2024-01-21T10:30:00.000"
}
```

---

### 场景 4: 可选参数（列索引为空）

**需求**: custNo 必传，baseTime 可选（不传）

**配置**:
```
参数1: custNo, 列索引: 0, 非时间字段
参数2: baseTime, 列索引: (留空), 时间字段
```

**操作**:
1. 保留 custNo 参数
2. 将 baseTime 的列索引清空

**请求示例**:
```json
{
  "custNo": "123456"
}
```

**说明**: baseTime 参数不会被传递给接口

## 💡 提示和技巧

### 1. 快速配置

- **复制场景**: 添加新场景后，可以复制现有场景的配置
- **批量修改**: 先配置一个场景，保存后再添加其他场景

### 2. 参数顺序

- 参数顺序不影响功能
- 建议按照接口文档的顺序配置，便于维护

### 3. 列索引

- 列索引从 0 开始（CSV 第 1 列 = 索引 0）
- 可以使用 Excel 或文本编辑器查看 CSV 列号

### 4. 时间字段

- 勾选"时间字段"后，系统会自动:
  - 移除字母 'a' 或 'A'
  - 标准化为 YYYY-MM-DDTHH:MM:SS.SSS 格式
  - 根据配置决定是否加 1 秒

### 5. 验证配置

- 保存配置后，刷新页面验证配置是否正确加载
- 执行对比前，检查"第一条请求调试信息"确认参数正确
