# 执行输出日志完整显示修复

## 问题描述

Web 页面中"执行输出"区域的日志显示不完整，执行到后面时前面的日志就看不到了。

## 问题原因

代码中有两个限制导致日志不完整：

1. **采样机制**：当输出行数超过 500 行时，开始采样显示（每 5 条显示 1 条）
2. **最大行数限制**：最多只保留 10 行日志，超过后会删除旧的行

这些限制在 `config.js` 中定义：
```javascript
const MAX_OUTPUT_LINES = 10;    // 最多保留10行
const SAMPLE_RATE = 5;          // 采样率（每5条显示1条）
```

## 修复方案

### 1. 调整配置参数

修改 `static/js/config.js`：
```javascript
const MAX_OUTPUT_LINES = 10000;  // 最多保留10000行（足够显示完整日志）
const SAMPLE_RATE = 1;           // 采样率设为1，显示所有日志
```

### 2. 移除采样逻辑

修改 `static/js/api-compare.js` 中的 `appendOutput` 函数：
- 移除采样判断逻辑
- 显示所有日志消息
- 保留最大行数限制（防止内存溢出）

修改 `static/js/ui.js` 中的 `appendOutput` 函数：
- 移除关键信息判断和采样逻辑
- 统一显示所有日志

## 修复效果

修复后的效果：

✅ **显示完整日志**：所有日志消息都会显示，不会被采样过滤  
✅ **可查看历史日志**：最多保留 10000 行日志，足够查看完整执行过程  
✅ **自动滚动**：新日志出现时自动滚动到底部  
✅ **手动滚动**：可以向上滚动查看历史日志  
✅ **性能优化**：超过 10000 行时自动删除最旧的日志，防止内存溢出  

## 使用说明

1. **查看实时日志**：执行任务时，日志会实时显示在"执行输出"区域
2. **查看历史日志**：使用鼠标滚轮或滚动条向上滚动，查看之前的日志
3. **清空日志**：点击"清空输出"按钮可以清空当前显示的日志
4. **日志颜色**：
   - 🔴 红色：错误信息
   - 🟢 绿色：成功信息
   - 🟡 黄色：警告信息
   - 🔵 蓝色：普通信息

## 技术细节

### 日志显示流程

1. 后端通过 Server-Sent Events (SSE) 流式推送日志
2. 前端接收日志并解析 JSON 数据
3. 调用 `appendOutput` 函数将日志添加到输出面板
4. 自动滚动到底部（可手动滚动查看历史）
5. 超过最大行数时删除最旧的日志

### 性能考虑

- **最大行数限制**：10000 行，足够显示完整日志
- **防抖滚动**：使用 50ms 防抖，避免频繁滚动影响性能
- **自动清理**：超过最大行数时自动删除旧日志，防止内存溢出
