# SQL特征说明文档

## 概述

本文档描述了 `sql1.sql` 中定义的贷款订单相关特征。这些特征基于用户的还款计划数据，用于分析用户的历史贷款行为。

**数据来源**：
- 目标用户：每天发起用信申请的用户 + 用信申请时间
- 还款计划表：`ods_mx_ast_asset_repay_plan_df`
- 借据信息表：`ods_mx_ast_asset_loan_info_df`
- 用信申请表：`ods_mx_aprv_approve_use_credit_apply_df`
- 授信申请表：`ods_mx_aprv_approve_credit_apply_df`

**缺失值处理**：所有特征使用 `coalesce(..., -999)` 处理空值，-999 表示无数据。

## 关键业务逻辑说明

### 订单状态判断

1. **在贷订单**：`t4.settled_time is null or t4.settled_time > use_create_time`（订单未整笔结清）
2. **逾期账单**：`loan_end_date < use_create_time 且 (settled_time is null or settled_time > loan_end_date)`
3. **续借订单**：`order_asc_rank > 1`（非首笔订单）
4. **提前结清**：`settled_time < use_create_time 且 settled_time < loan_end_date`

### 时间段划分

| 时间段 | 小时范围 |
|--------|----------|
| 早晨 (morning) | 05:00 - 10:59 |
| 中午 (noon) | 11:00 - 13:59 |
| 下午 (afternoon) | 14:00 - 17:59 |
| 傍晚 (evening) | 18:00 - 22:59 |
| 深夜 (night) | 23:00 - 04:59 |

特征名遵循以下命名规则：
```
{模块}_{订单范围}_{统计维度}
```

- **模块**：`multi_loan_order_info` / `multi_loan_in_loan_order`
- **订单范围**：`furthestsingleorder`(最远) / `latest1singleorder`(最近第1笔) / `latest2singleorder`(最近第2笔) / `inloanorders`(在贷) / `multiloanorders`(续借)
- **统计维度**：`completetermcnt`(结清期数) / `overduetermcnt`(逾期期数) / `ratio`(比率) 等
