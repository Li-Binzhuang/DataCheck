# CDC项目：前200条记录输出 - 快速参考

## 一句话说明
为了方便快速查看和测试，每个板块的Notebook现在会额外输出一个包含前200条记录的小文件。

## 输出文件清单

| 板块 | 样本文件名 | 位置 |
|-----|-----------|------|
| 第一板块 | `cdc1_features_consultas_sample200.csv` | `CDC/outputs/` |
| 第二板块 | `cdc2_features_creditos_sample200.csv` | `CDC/outputs/` |
| 第三板块 | `cdc3_features_clave_prevencion_sample200.csv` | `CDC/outputs/` |
| BOSS板块 | `cdcboss_features_sample200.csv` | `CDC/outputs/` |

## 使用步骤

1. 打开任意Notebook（如 `第一板块衍生.ipynb`）
2. 运行所有代码格（Shift + Enter 或点击"Run All"）
3. 查看 `outputs/` 目录，会看到三个文件：
   - 全量特征表（12546条）
   - 平铺明细表（全量）
   - **前200条样本**（新增）

## 开关控制

在每个Notebook的第4个代码格中：

```python
WRITE_SAMPLE_200 = True   # 输出前200条样本
WRITE_SAMPLE_200 = False  # 不输出样本
```

## 为什么需要这个功能？

- ✅ **快速查看**: 200条记录的文件秒开，全量文件可能需要等待
- ✅ **节省时间**: 开发调试时不需要加载12546条记录
- ✅ **便于测试**: 用小文件快速验证数据处理逻辑
- ✅ **数据一致**: 样本数据与全量数据完全一致（只是行数不同）

## 常见问题

**Q: 样本文件包含哪些记录？**  
A: 前200个apply_id的特征记录，顺序与全量文件一致。

**Q: 样本文件的数据格式与全量文件一样吗？**  
A: 完全一样，包括列名、数据类型、小数位数等。

**Q: 如果我只想要全量文件，不想要样本文件？**  
A: 将 `WRITE_SAMPLE_200` 改为 `False` 即可。

**Q: 样本文件会影响全量文件的输出吗？**  
A: 不会，两者是独立输出的。

## 文件大小对比

以第一板块为例：
- 全量特征表: ~50MB（12546条 × 988列）
- 前200条样本: ~0.8MB（200条 × 988列）

样本文件只有全量文件的 **1.6%**，打开速度快很多！
