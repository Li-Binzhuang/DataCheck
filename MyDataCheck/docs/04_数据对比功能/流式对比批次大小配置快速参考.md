# 流式对比批次大小配置 - 快速参考

## 问题
流式对比批次大小配置无法保存和加载，执行时未按配置的批次大小处理。

## 原因
前端JS代码未处理 `global_batch_size` 配置的收集和加载。

## 修复
✅ 已修复 `collectConfig()` 和 `loadConfig()` 函数
✅ 已修复默认批次大小为50
✅ 配置现在可以正确保存和加载

## 使用方法

### 设置批次大小
1. 打开"接口数据对比"页面
2. 找到"流式对比批次大小"输入框
3. 输入期望值（建议50-200）
4. 点击"保存配置"

### 验证配置
- 刷新页面，检查是否正确加载
- 执行任务，观察输出日志中的批次信息

## 配置说明

| 配置项 | 默认值 | 范围 | 说明 |
|--------|--------|------|------|
| 批次大小 | 50 | 1-1000 | 每批处理的行数 |

### 批次大小选择

- **小批次 (50)**: 内存占用低，适合大文件
- **中批次 (100-150)**: 平衡性能和内存
- **大批次 (200+)**: 处理速度快，内存占用高

## 输出示例

## 配置文件

位置: `MyDataCheck/api_comparison/config.json`

```json
{
  "global_config": {
    "default_batch_size": 50
  },
  "scenarios": [
    {
      "batch_size": 50
    }
  ]
}
```

## 注意事项

1. ✅ 保存后立即生效，无需重启
2. ✅ 每个场景使用全局配置的批次大小
3. ⚠️ 批次越大，内存占用越高
4. 💡 建议根据数据量和内存调整
