## 技术原理

**问题**: 逐行写入 → 频繁I/O → 速度慢

**解决**: 批量写入 → 减少I/O → 速度快

```python
# 优化前: 每行一次I/O
for row in data:
    writer.writerow(row)  # 10000次I/O

# 优化后: 批量I/O
buffer = []
for row in data:
    buffer.append(row)
    if len(buffer) >= 5000:
        writer.writerows(buffer)  # 只需2次I/O
        buffer.clear()
```

## 常见问题

**Q: 需要修改现有代码吗？**  
A: 不需要。接口数据对比功能已自动应用优化。

**Q: 会增加内存占用吗？**  
A: 会略微增加（约 batch_size × 行大小），但完全可接受。

**Q: 输出格式会变化吗？**  
A: 不会。输出格式完全一致，只是速度更快。

**Q: 如何选择 batch_size？**  
A: 默认 5000 适合大多数场景。内存受限用 1000，大文件用 10000。

**Q: 可以用于其他场景吗？**  
A: 可以。任何需要写入大量CSV数据的场景都可以使用。
