# CSV数据对比 - 忽略0和NaN功能说明

## 功能概述

在CSV数据对比模块中，新增了"忽略0和NaN"配置项，允许用户在对比时将0和NaN/空值视为一致。

## 使用场景

### 场景示例

在某些业务场景中，0值和空值（NaN、null、空字符串）在业务逻辑上是等价的：

**模型特征表的值：**
```
feature_a: 0
feature_b: 0.0
feature_c: (空)
```

**接口/灰度/从库特征表的值：**
```
feature_a: NaN
feature_b: null
feature_c: 0
```

启用"忽略0和NaN"后，上述三组值都会被视为一致，不会被标记为差异。

## 配置方法

### Web界面配置

1. 在"csv数据对比"页面的"对比配置"部分
2. 找到"忽略默认填充值"配置项
3. 在其下方可以看到"忽略0和NaN"配置项
4. 勾选该选项即可启用

### 配置说明

- **默认状态**：不勾选（不忽略）
- **勾选后**：0 和 NaN/空值会被视为一致
- **支持的空值类型**：
  - Python的None
  - 字符串 "null", "none", ""（空字符串）
  - 字符串 "nan", "na", "n/a"
  - 数值0（包括0、0.0等）
  - 浮点数NaN

## 工作原理

### 对比逻辑

当启用"忽略0和NaN"选项时，对比器会执行以下检查：

1. **检查第一个文件的值**：
   - 是否为空值（null、none、空字符串、nan、na、n/a）
   - 是否为数值0
   - 是否为NaN（通过 `value != value` 检测）

2. **检查第二个文件的值**：
   - 执行相同的检查

3. **判断一致性**：
   - 如果两个值都是"0或NaN/空值"，则认为一致
   - 否则按正常逻辑对比

### 示例对比结果

| 模型特征值 | 接口特征值 | 忽略0和NaN=否 | 忽略0和NaN=是 |
|-----------|-----------|--------------|--------------|
| 0         | NaN       | ❌ 差异      | ✅ 一致      |
| 0.0       | null      | ❌ 差异      | ✅ 一致      |
| (空)      | 0         | ❌ 差异      | ✅ 一致      |
| NaN       | null      | ✅ 一致      | ✅ 一致      |
| 0         | 0         | ✅ 一致      | ✅ 一致      |
| 1         | 0         | ❌ 差异      | ❌ 差异      |
| 1         | NaN       | ❌ 差异      | ❌ 差异      |

## 与"忽略默认填充值"的区别

### 忽略默认填充值

- **目的**：处理业务中的默认填充值
- **规则**：-999 和 null 视为一致
- **适用场景**：业务系统使用-999作为默认填充值

### 忽略0和NaN

- **目的**：处理0值和空值的等价性
- **规则**：0 和 NaN/空值 视为一致
- **适用场景**：业务逻辑中0和空值等价

### 可以同时启用

这两个选项可以同时启用，它们会按顺序检查：

1. 先检查"忽略0和NaN"
2. 再检查"忽略默认填充值"
3. 最后执行正常的值对比

## 注意事项

1. **性能影响**：启用此选项会增加少量的值检查逻辑，但对整体性能影响很小
2. **业务语义**：请确认在您的业务场景中，0和空值确实是等价的
3. **数据质量**：如果数据中大量出现0和空值的混用，建议检查数据质量
4. **与其他选项配合**：可以与"忽略默认填充值"、"转换特征值为数值"等选项配合使用

## 常见问题

### Q1：为什么需要这个功能？

A：在某些业务场景中，特征值为0和特征值为空（未计算）在业务逻辑上是等价的。例如：
- 用户没有某项行为时，特征值可能是0或null
- 某些计算结果为0时，可能被存储为null以节省空间

### Q2：这个功能会影响其他数值的对比吗？

A：不会。只有当两个值都是"0或NaN/空值"时才会被视为一致。其他数值的对比不受影响。

### Q3：如果只有一个值是0，另一个是非0数值，会怎样？

A：会被正常标记为差异。例如：
- 模型值=0，接口值=1 → 差异
- 模型值=0，接口值=0 → 一致
- 模型值=0，接口值=NaN → 一致（启用选项时）

### Q4：这个功能支持哪些数据类型？

A：支持所有可以转换为数值的类型，包括：
- 整数：0
- 浮点数：0.0
- 字符串："0", "0.0"
- 空值：null, none, "", nan, na, n/a
