# 接口请求失败记录输出 - 快速参考

## 功能说明
在接口数据对比时，自动将请求失败的记录输出到单独的CSV文件。

## 输出文件
```
{输出前缀}_{时间戳}_api_data_error_records.csv
```

## 文件字段
| 字段 | 说明 |
|------|------|
| applyid | 申请ID |
| baseTime | 基准时间 |
| 错误code | 错误信息 |

## 配置方法

### 在 config.json 中添加：
```json
{
  "output_config": {
    "output_intermediate_files": true,
    "output_error_records": true
  }
}
```

## 使用示例

### 1. 启用错误记录输出（默认）
```json
{
  "output_config": {
    "output_intermediate_files": true,
    "output_error_records": true
  }
}
```

### 2. 关闭错误记录输出
```json
{
  "output_config": {
    "output_intermediate_files": true,
    "output_error_records": false
  }
}
```

## 输出示例

### 错误记录文件内容：
```csv
applyid,baseTime,错误code
123,2024-01-15 10:30:00,接口请求失败: timeout
456,2024-01-15 11:00:00,参数 'baseTime' 为空: 行 5, 列 1
789,2024-01-15 12:00:00,JSON解析失败
```

## 常见错误类型

1. **接口请求失败**: 网络超时、连接失败
2. **参数为空**: 必填参数在CSV中为空
3. **列索引无效**: 参数配置的列索引超出范围
4. **JSON解析失败**: 接口返回数据格式错误
5. **处理异常**: 其他处理过程中的异常

## 注意事项

✅ **必须启用中间文件输出** (`output_intermediate_files=true`)  
✅ **默认自动输出错误记录**  
✅ **不影响正常请求的处理速度**  
✅ **只在有错误时才生成文件**

## 快速排查

### 1. 查看错误记录文件
```bash
# 查找最新的错误记录文件
ls -lt outputdata/*_error_records.csv | head -1
```

### 2. 统计错误类型
```bash
# 统计各类错误的数量
cut -d',' -f3 outputdata/*_error_records.csv | sort | uniq -c
```

### 3. 提取失败的applyId
```bash
# 提取所有失败的applyId
cut -d',' -f1 outputdata/*_error_records.csv | tail -n +2
```
