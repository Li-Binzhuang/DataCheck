# 任务恢复功能说明

## 功能概述

任务恢复功能允许用户在刷新页面或重新打开浏览器后，继续查看正在执行的任务进度和历史日志。

## 主要特性

### 1. 自动检测运行中的任务
- 页面加载时自动检测是否有正在执行的任务
- 如果有，弹出提示询问是否恢复查看

### 2. 任务状态持久化
- 任务状态保存到文件系统
- 包括任务名称、状态、进度、当前步骤等信息
- 服务器重启后仍可查询历史任务

### 3. 日志持久化
- 所有输出日志保存到文件
- 支持查看历史日志
- 支持增量加载新日志

### 4. 实时进度更新
- 恢复任务后，自动轮询最新进度
- 每2秒更新一次状态和日志
- 任务完成后自动停止轮询

## 使用场景

### 场景1：锁屏后恢复
```
1. 用户启动接口数据对比任务
2. 任务执行中，用户锁屏
3. 一段时间后，Web界面显示 "network error"
4. 用户刷新页面
5. 系统自动检测到运行中的任务
6. 弹出提示："检测到有正在执行的任务，是否恢复查看？"
7. 用户点击"确定"
8. 显示历史日志 + 实时更新新日志
```

### 场景2：关闭浏览器后恢复
```
1. 用户启动任务
2. 关闭浏览器
3. 任务在服务器端继续执行
4. 用户重新打开浏览器访问页面
5. 系统提示恢复任务
6. 用户可以继续查看进度
```

### 场景3：查看任务历史
```
1. 用户可以查看所有历史任务
2. 包括已完成、失败、停止的任务
3. 可以查看任何任务的完整日志
```

## 技术实现

### 后端实现

#### 1. 任务管理器（TaskManager）

**文件位置**: `MyDataCheck/common/task_manager.py`

**主要功能**:
- 创建任务：`TaskManager.create_task(task_name, task_type)`
- 更新任务：`TaskManager.update_task(task_id, status=..., progress=...)`
- 添加日志：`TaskManager.add_log(task_id, message, level)`
- 查询任务：`TaskManager.get_task(task_id)`
- 获取日志：`TaskManager.get_logs(task_id, last_n=100)`

**数据存储**:
- 任务信息：`MyDataCheck/logs/tasks/{task_id}_info.json`
- 任务日志：`MyDataCheck/logs/tasks/{task_id}_logs.jsonl`

**任务状态**:
- `pending`: 等待执行
- `running`: 执行中
- `completed`: 已完成
- `failed`: 失败
- `stopped`: 已停止

#### 2. API接口

**获取所有任务**:
```
GET /api/tasks
GET /api/tasks?status=running  # 过滤运行中的任务
```

**获取指定任务**:
```
GET /api/tasks/{task_id}
```

**获取任务日志**:
```
GET /api/tasks/{task_id}/logs
GET /api/tasks/{task_id}/logs?last_n=100  # 最后100条
GET /api/tasks/{task_id}/logs?from_file=true  # 从文件读取
```

## 使用示例

### 示例1：正常流程

```javascript

executeConfig();

// 4. 用户锁屏，前端显示 "network error"

// 6. 检测到运行中的任务

// 返回: [{task_id: "xxx", task_name: "接口数据对比 (3个场景)", status: "running", ...}]

// 9. 加载历史日志

// 显示所有历史日志

// 10. 开始轮询新日志

    fetch('/api/tasks/xxx')  // 获取最新状态
    fetch('/api/tasks/xxx/logs?from_file=true')  // 获取新日志
}, 2000)

// 11. 任务完成

// 前端检测到完成，停止轮询
```

## 注意事项

### 1. 性能考虑

- **内存限制**: 每个任务最多保存1000条日志在内存中
- **文件大小**: 日志文件可能很大，建议定期清理
- **轮询频率**: 默认2秒，可根据需要调整

### 2. 并发任务

- 支持多个任务同时执行
- 每个任务独立管理状态和日志
- 恢复时只显示最新的一个运行中任务

### 3. 任务清理

- 任务完成后不会自动删除
- 需要手动清理或实现自动清理机制
- 建议保留最近N天的任务记录

### 4. 错误处理

- 如果任务文件损坏，会返回空数据
- 如果任务不存在，会返回错误信息
- 网络错误会在控制台显示，不影响页面
