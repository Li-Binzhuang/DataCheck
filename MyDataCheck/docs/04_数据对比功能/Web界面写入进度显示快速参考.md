# Web界面写入进度显示 - 快速参考

## 问题
Web界面执行输出区域看不到写入进度

## 原因
使用了 `end='\r'`，Web界面的 `OutputCapture` 只处理 `\n`

## 解决方案

### 1. 优化 OutputCapture（已修改）
**文件**: `web/utils.py`
- ✅ 同时处理 `\n` 和 `\r`
- ✅ 过滤空行

### 2. 移除 end='\r'（已修改）
**修改的文件**:
- ✅ `api_comparison/job/fetch_api_data.py` - 写入进度
- ✅ `api_comparison/job/streaming_comparator.py` - 批次进度
- ✅ `common/csv_tool.py` - CSV写入进度

## 效果

### 修改前
```
接口返回数据总数: 17900
接口数据文件写入完成
```
❌ 看不到写入进度

### 修改后
```
接口返回数据总数: 17900
  写入进度: 20/17900 行 (0.1%)
  写入进度: 40/17900 行 (0.2%)
  写入进度: 60/17900 行 (0.3%)
  ...
  写入进度: 17900/17900 行 (100.0%)
接口数据文件写入完成
```
✅ 可以看到写入进度（每20行更新一次）
