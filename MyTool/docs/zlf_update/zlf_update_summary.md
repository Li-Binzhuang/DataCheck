## 修改规则

### 保留的填充方式
- `fillna("")` - 字符串空值填充保持不变
- `fillna('')` - 字符串空值填充保持不变

### 修改的填充方式
- `fillna(0)` → `fillna(-999)` - 整数特征空值填充
- `fillna(0.0)` → `fillna(-999.0)` - 浮点数特征空值填充

### 标识方式
在每个修改的行前添加注释:
```python
# zlf update: 特征值为空时填充-999
```

## 修改原因
1. **统一空值标识**: 使用 `-999` 作为特征空值的统一标识,便于下游模型识别缺失值
2. **区分真实零值**: 避免将真实的0值与缺失值混淆
3. **提高模型效果**: 明确的缺失值标识有助于模型更好地处理缺失数据

## 影响范围
- 所有特征计算中的空值填充
- 包括计数(cnt)、占比(ratio)、均值(mean)、标准差(std)等统计特征
- 不影响字符串类型字段的填充
- 不影响日期时间字段的处理

## 修改二：浮点数精度控制

### 问题发现
在检查四个板块的CSV输出时，发现BOSS板块缺少浮点数精度控制，可能导致特征值被截断。

### 修改的文件
`BOSS板块衍生.ipynb` - CSV输出部分

### 修改内容
在CSV输出前添加 `.round(6)` 处理，统一保留6位小数：

```python
# zlf update: 对数值特征列保留6位小数
_features_to_write = features_df.copy()
_round_cols = [c for c in _features_to_write.columns if c not in {"apply_id", "request_time"}]
_features_to_write[_round_cols] = _features_to_write[_round_cols].apply(pd.to_numeric, errors="coerce").round(6)
_features_to_write.to_csv(csv_path, index=False, encoding="utf-8-sig")
```

### 处理规则
- 数值特征：保留6位小数
- apply_id：不处理（保持原样）
- request_time：不处理（保持原样）
- 全量数据和sample200：都使用相同处理
