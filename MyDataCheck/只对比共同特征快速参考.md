# 只对比共同特征 - 快速参考

## 🎯 功能说明

只对比两个文件中都存在的特征列，自动忽略单边存在的列。

## 📍 位置

Web界面 → 数据对比 → "只对比共同特征"选项（位于"启用特征名称映射"下方）

## ⚙️ 配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 只对比共同特征 | 是否只对比共同列 | ❌ 不勾选 |

## 🔄 执行流程

```
1. 提取特征列（按特征起始列索引）
   ↓
2. 应用特征名称映射（如果启用）
   ↓
3. 筛选共同特征（如果启用"只对比共同特征"）
   ↓
4. 执行数据对比
```

## 📝 使用示例

### 场景1：基本使用

**file1:** `id`, `age`, `income`, `pt`, `score`
**file2:** `id`, `age`, `income`, `score`

**不勾选"只对比共同特征"：**
- 对比：`age`, `income`, `pt`, `score`
- 结果：报告`pt`列在file2中缺失 ❌

**勾选"只对比共同特征"：**
- 对比：`age`, `income`, `score`
- 结果：只对比共同列，不报告缺失 ✅

### 场景2：结合特征名称映射

**配置：**
```
☑ 启用特征名称映射
  映射文件: file1
  前缀: local_all_sms_
  后缀: _v3
☑ 只对比共同特征
```

**file1原始列：** `age`, `income`, `pt`, `score`
**file1映射后：** `local_all_sms_age_v3`, `local_all_sms_income_v3`, `local_all_sms_pt_v3`, `local_all_sms_score_v3`
**file2列：** `local_all_sms_age_v3`, `local_all_sms_income_v3`, `local_all_sms_score_v3`

**结果：**
- 共同列：3个（age_v3, income_v3, score_v3）
- 排除列：`local_all_sms_pt_v3`
- 只对比3个共同列 ✅

## ⚡ 快速上手

1. 上传两个CSV文件
2. 配置主键列和特征起始列
3. （可选）启用特征名称映射
4. 勾选"只对比共同特征"
5. 点击"执行对比"

## ⚠️ 注意事项

- ✅ 默认不启用，保持现有逻辑
- ✅ 主键列不受影响
- ✅ 特征起始列索引仍然有效
- ✅ 配置可保存
- ❌ 如果没有共同列会报错

## 💡 推荐用法

**最佳实践：**
```
1. 先启用"特征名称映射"统一列名
2. 再启用"只对比共同特征"筛选列
3. 执行对比
```

这样可以确保列名一致后再筛选共同列，避免因列名不一致导致的误判。
