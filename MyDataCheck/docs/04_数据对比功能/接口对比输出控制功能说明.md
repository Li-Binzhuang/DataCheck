# 接口数据对比 - 输出文件控制功能

## 功能说明

在接口数据对比功能中新增了输出文件控制选项，允许用户选择是否输出以下两个文件：

1. **全量数据合并文件** (`_全量数据合并.csv`)
2. **接口数据文件** (`_api_data.csv`)

## 使用方法

### 前端界面

在"接口数据对比"页面的"全局默认配置"部分，新增了"📄 输出文件控制"选项：

```
📄 输出文件控制
☑ 输出全量数据合并文件 (_全量数据合并.csv)
☑ 输出接口数据文件 (_api_data.csv)
```

- **默认状态**: 两个选项都勾选（输出所有文件）
- **可选配置**: 可以取消勾选任一选项，不输出对应的文件

### 配置文件

在 `api_comparison/config.json` 中，新增了 `output_config` 配置项：

```json
{
  "scenarios": [...],
  "global_config": {
    "default_thread_count": 150,
    "default_timeout": 60,
    "default_convert_feature_to_number": true
  },
  "output_config": {
    "output_merged_data": true,
    "output_api_data": true
  }
}
```

**配置说明**:
- `output_merged_data`: 是否输出全量数据合并文件（默认 `true`）
- `output_api_data`: 是否输出接口数据文件（默认 `true`）

## 输出文件说明

### 1. 全量数据合并文件 (`_全量数据合并.csv`)

**文件名格式**: `{前缀}_{时间戳}_compare_全量数据合并.csv`

**内容**: 
- 包含原始CSV文件和接口返回数据的所有字段
- 字段名添加后缀区分来源（`_原始` 和 `_接口`）
- 用于全面对比两个数据源的所有字段

**适用场景**:
- 需要查看所有字段的详细对比
- 需要导出完整数据进行进一步分析
- 数据量较大时可能占用较多磁盘空间

### 2. 接口数据文件 (`_api_data.csv`)

**文件名格式**: `{前缀}_{时间戳}_api_data.csv`

**内容**:
- 接口返回的原始数据
- 包含所有接口返回的字段
- 用于查看接口的完整响应

**适用场景**:
- 需要保留接口原始数据
- 需要单独分析接口返回结果
- 用于后续的数据处理或分析

### 3. 对比报告文件（始终输出）

**文件名格式**: 
- `{前缀}_{时间戳}_compare_analysis_report.csv` - 分析报告
- `{前缀}_{时间戳}_compare_feature_stats.csv` - 特征统计

**说明**: 
- ✅ 这两个文件**始终会输出**，不受控制选项影响
- ✅ 即使禁用上述两个可选文件，对比功能仍能正常工作
- ✅ 接口数据会临时存储用于对比，对比完成后自动删除
- ✅ 对比报告包含完整的对比结果和统计信息

> 💡 **重要提示**: 无论输出控制开关如何设置，对比功能都能正常执行，并且会正常输出这两个核心报告文件。这确保了即使选择最小化输出，也能获得完整的对比分析结果。

## 使用场景

### 场景 1: 只需要对比报告

如果只关心对比结果，不需要保留原始数据和全量合并数据：

```
☐ 输出全量数据合并文件
☐ 输出接口数据文件
```

**输出文件**:
- ✅ `_compare_analysis_report.csv`
- ✅ `_compare_feature_stats.csv`
- ❌ `_全量数据合并.csv`
- ❌ `_api_data.csv`

**优势**: 节省磁盘空间，输出更简洁

### 场景 2: 需要接口数据但不需要全量合并

如果需要保留接口原始数据，但不需要全量合并：

```
☐ 输出全量数据合并文件
☑ 输出接口数据文件
```

**输出文件**:
- ✅ `_compare_analysis_report.csv`
- ✅ `_compare_feature_stats.csv`
- ❌ `_全量数据合并.csv`
- ✅ `_api_data.csv`

### 场景 3: 输出所有文件（默认）

如果需要完整的数据和对比结果：

```
☑ 输出全量数据合并文件
☑ 输出接口数据文件
```

**输出文件**:
- ✅ `_compare_analysis_report.csv`
- ✅ `_compare_feature_stats.csv`
- ✅ `_全量数据合并.csv`
- ✅ `_api_data.csv`

## 技术实现

### 后端实现

1. **配置读取**: 从 `config.json` 的 `output_config` 中读取配置
2. **条件输出**: 
   - 如果 `output_api_data=false`，使用临时文件存储接口数据，对比完成后删除
   - 如果 `output_merged_data=false`，跳过全量数据合并文件的写入
3. **向后兼容**: 如果配置文件中没有 `output_config`，默认输出所有文件

### 前端实现

1. **UI 控件**: 在全局配置区域添加两个复选框
2. **配置保存**: 保存配置时包含 `output_config` 对象
3. **配置加载**: 加载配置时恢复复选框状态
4. **默认值**: 默认勾选（输出所有文件）

## 注意事项

1. **对比报告始终输出**: 分析报告和特征统计文件始终会输出，不受控制
2. **临时文件处理**: 如果禁用接口数据文件输出，系统会使用临时文件进行对比，对比完成后自动删除
3. **配置持久化**: 输出控制选项会保存到配置文件中，下次加载时自动恢复
4. **向后兼容**: 旧的配置文件（没有 `output_config`）会使用默认值（输出所有文件）

## 常见问题 (FAQ)

### Q1: 如果两个可选文件都不输出，对比功能还能正常工作吗？

**A**: ✅ **可以正常工作**。即使禁用两个可选输出文件：
- 对比功能会正常执行
- 接口数据会临时存储用于对比
- 对比完成后临时文件会自动删除
- 两个核心报告文件（`_analysis_report.csv` 和 `_feature_stats.csv`）会正常输出
- 你仍然能获得完整的对比分析结果

### Q2: 禁用输出后，还能看到对比结果吗？

**A**: ✅ **可以**。对比报告文件（`_analysis_report.csv` 和 `_feature_stats.csv`）始终会输出，包含：
- 详细的特征值对比结果
- 每行数据的对比详情
- 特征级别的统计信息
- 匹配率和异常占比

### Q3: 什么时候应该禁用这两个文件？

**A**: 建议在以下场景禁用：
- 批量对比多个文件，节省磁盘空间
- 只关心对比结果，不需要保留原始数据
- 处理大文件时，减少磁盘I/O操作
- 自动化测试场景，只需要对比报告

### Q4: 禁用输出会影响性能吗？

**A**: ✅ **会提升性能**：
- 减少磁盘写入操作
- 节省磁盘空间
- 加快执行速度
- 特别是处理大文件时效果明显
