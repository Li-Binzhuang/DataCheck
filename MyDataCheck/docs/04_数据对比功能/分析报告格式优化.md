# 分析报告格式优化

## 问题描述

在流式对比模式（`output_intermediate_files: false`）下，生成的 `_analysis_report.csv` 文件格式不够详细，只显示：
- 行号
- cust_no
- use_create_time
- use_credit_apply_id
- 是否有差异
- 差异特征数量
- 差异特征列表

用户希望看到更详细的对比信息，包括每个特征的 CSV 值和 API 值。

## 修改方案

### 修改前的格式

```csv
行号,cust_no,use_create_time,use_credit_apply_id,是否有差异,差异特征数量,差异特征列表
2,123456,2025-01-27T10:00:00.000,1073458287442403329,是,3,feature1, feature2, feature3
```

**问题**：
- 只能看到哪些特征有差异
- 看不到具体的 CSV 值和 API 值
- 需要手动查找原始数据才能对比

### 修改后的格式

```csv
特征名,applyId,request_time,CSV值,API值
feature1,1073458287442403329,2025-01-27T10:00:00.000,0.5,0.6
feature2,1073458287442403329,2025-01-27T10:00:00.000,1.2,1.3
feature3,1073458287442403329,2025-01-27T10:00:00.000,abc,def
```

**优点**：
- ✅ 每行显示一个特征的详细对比
- ✅ 直接看到 CSV 值和 API 值
- ✅ 方便在 Excel 中筛选和分析
- ✅ applyId 优先从接口入参获取，确保有值
- ✅ 去掉冗余的 baseTime 列

## 列说明

| 列名 | 说明 | 示例 |
|------|------|------|
| 特征名 | 特征字段名称 | `cdc1_m3_max_dpd_amt` |
| applyId | 申请ID（优先从接口入参获取，其次从CSV列获取） | `1073458287442403329` |
| request_time | 请求时间（处理后的时间） | `2025-01-27T10:00:00.000` |
| CSV值 | 原始CSV文件中的值 | `0.5` |
| API值 | 接口返回的值 | `0.6` |

## 使用场景

### 1. 快速定位差异

在 Excel 中打开文件，可以：
- 按特征名排序，查看某个特征的所有差异
- 按 applyId 筛选，查看某个申请的所有差异特征
- 直接对比 CSV 值和 API 值

### 2. 批量分析

- 行：特征名
- 值：计数（查看每个特征的差异数量）
- 筛选：applyId（查看特定申请的差异）

## 技术实现

### 修改点

修改 `streaming_comparator.py` 中的 `streaming_compare` 方法：

**1. 修改表头**
```python
analysis_headers = [
    "特征名", "applyId", "request_time", "CSV值", "API值"
]
```

**2. 优化 applyId 获取逻辑**
```python
# 优先从接口入参中获取 applyId
apply_id = original_values.get("applyId", original_values.get("apply_id", ""))

# 如果接口入参中没有，尝试从 CSV 列中查找
if not apply_id:
    apply_id_fields = ['apply_id', 'applyId', 'use_credit_id', ...]
    for field_name in apply_id_fields:
        for i, header in enumerate(headers):
            if header.lower() == field_name.lower() and i < len(row):
                apply_id = row[i].strip()
                break
        if apply_id:
            break

# 如果还是没有，使用 custNo
if not apply_id:
    apply_id = cust_no
```

**3. 修改写入逻辑（去掉 baseTime）**
```python
writer.writerow([
    feature_name,
    apply_id,           # applyId
    use_create_time,    # request_time
    csv_value,
    api_value
])
```

### 数据流程

1. **读取 CSV**：读取原始数据文件
2. **请求接口**：并发请求接口获取 API 数据
3. **对比特征**：逐个对比每个特征的值
4. **收集差异**：只保存不匹配的特征及其值
5. **写入报告**：每个差异特征写入一行

## 注意事项

### 1. 文件大小

由于每个差异特征都会写入一行，如果差异很多，文件会比较大。

**示例**：
- 10000 行数据
- 每行平均 5 个差异特征
- 总共 50000 行输出

**建议**：
- 使用 Excel 的筛选功能
- 或使用 Python/Pandas 进行分析

### 2. 内存占用

流式对比模式已经优化了内存占用：
- 边请求边对比边写入
- 不在内存中累积所有数据
- 及时释放已处理的数据

### 3. 性能影响

写入详细格式不会显著影响性能：
- 写入操作是流式的
- 每批处理完立即写入
- 不需要额外的内存

## 与其他模式的对比

### 完整输出模式（`output_intermediate_files: true`）

生成的 `_全量数据合并.csv` 包含：
- 所有原始列
- 所有接口返回的列
- 对比结果列

**优点**：包含完整数据  
**缺点**：文件很大，不易分析

### 流式对比模式（`output_intermediate_files: false`）

生成的 `_analysis_report.csv` 包含：
- 只有差异的特征
- 详细的对比信息
- 紧凑的格式

**优点**：文件小，易于分析  
**缺点**：不包含匹配的特征
