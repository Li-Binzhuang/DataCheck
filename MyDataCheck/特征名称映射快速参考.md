# 特征名称映射功能 - 快速参考

## 🎯 功能说明

在数据对比前，为指定文件的特征列名添加前缀和/或后缀，实现列名统一后再对比。

## 📍 位置

Web界面 → 数据对比 → "启用特征名称映射"选项

## ⚙️ 配置参数

| 参数 | 说明 | 示例 |
|------|------|------|
| 启用特征名称映射 | 是否启用映射功能 | ✅ 勾选 |
| 映射文件 | 选择要映射的文件 | `模型特征表 (file1)` 或 `接口/灰度/从库特征表 (file2)` |
| 特征列名前缀 | 添加到列名前面的字符串 | `model_` |
| 特征列名后缀 | 添加到列名后面的字符串 | `_v1` |

## 💡 使用场景

### 场景1：列名有前缀差异
```
file1: age, income, score
file2: model_age, model_income, model_score

解决方案：
- 映射文件：file1
- 前缀：model_
- 后缀：（留空）

结果：age → model_age
```

### 场景2：列名有后缀差异
```
file1: age_v1, income_v1, score_v1
file2: age, income, score

解决方案：
- 映射文件：file2
- 前缀：（留空）
- 后缀：_v1

结果：age → age_v1
```

### 场景3：列名有前缀和后缀差异
```
file1: age, income, score
file2: api_age_new, api_income_new, api_score_new

解决方案：
- 映射文件：file1
- 前缀：api_
- 后缀：_new

结果：age → api_age_new
```

## 📝 配置示例

```json
{
  "enable_column_mapping": true,
  "mapping_file": "file1",
  "mapping_prefix": "model_",
  "mapping_suffix": "_v1"
}
```

## ⚠️ 注意事项

1. 映射只影响特征列，不影响主键列
2. 映射不修改原始文件，只在内存中进行
3. 默认不启用，需要手动勾选
4. 配置可保存到 `data_comparison/config.json`

## 🚀 快速开始

1. 上传两个CSV文件
2. 勾选"启用特征名称映射"
3. 选择映射文件（file1或file2）
4. 输入前缀和/或后缀
5. 点击"执行对比"

## 📞 常见问题

**Q: 可以为两个文件都添加映射吗？**
A: 目前只支持为一个文件添加映射。

**Q: 映射会影响主键列吗？**
A: 不会，映射只影响特征列。

**Q: 如何取消映射？**
A: 取消勾选"启用特征名称映射"即可。

**Q: 映射后找不到匹配的列怎么办？**
A: 请检查映射后的列名是否与另一个文件的列名完全一致。
