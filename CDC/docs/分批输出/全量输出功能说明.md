# 全量输出功能说明

## 功能概述

在分批输出的基础上，新增了全量CSV文件输出功能，由开关 `WRITE_FULL_CSV` 控制。

---

## 开关说明

### WRITE_FULL_CSV
- **类型**: Boolean
- **默认值**: `True`
- **位置**: 每个板块衍生脚本的输出配置部分
- **作用**: 控制是否输出包含所有数据的全量CSV文件

---

## 输出文件

### 分批文件（始终输出）
每个板块会输出多个分批文件，每个文件500条数据：

| 板块 | 文件名格式 | 示例 |
|------|-----------|------|
| 第一板块 | `cdc1_features_batch{批次号:03d}_{起始行}-{结束行}.csv` | `cdc1_features_batch001_1-500.csv` |
| 第二板块 | `cdc2_features_batch{批次号:03d}_{起始行}-{结束行}.csv` | `cdc2_features_batch001_1-500.csv` |
| 第三板块 | `cdc3_features_batch{批次号:03d}_{起始行}-{结束行}.csv` | `cdc3_features_batch001_1-500.csv` |
| BOSS板块 | `cdcboss_features_batch{批次号:03d}_{起始行}-{结束行}.csv` | `cdcboss_features_batch001_1-500.csv` |

### 全量文件（由WRITE_FULL_CSV控制）
每个板块会输出一个包含所有数据的全量文件：

| 板块 | 文件名 | 说明 |
|------|--------|------|
| 第一板块 | `cdc1_features_full_data.csv` | 包含所有apply_id的特征 |
| 第二板块 | `cdc2_features_full_data.csv` | 包含所有apply_id的特征 |
| 第三板块 | `cdc3_features_full_data.csv` | 包含所有apply_id的特征 |
| BOSS板块 | `cdcboss_features_full_data.csv` | 包含所有apply_id的特征 |

---

## 使用方法

### 1. 启用全量输出（默认）
```python
WRITE_FULL_CSV = True  # 输出全量文件
```

**输出**：
- 分批文件：`cdc1_features_batch001_1-500.csv`, `cdc1_features_batch002_501-1000.csv`, ...
- 全量文件：`cdc1_features_full_data.csv`

### 2. 禁用全量输出
```python
WRITE_FULL_CSV = False  # 不输出全量文件
```

**输出**：
- 分批文件：`cdc1_features_batch001_1-500.csv`, `cdc1_features_batch002_501-1000.csv`, ...
- 全量文件：无

## 文件内容

### 分批文件和全量文件的内容完全相同
- 都包含相同的列（apply_id, request_time, 所有特征列）
- 都经过相同的处理（round(6), fillna(-999.0)）
- 唯一区别是行数

## 使用场景

### 使用分批文件
- **快速查看**：只需要查看部分数据
- **分批处理**：需要分批读取和处理数据
- **内存限制**：一次性加载全量数据会超出内存

### 使用全量文件
- **完整分析**：需要对所有数据进行统计分析
- **模型训练**：需要完整的训练数据集
- **数据合并**：需要与其他数据源合并
- **便捷性**：不想处理多个分批文件

## 注意事项

### 1. 磁盘空间
全量文件可能较大，确保有足够的磁盘空间：
- 第一板块：约12546行，文件大小约几十MB
- 根据特征数量和数据量，文件大小会有所不同

### 2. 内存使用
输出全量文件时，所有数据会在内存中处理，确保有足够的内存。

### 3. 输出顺序
1. 先输出所有分批文件
2. 再输出全量文件（如果启用）

### 4. 文件覆盖
每次运行都会覆盖之前的输出文件，注意备份重要数据。

## 常见问题

### Q1: 如何只输出全量文件，不输出分批文件？
A: 目前不支持。分批文件始终会输出，全量文件是可选的。

### Q2: 全量文件和分批文件的数据一致吗？
A: 完全一致。全量文件就是所有分批文件的合并。

### Q3: 可以修改全量文件的文件名吗？
A: 可以，修改脚本中的 `full_filename` 变量即可。

### Q4: 如何验证全量文件和分批文件的一致性？
```python
import pandas as pd
from pathlib import Path

# 读取全量文件
full_df = pd.read_csv("outputs/cdc1_features_full_data.csv")

# 读取所有分批文件并合并
batch_files = sorted(Path("outputs").glob("cdc1_features_batch*.csv"))
batch_dfs = [pd.read_csv(f) for f in batch_files]
merged_df = pd.concat(batch_dfs, ignore_index=True)

# 比较
print("全量文件行数:", len(full_df))
print("分批文件合并后行数:", len(merged_df))
print("是否一致:", full_df.equals(merged_df))
```
