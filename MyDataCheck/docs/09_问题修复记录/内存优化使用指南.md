| 场景 | 推荐行数 | 推荐列数 | 预计内存 |
|------|---------|---------|---------|
| 小规模 | < 1万行 | < 100列 | < 50 MB |

### 最佳实践

1. **分批处理超大文件**
   - 如果文件超过100万行，建议分批处理
   - 可以按时间范围或ID范围分割

2. **合理配置服务器**
   - 推荐内存：至少 2 GB
   - 推荐CPU：2核心以上

3. **监控内存使用**
   - 使用系统监控工具观察内存占用
   - 如发现异常，请及时反馈

## 技术细节（开发者）

### 优化原理

1. **流式写入**
   - 不再在内存中累积所有数据
   - 边处理边写入文件
   - 处理完立即释放内存

2. **增量处理**
   - 逐行处理数据
   - 避免创建大量临时对象
   - 及时回收不再使用的对象

3. **缓冲区管理**
   - 每1000行自动刷新缓冲区
   - 平衡内存占用和写入性能

### 代码示例

**使用流式写入器**:
```python
from csv_tool import CSVStreamWriter

# 推荐：使用流式写入
with CSVStreamWriter("output.csv", headers) as writer:
    for row in process_data():
        writer.write_row(row)
        # 数据立即写入，不占用内存

# 不推荐：传统方式（会占用大量内存）
all_rows = []
for row in process_data():
    all_rows.append(row)  # 累积在内存中
write_csv_file("output.csv", headers, all_rows)
```
