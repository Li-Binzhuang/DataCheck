## 优化2：自动清理日志

### 问题
- 每次执行都保存完整日志，占用大量磁盘空间
- 历史日志冗余，实际只需要保留摘要
- 日志文件越来越多，影响性能

### 解决方案

#### 自动清理机制

**文件**: `MyDataCheck/common/task_manager.py`

**新增方法**:

1. **cleanup_completed_task_logs()** - 清理已完成任务的日志
```python
@classmethod
def cleanup_completed_task_logs(cls, task_id: str, keep_summary: bool = True) -> bool:
    """
    清理已完成任务的日志（保留任务信息）
    
    Args:
        task_id: 任务ID
        keep_summary: 是否保留摘要信息（默认True）
    
    Returns:
        bool: 是否清理成功
    """
    # 检查任务是否已完成
    if task['status'] not in ['completed', 'failed', 'stopped']:
        return False
    
    # 清理内存中的日志
    if keep_summary:
        # 保留最后10条日志作为摘要
        logs = list(instance._logs[task_id])
        summary_logs = logs[-10:] if len(logs) > 10 else logs
        instance._logs[task_id] = deque(summary_logs, maxlen=cls.MAX_MEMORY_LOGS)
    else:
        # 完全清空
        instance._logs[task_id].clear()
    
    # 删除日志文件
    instance._delete_log_file(task_id)
    
    # 如果保留摘要，重新写入摘要日志
    if keep_summary:
        for log in instance._logs[task_id]:
            instance._append_log_to_file(task_id, log)
```

2. **cleanup_old_tasks()** - 清理N天前的旧任务
```python
@classmethod
def cleanup_old_tasks(cls, days: int = 7, status_filter: List[str] = None) -> int:
    """
    清理N天前的旧任务
    
    Args:
        days: 清理多少天前的任务
        status_filter: 只清理指定状态的任务（None表示全部）
    
    Returns:
        int: 清理的任务数量
    """
    from datetime import datetime, timedelta
    cutoff_time = datetime.now() - timedelta(days=days)
    
    tasks_to_delete = []
    for task_id, task in instance._tasks.items():
        created_at = datetime.fromisoformat(task['created_at'])
        if created_at < cutoff_time:
            if status_filter is None or task['status'] in status_filter:
                tasks_to_delete.append(task_id)
    
    # 删除任务
    deleted_count = 0
    for task_id in tasks_to_delete:
        if cls.delete_task(task_id):
            deleted_count += 1
    
    return deleted_count
```

```json
}
```

**响应**:
```json
{
    "success": true,
    "message": "已清理 5 个 7 天前的任务",
    "count": 5
}
```

任务完成后会自动清理日志，无需手动操作。

**清理策略**:
- ✅ 保留任务信息（状态、进度、时间等）
- ✅ 保留最后10条日志作为摘要
- ❌ 删除完整日志文件
- ❌ 清理内存中的日志

## 注意事项

### 1. 日志恢复

⚠️ **清理后的日志无法恢复**

- 清理前请确认不再需要完整日志
- 建议保留摘要（keep_summary=True）
- 重要任务可以手动备份日志

### 2. 清理时机

✅ **建议清理时机**:
- 任务完成后自动清理（已实现）
- 定期清理旧任务（每周一次）
- 磁盘空间不足时手动清理

❌ **不建议清理**:
- 任务执行中
- 需要调试的任务
- 重要的历史任务

### 3. 性能影响

- 清理操作很快（毫秒级）
- 不影响正在执行的任务
- 建议在任务较少时清理
