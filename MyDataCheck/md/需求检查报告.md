### ✅ 1. 多场景支持

**实现状态**：✅ 完全支持

**实现方式**：
- `执行配置.json` 中的 `scenarios` 数组支持配置多个场景
- 每个场景可以独立启用/禁用（`enabled` 字段）
- 主流程会遍历所有启用的场景并依次执行

**代码位置**：
- `执行对比流程.py` 第109-136行：过滤并执行所有启用的场景

---

### ✅ 2. 不同文件对应不同接口

**实现状态**：✅ 完全支持

**实现方式**：
- 每个场景可以配置独立的 `input_csv_file`（输入文件）
- 每个场景可以配置独立的 `api_url`（接口URL）
- 每个场景的输出文件使用独立的 `output_file_prefix` 区分

**代码位置**：
- `流程执行器.py` 第197-200行：从场景配置中读取 `input_csv_file` 和 `api_url`
- `流程执行器.py` 第219-227行：为每个场景生成独立的输出文件名

**示例配置**：
```json
{
  "scenarios": [
    {
      "name": "文件1_接口1",
      "input_csv_file": "文件1.csv",
      "api_url": "http://api1.example.com/endpoint1"
    },
    {
      "name": "文件2_接口2",
      "input_csv_file": "文件2.csv",
      "api_url": "http://api2.example.com/endpoint2"
    }
  ]
}
```

### ✅ 3. 不同CSV文件的列索引不一致

**实现状态**：✅ 完全支持

**实现方式**：
- **方式1：自动检测**（推荐）
  - 设置 `auto_detect_columns: true`
  - 系统会自动检测每个CSV文件的列索引
  - 检测结果按场景名称独立保存到 `列索引配置.json`
  
- **方式2：手动配置**
  - 设置 `auto_detect_columns: false`
  - 在 `column_config` 中手动指定每个场景的列索引

**代码位置**：
- `流程执行器.py` 第260-277行：自动检测模式的处理逻辑
- `流程执行器.py` 第278-286行：手动配置模式的处理逻辑
- `检测列索引.py` 第21-104行：自动检测列索引的实现
- `流程执行器.py` 第44-123行：智能匹配已有配置的逻辑

**示例配置（自动检测）**：
```json
{
  "scenarios": [
    {
      "name": "文件1_接口1",
      "input_csv_file": "文件1.csv",
      "auto_detect_columns": true,
      "column_config": {
        "cust_no_column": 0,      // 备用值（检测失败时使用）
        "use_create_time_column": 2
      }
    },
    {
      "name": "文件2_接口2",
      "input_csv_file": "文件2.csv",
      "auto_detect_columns": true,
      "column_config": {
        "cust_no_column": 1,      // 文件2的cust_no在第1列
        "use_create_time_column": 3  // 文件2的时间字段在第3列
      }
    }
  ]
}
```

**示例配置（手动配置）**：
```json
{
  "scenarios": [
    {
      "name": "文件1_接口1",
      "input_csv_file": "文件1.csv",
      "auto_detect_columns": false,
      "column_config": {
        "cust_no_column": 0,
        "use_create_time_column": 2,
        "feature_start_column": 3
      }
    },
    {
      "name": "文件2_接口2",
      "input_csv_file": "文件2.csv",
      "auto_detect_columns": false,
      "column_config": {
        "cust_no_column": 1,      // 不同的列索引
        "use_create_time_column": 4,  // 不同的列索引
        "feature_start_column": 5
      }
    }
  ]
}
```

## 使用建议

1. **推荐使用自动检测模式**（`auto_detect_columns: true`）
   - 系统会自动检测每个CSV文件的列索引
   - 检测结果会自动保存，下次执行时直接使用

2. **如果列索引固定且已知**
   - 可以设置 `auto_detect_columns: false`
   - 在 `column_config` 中手动指定列索引

3. **添加新场景**
   - 在 `执行配置.json` 的 `scenarios` 数组中添加新场景
   - 设置 `enabled: true` 启用场景
   - 配置对应的 `input_csv_file` 和 `api_url`

4. **临时禁用场景**
   - 设置 `enabled: false` 即可跳过该场景
