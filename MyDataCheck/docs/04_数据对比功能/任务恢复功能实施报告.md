## 问题背景

### 用户反馈的问题

用户在使用接口数据对比功能时遇到以下问题：

1. **锁屏后显示network error**
   - 用户启动任务后锁屏
   - 一段时间后解锁，Web界面显示"错误: network error"
   - 无法看到任务是否还在执行
   - 无法查看执行进度

2. **刷新页面后丢失输出**
   - 前端SSE连接断开后，刷新页面
   - 之前的所有输出日志丢失
   - 不知道任务执行到哪一步

3. **关闭浏览器后无法恢复**
   - 关闭浏览器后重新打开
   - 无法知道任务是否还在执行
   - 无法查看任务进度

### 根本原因

1. **前端SSE连接超时**
   - 浏览器的Server-Sent Events连接有超时限制
   - 锁屏后，浏览器可能降低后台标签页的优先级
   - 长时间无数据传输，连接被关闭

2. **任务状态未持久化**
   - 任务状态只保存在内存中
   - 前端连接断开后，无法查询任务状态

3. **日志未持久化**
   - 输出日志只通过SSE发送到前端
   - 前端断开后，日志丢失

---

## 解决方案

### 核心思路

1. **任务状态持久化** - 将任务状态保存到文件系统
2. **日志持久化** - 将所有输出日志保存到文件
3. **自动检测恢复** - 页面加载时自动检测运行中的任务
4. **实时更新** - 通过轮询获取最新状态和日志

**关键代码**:
```python
class TaskManager:
    """任务状态管理器（单例模式）"""
    
    @classmethod
    def create_task(cls, task_name: str, task_type: str = "unknown") -> str:
        """创建任务"""
        task_id = str(uuid.uuid4())
        task_info = {
            'task_id': task_id,
            'task_name': task_name,
            'task_type': task_type,
            'status': 'pending',
            'progress': 0,
            'total': 0,
            'current_step': '',
            'created_at': datetime.now().isoformat(),
            # ...
        }
        instance._tasks[task_id] = task_info
        instance._save_task_info(task_id)
        return task_id
    
    @classmethod
    def add_log(cls, task_id: str, message: str, level: str = "info") -> bool:
        """添加日志"""
        log_entry = {
            'timestamp': datetime.now().isoformat(),
            'level': level,
            'message': message
        }
        instance._logs[task_id].append(log_entry)
        instance._append_log_to_file(task_id, log_entry)
        return True
```

```python
class TaskOutputCapture(OutputCapture):
    def write(self, text):
        super().write(text)  # 发送到队列
        # 同时保存到任务管理器
        if self.task_id and text.strip():
            level = "info"
            if "错误" in text or "❌" in text:
                level = "error"
            elif "成功" in text or "✅" in text:
                level = "success"
            TaskManager.add_log(self.task_id, text.strip(), level)
```

```python
@api_bp.route('/api/tasks', methods=['GET'])
def get_tasks():
    """获取所有任务列表"""

@api_bp.route('/api/tasks/<task_id>', methods=['GET'])

@api_bp.route('/api/tasks/<task_id>/logs', methods=['GET'])
def get_task_logs(task_id):
    """获取指定任务的日志"""
    last_n = request.args.get('last_n', type=int)
    from_file = request.args.get('from_file', 'false').lower() == 'true'
    logs = TaskManager.get_logs(task_id, last_n=last_n, from_file=from_file)
    return jsonify({'success': True, 'logs': logs})
```

```javascript
function checkRunningTasks() {
    fetch('/api/tasks?status=running')
        .then(response => response.json())
        .then(data => {
            if (data.success && data.tasks && data.tasks.length > 0) {
                showTaskRecoveryPrompt(data.tasks[0]);
            }
        });
}
```

2. **recoverTask(taskId)** - 恢复任务
```javascript
function recoverTask(taskId) {
    clearOutput('api');
    appendOutput('正在恢复任务...', 'info', 'api');
    
    // 获取任务信息
    fetch(`/api/tasks/${taskId}`)
        .then(response => response.json())
        .then(data => {
            // 显示任务信息
            // 加载历史日志
            loadTaskLogs(taskId);
            // 开始轮询
            if (data.task.status === 'running') {
                startLogPolling(taskId);
            }
        });
}
```

3. **startLogPolling(taskId)** - 轮询更新
```javascript
function startLogPolling(taskId) {
    logPollingInterval = setInterval(() => {
        // 查询任务状态
        fetch(`/api/tasks/${taskId}`)
            .then(response => response.json())
            .then(data => {
                // 更新进度
                // 加载新日志
                // 检查是否完成
                if (data.task.status !== 'running') {
                    stopLogPolling();
                }
            });
    }, 2000);  // 每2秒轮询
}
```

## 性能影响

### 存储空间

| 项目 | 估算大小 | 说明 |
|------|---------|------|
| 任务信息文件 | ~1KB | 每个任务一个JSON文件 |
| 任务日志文件 | 100KB-10MB | 取决于任务输出量 |
| 内存占用 | ~1MB | 每个任务最多1000条日志 |

### 网络开销

| 操作 | 频率 | 数据量 |
|------|------|--------|
| 检测任务 | 页面加载时1次 | ~1KB |
| 轮询状态 | 每2秒1次 | ~1KB |
| 加载日志 | 每2秒1次 | 1-100KB |

### 建议

1. **定期清理**: 建议每周清理一次旧任务日志
2. **限制日志**: 如果日志过多，可以限制加载数量
3. **调整轮询**: 如果网络慢，可以调整轮询间隔
