# feature_stats.csv 格式优化说明

## 修改内容

优化了 `feature_stats.csv` 文件的输出格式，使其更易读和分析。

## 新格式

### 文件结构

```csv
特征统计,,,,,,
无异常特征总数,150,,,,,,
有异常特征总数,50,,,,,,
,,,,,,
特征名,是否有异常,比对数据条数,匹配数量,异常数量,匹配率(%),异常率(%)
cdc_feature_1,有异常,1000,800,200,80.00,20.00
cdc_feature_2,有异常,1000,850,150,85.00,15.00
cdc_feature_3,有异常,1000,900,100,90.00,10.00
...
cdc_feature_n,无异常,1000,1000,0,100.00,0.00
```

### 关键改进

1. **第1-4行：汇总信息**
   - 第1行：标题"特征统计"
   - 第2行：无异常特征总数
   - 第3行：有异常特征总数
   - 第4行：空行（分隔符）

2. **第5行：列名**
   - 特征名
   - 是否有异常
   - 比对数据条数
   - 匹配数量
   - 异常数量
   - 匹配率(%)
   - 异常率(%)

3. **第6行开始：特征明细**
   - **按异常率降序排列**（异常最多的在最上面）
   - 包含所有特征（有异常和无异常）

## 旧格式 vs 新格式

### 旧格式问题

1. ❌ 列名在第1行，不便于查看汇总信息
2. ❌ 先显示无异常的，再显示有异常的
3. ❌ 需要滚动到后面才能看到异常最多的特征

### 新格式优势

1. ✅ 汇总信息在最前面，一眼就能看到整体情况
2. ✅ 列名在第5行，便于Excel筛选和排序
3. ✅ 异常最多的特征在最上面，便于快速定位问题
4. ✅ 统一了三种模式的输出格式

## 使用示例

### 在Excel中查看

1. 打开 `feature_stats.csv`
2. 前4行显示汇总信息
3. 第5行是列名，可以设置为冻结行
4. 从第6行开始是特征明细，异常率最高的在最上面

### 快速定位问题特征

```
异常率(%) > 10% 的特征会显示在最前面
异常率(%) = 0% 的特征会显示在最后面
```
