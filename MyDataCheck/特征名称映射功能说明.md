# 特征名称映射功能说明

## 功能概述

在CSV数据对比模块中新增了"特征名称映射"功能，允许在对比前为指定文件的特征列名添加前缀和/或后缀，然后再进行对比。

## 使用场景

当两个文件的特征列名不完全一致时，可以通过添加前缀/后缀来统一列名格式，从而实现正确的对比。

### 示例场景

**场景1：列名有前缀差异**
- 文件1的特征列：`age`, `income`, `score`
- 文件2的特征列：`model_age`, `model_income`, `model_score`
- 解决方案：为文件1添加前缀 `model_`

**场景2：列名有后缀差异**
- 文件1的特征列：`age_v1`, `income_v1`, `score_v1`
- 文件2的特征列：`age`, `income`, `score`
- 解决方案：为文件2添加后缀 `_v1`

**场景3：列名有前缀和后缀差异**
- 文件1的特征列：`age`, `income`, `score`
- 文件2的特征列：`api_age_new`, `api_income_new`, `api_score_new`
- 解决方案：为文件1添加前缀 `api_` 和后缀 `_new`

## 功能配置

### Web界面配置

1. 在"数据对比"页面，找到"启用特征名称映射"选项
2. 勾选该选项后，会展开映射配置区域
3. 配置以下参数：
   - **映射文件**：选择要添加前缀/后缀的文件
     - `模型特征表 (file1)`：为第一个文件的特征列添加映射
     - `接口/灰度/从库特征表 (file2)`：为第二个文件的特征列添加映射
   - **特征列名前缀**：要添加的前缀（可选）
   - **特征列名后缀**：要添加的后缀（可选）

### 配置示例

```json
{
  "scenarios": [{
    "name": "当前配置",
    "enabled": true,
    "sql_file": "file1.csv",
    "api_file": "file2.csv",
    "sql_key_column": [0],
    "api_key_column": [0],
    "sql_feature_start": 1,
    "api_feature_start": 1,
    "convert_feature_to_number": true,
    "enable_column_mapping": true,
    "mapping_file": "file1",
    "mapping_prefix": "model_",
    "mapping_suffix": "_v1"
  }]
}
```

## 功能特点

### 1. 灵活配置
- 可以只添加前缀
- 可以只添加后缀
- 可以同时添加前缀和后缀
- 可以选择为哪个文件添加映射

### 2. 默认行为
- 默认不启用映射功能（保持现有逻辑）
- 只有勾选"启用特征名称映射"后才会应用映射

### 3. 映射逻辑
- 映射只影响特征列（从`feature_start`开始的列）
- 主键列不受影响
- 映射在对比前执行，不会修改原始文件

## 使用步骤

### 步骤1：上传文件
上传两个需要对比的CSV文件

### 步骤2：配置基本参数
- 设置主键列索引
- 设置特征起始列索引
- 设置其他对比选项

### 步骤3：配置特征名称映射（可选）
1. 勾选"启用特征名称映射"
2. 选择要映射的文件（file1或file2）
3. 输入前缀和/或后缀
4. 示例：
   - 前缀：`model_`
   - 后缀：`_v1`
   - 结果：`age` → `model_age_v1`

### 步骤4：执行对比
点击"执行对比"按钮，系统会：
1. 读取两个文件
2. 应用特征名称映射（如果启用）
3. 根据映射后的列名进行对比
4. 生成对比报告

## 技术实现

### 后端实现
- 文件：`data_comparison/job/data_comparator.py`
- 函数：`compare_two_files()`
- 新增参数：
  - `enable_column_mapping`: 是否启用映射
  - `mapping_file`: 映射文件（'file1'或'file2'）
  - `mapping_prefix`: 前缀
  - `mapping_suffix`: 后缀

### 前端实现
- 文件：`templates/index.html`
- 文件：`static/js/data-compare.js`
- 新增UI组件：
  - 映射启用复选框
  - 映射配置区域（文件选择、前缀、后缀输入框）
- 新增函数：
  - `toggleMappingConfig()`: 切换映射配置显示

### 路由实现
- 文件：`web/routes/compare_routes.py`
- 函数：`execute_compare_flow()`
- 传递映射参数到对比函数

## 注意事项

1. **映射不修改原文件**：映射只在内存中进行，不会修改原始CSV文件
2. **只映射特征列**：主键列和特征起始列之前的列不受影响
3. **配置可保存**：映射配置会保存到`data_comparison/config.json`
4. **向后兼容**：默认不启用映射，保持现有逻辑不变

## 常见问题

### Q1：映射后找不到匹配的特征列怎么办？
A：请检查映射后的列名是否与另一个文件的列名完全一致（包括大小写）。

### Q2：可以为两个文件都添加映射吗？
A：目前只支持为一个文件添加映射。如果需要为两个文件都添加，建议先处理文件列名后再上传。

### Q3：映射会影响主键列吗？
A：不会。映射只影响特征列（从`feature_start`开始的列），主键列不受影响。

### Q4：如何取消映射？
A：取消勾选"启用特征名称映射"即可恢复默认行为。
