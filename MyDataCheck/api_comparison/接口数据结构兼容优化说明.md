# 接口数据结构兼容优化说明

## 优化目标

优化代码以兼容不同的接口返回数据结构，特别是 `data` 字段的不同结构。

## 支持的接口返回结构

### 结构1: 特征字段在顶层
```json
{
  "retCode": 0.0,
  "retMsg": "success",
  "success": 1.0,
  "multi_loan_in_loan_order_all_completedloanamountindoubletype": "1000.000000",
  "multi_loan_order_info_furthestsingleorder_termscnt": "4",
  "timestamp": 1769049417647.0
}
```

### 结构2: 特征字段在data字段中（扁平结构）
```json
{
  "retCode": 0.0,
  "retMsg": "success",
  "success": 1.0,
  "data": {
    "multi_loan_in_loan_order_all_completedloanamountindoubletype": "1000.000000",
    "multi_loan_order_info_furthestsingleorder_termscnt": "4"
  },
  "timestamp": 1769049417647.0
}
```

### 结构3: 特征字段在data字段的嵌套结构中
```json
{
  "retCode": 0.0,
  "retMsg": "success",
  "success": 1.0,
  "data": {
    "result": {
      "multi_loan_in_loan_order_all_completedloanamountindoubletype": "1000.000000",
      "multi_loan_order_info_furthestsingleorder_termscnt": "4"
    }
  },
  "timestamp": 1769049417647.0
}
```

### 结构4: data字段为空（会在顶层查找）
```json
{
  "retCode": 0.0,
  "retMsg": "success",
  "success": 1.0,
  "data": {},
  "timestamp": 1769049417647.0
}
```

## 优化内容

### 1. 优化 `_find_feature_value_in_api_response()` 方法

**优化前**：
- 只在 `data` 字段的第一层查找
- 如果 `data` 字段为空，不会在顶层查找

**优化后**：
- ✅ **递归搜索**：支持在 `data` 字段的任意嵌套层级中查找
- ✅ **智能查找策略**：
  1. 优先在 `data` 字段中查找（如果 `data` 非空）
  2. 如果 `data` 字段为空，自动在顶层查找
  3. 递归搜索整个响应结构
  4. 支持大小写不敏感匹配
- ✅ **跳过系统字段**：自动跳过 `retCode`, `retMsg`, `success`, `timestamp` 等系统字段

### 2. 优化 `collect_leaf_field_names()` 函数

**优化前**：
- 会收集系统字段（如 `retCode`, `retMsg` 等）
- 如果 `data` 字段为空，不会收集任何特征字段

**优化后**：
- ✅ **跳过系统字段**：不收集 `retCode`, `retMsg`, `success`, `timestamp` 等系统字段
- ✅ **智能处理 data 字段**：
  - 如果 `data` 字段非空，递归收集 `data` 内部的字段
  - 如果 `data` 字段为空，会在顶层查找特征字段
- ✅ **递归收集**：支持任意嵌套层级的字段收集

### 3. 优化 `collect_field_paths()` 函数

**优化前**：
- 会收集系统字段的路径
- 如果 `data` 字段为空，不会收集路径映射

**优化后**：
- ✅ **跳过系统字段**：不收集系统字段的路径
- ✅ **智能处理 data 字段**：正确处理 `data` 字段的嵌套结构
- ✅ **路径映射**：正确生成字段名到路径的映射

## 查找策略

### 特征值查找的优先级

1. **优先在 data 字段中查找**（如果 `data` 字段存在且非空）
   - 递归搜索 `data` 字段的所有嵌套层级
   - 精确匹配优先，然后大小写不敏感匹配

2. **在顶层查找**（如果 `data` 字段为空或未找到）
   - 跳过系统字段（`retCode`, `retMsg`, `success`, `timestamp`, `data`）
   - 精确匹配优先，然后大小写不敏感匹配

3. **递归搜索整个响应**
   - 如果以上策略都未找到，递归搜索整个响应结构

4. **使用点号分隔的嵌套路径**
   - 最后尝试使用点号分隔的路径（如 `data.result.field`）

## 使用示例

### 示例1: data字段有值
```python
api_response = {
    "retCode": 0.0,
    "success": 1.0,
    "data": {
        "field1": "value1"
    }
}
# 查找 "field1" → 在 data 中找到 → 返回 "value1"
```

### 示例2: data字段为空，特征在顶层
```python
api_response = {
    "retCode": 0.0,
    "success": 1.0,
    "data": {},
    "field1": "value1"
}
# 查找 "field1" → data为空，在顶层找到 → 返回 "value1"
```

### 示例3: data字段嵌套结构
```python
api_response = {
    "retCode": 0.0,
    "success": 1.0,
    "data": {
        "result": {
            "field1": "value1"
        }
    }
}
# 查找 "field1" → 递归搜索data，在data.result中找到 → 返回 "value1"
```

## 注意事项

1. **性能考虑**：递归搜索可能会稍微增加查找时间，但对于正常的数据量影响很小

2. **字段冲突**：如果不同层级有相同名称的字段，会优先使用找到的第一个值

3. **系统字段**：代码会自动跳过系统字段，不会将它们作为特征字段处理
