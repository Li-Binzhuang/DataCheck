# 分批输出开关功能说明

## 概述

为CDC项目的四个衍生脚本添加了分批输出开关功能，允许用户选择是分批输出多个文件，还是直接输出单个全量文件。

## 修改的脚本

1. **第一板块衍生.ipynb**
2. **第二板块衍生.ipynb**
3. **第三板块衍生.ipynb**
4. **BOSS板块衍生.ipynb**

## 新增配置项

### ENABLE_BATCH_OUTPUT

```python
ENABLE_BATCH_OUTPUT = True  # 是否启用分批输出（True: 分批输出多个文件，False: 只输出单个全量文件）
```

- **类型**: Boolean
- **默认值**: True
- **作用**: 控制特征文件的输出方式

## 使用说明

### 启用分批输出（ENABLE_BATCH_OUTPUT = True）

当设置为 `True` 时，脚本会按照 `BATCH_SIZE` 设置的大小将数据分批输出到多个文件：

```python
ENABLE_BATCH_OUTPUT = True
BATCH_SIZE = 2000  # 每个文件2000条数据
```

**输出示例**：
```
[INFO] 开始分批输出特征文件
[INFO] 总数据量: 12546 行
[INFO] 批次大小: 2000 行/文件
[INFO] 输出文件数: 7 个

[WRITE] 批次 1/7: cdc1_features_batch001_1-2000.csv
        行范围: 1 - 2000 (2000 行)
[WRITE] 批次 2/7: cdc1_features_batch002_2001-4000.csv
        行范围: 2001 - 4000 (2000 行)
...
[SUCCESS] 分批输出完成！
```

**输出文件命名规则**：
- 第一板块: `cdc1_features_batch{序号}_{起始行}-{结束行}.csv`
- 第二板块: `cdc2_features_batch{序号}_{起始行}-{结束行}.csv`
- 第三板块: `cdc3_features_batch{序号}_{起始行}-{结束行}.csv`
- BOSS板块: `cdcboss_features_batch{序号}_{起始行}-{结束行}.csv`

### 禁用分批输出（ENABLE_BATCH_OUTPUT = False）

当设置为 `False` 时，脚本会直接输出单个全量文件，不进行分批：

```python
ENABLE_BATCH_OUTPUT = False
```

**输出示例**：
```
[INFO] 分批输出已禁用，直接输出单个特征文件
[WRITE] 特征文件: cdc1_features.csv
        总行数: 12546 行
[SUCCESS] 特征文件输出完成！
written:
- outputs/cdc1_features.csv
```

**输出文件命名规则**：
- 第一板块: `cdc1_features.csv`
- 第二板块: `cdc2_features.csv`
- 第三板块: `cdc3_features.csv`
- BOSS板块: `cdcboss_features.csv`

## 配置示例

### 第一板块衍生

```python
WRITE_OUTPUTS = True
WRITE_FLAT_CSV = False
ENABLE_BATCH_OUTPUT = True   # 启用分批输出
BATCH_SIZE = 2100            # 每批2100条
WRITE_FULL_CSV = True        # 同时输出全量文件
```

### 第二板块衍生

```python
ENABLE_BATCH_OUTPUT = False  # 禁用分批输出，直接输出单个文件
BATCH_SIZE = 200
WRITE_FULL_CSV = False
```

### 第三板块衍生

```python
ENABLE_BATCH_OUTPUT = True   # 启用分批输出
BATCH_SIZE = 6150
WRITE_FULL_CSV = True
```

### BOSS板块衍生

```python
WRITE_FEATURES_CSV = True
ENABLE_BATCH_OUTPUT = True   # 启用分批输出
BATCH_SIZE = 6150
WRITE_FULL_CSV = True
```

## 与其他配置项的关系

### WRITE_FULL_CSV

`WRITE_FULL_CSV` 配置项与 `ENABLE_BATCH_OUTPUT` 是独立的：

- **ENABLE_BATCH_OUTPUT = True, WRITE_FULL_CSV = True**
  - 输出分批文件 + 输出全量文件（两种都输出）
  
- **ENABLE_BATCH_OUTPUT = True, WRITE_FULL_CSV = False**
  - 只输出分批文件
  
- **ENABLE_BATCH_OUTPUT = False, WRITE_FULL_CSV = True**
  - 输出单个特征文件 + 输出全量文件（实际上是同一个文件）
  
- **ENABLE_BATCH_OUTPUT = False, WRITE_FULL_CSV = False**
  - 只输出单个特征文件

## 使用建议

### 何时使用分批输出

1. **数据量大**：当数据量超过10000条时，分批输出便于管理和传输
2. **分批处理**：需要对数据进行分批处理或验证时
3. **内存限制**：在内存受限的环境中，分批读取更安全

### 何时使用单文件输出

1. **数据量小**：数据量较小（<5000条）时，单文件更方便
2. **后续处理**：后续需要对全量数据进行统一处理时
3. **简化管理**：不需要管理多个文件时

## 注意事项

1. **文件命名**：分批文件和单文件的命名规则不同，请注意区分
2. **输出目录**：所有文件都输出到 `outputs/` 目录
3. **编码格式**：所有文件使用 `utf-8-sig` 编码，支持中文
4. **数值精度**：特征值保留6位小数，空值填充为 -999.0
5. **配置独立**：每个板块的配置是独立的，可以分别设置
