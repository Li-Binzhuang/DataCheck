# 接口 500 错误排查指南

## 错误信息

```
请求失败 - params={'applyId': '1111232627889987585', 'baseTime': '2025-12-10T07:38:46.688'}, 
错误: 500 Server Error for url: http://10.129.2.25:8081/riskanalysis/Cdc/features
```

## 错误分析

### 500 错误含义

**HTTP 500 Internal Server Error** 表示服务器端出现了错误，不是客户端的问题。

### 可能的原因

#### 1. 服务器端异常 ⭐ 最可能

**现象**: 
- 接口服务器内部错误
- 数据库查询失败
- 业务逻辑异常

**排查**:
```bash
# 检查服务器日志
# 联系后端开发查看错误堆栈
```

**常见情况**:
- 数据库连接超时
- SQL 查询错误
- 空指针异常
- 数据格式错误

#### 2. 参数问题

**可能的参数问题**:

a) **applyId 不存在**
```
applyId: '1111232627889987585'
```
- 这个 ID 在数据库中可能不存在
- 或者已被删除

b) **baseTime 格式问题**
```
baseTime: '2025-12-10T07:38:46.688'
```
- 注意：时间是 **2025年**（未来时间）
- 服务器可能不接受未来时间
- 毫秒精度可能有问题

c) **时区问题**
- 时间没有时区信息
- 服务器可能期望 UTC 时间

#### 3. 服务器资源问题

**可能情况**:
- 服务器内存不足
- 数据库连接池耗尽
- 查询超时

#### 4. 接口版本或配置变更

**可能情况**:
- 接口参数要求变更
- 需要额外的认证
- 接口地址变更

## 常见解决方案

### 方案 1: 修正时间格式

**问题**: 时间是 2025 年（未来时间）

**解决**:
```python
# 检查数据源，确保时间正确
# 如果是测试数据，使用当前时间
from datetime import datetime
base_time = datetime.now().strftime("%Y-%m-%dT%H:%M:%S.%f")[:-3]
```

### 方案 2: 添加时区信息

**问题**: 缺少时区

**解决**:
```python
# 添加 UTC 时区
base_time = "2025-12-10T07:38:46.688Z"  # 注意末尾的 Z

# 或使用本地时区
base_time = "2025-12-10T07:38:46.688+08:00"  # 东八区
```

### 方案 3: 调整毫秒精度

**问题**: 毫秒位数可能不对

**解决**:
```python
# 确保3位毫秒
time_str = "2025-12-10T07:38:46.688"  # 3位毫秒

# 或去掉毫秒
time_str = "2025-12-10T07:38:46"
```

### 方案 5: 添加重试机制

**问题**: 临时性服务器错误

**解决**:
```python
import time
from requests.adapters import HTTPAdapter
from requests.packages.urllib3.util.retry import Retry

# 配置重试
retry_strategy = Retry(
    total=3,
    status_forcelist=[500, 502, 503, 504],
    backoff_factor=1
)
adapter = HTTPAdapter(max_retries=retry_strategy)
session = requests.Session()
session.mount("http://", adapter)

# 使用 session 发送请求
response = session.post(url, json=params, timeout=60)
```

### 工具 2: 参数验证脚本

```python
def validate_params(apply_id, base_time):
    """验证参数有效性"""
    issues = []
    
    # 检查 applyId
    if not apply_id:
        issues.append("applyId 为空")
    elif not apply_id.isdigit():
        issues.append("applyId 不是纯数字")
    
    # 检查 baseTime
    if not base_time:
        issues.append("baseTime 为空")
    else:
        # 检查时间格式
        from datetime import datetime
        try:
            dt = datetime.fromisoformat(base_time.replace('Z', '+00:00'))
            
            # 检查是否是未来时间
            if dt > datetime.now():
                issues.append(f"baseTime 是未来时间: {base_time}")
            
            # 检查是否太久远
            if dt.year < 2020:
                issues.append(f"baseTime 太久远: {base_time}")
                
        except ValueError as e:
            issues.append(f"baseTime 格式错误: {e}")
    
    return issues
```
