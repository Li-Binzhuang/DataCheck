## v1.3 - 主键值获取逻辑优化
**时间：** 2026-01-XX

### 优化内容
1. **主键值获取优化**
   - 优先从在线文件（Sql文件/文件1）获取主键值
   - 如果在线文件的主键值为空，则从离线文件（接口文件/文件2）获取
   - 确保输出文件中的主键值始终有值

2. **实现逻辑**
   - 先用 `key_value_offline` 进行记录匹配
   - 匹配成功后，优先使用 `online_row` 的主键值作为输出的 `key_value`
   - 如果 `online_row` 的主键值为空，则使用 `key_value_offline`

## v1.4 - 修复转换函数处理引号问题
**时间：** 2026-01-26

### 优化内容
1. **修复引号处理问题**
   - 发现部分数据中包含多余的英文双引号
   - 优化转换函数，确保无论是否能转换为数值，都会去除字符串两端的引号
   - 修改返回逻辑：转换失败时返回去除引号后的字符串，而不是原值

2. **增强的转换规则**
   - `"8"` → 去除双引号 → `8` → 转换为数字 `8`
   - `'"8"'` → 去除外层引号 → `"8"` → 去除双引号 → `8` → 转换为数字 `8`
   - `'8'` → 去除单引号 → `8` → 转换为数字 `8`
   - `"8.5"` → 去除双引号 → `8.5` → 转换为浮点数 `8.5`
   - `"abc"` → 去除双引号 → `abc` → 无法转换，返回字符串 `abc`
   - `"  8  "` → 去除双引号 → `  8  ` → 去除空格 → `8` → 转换为数字 `8`

3. **测试验证**
   - 创建了完整的测试用例
   - 所有14个测试用例全部通过
   - 验证了各种边界情况

## 使用指南

### 基本流程
1. 选择对应的Tab（接口数据对比/线上灰度落数对比/数据对比）
2. 上传Sql文件和接口文件
3. 配置主键列索引和特征起始列索引
4. 选择是否转换特征值为数值（默认勾选）
5. 设置输出文件前缀（可选）
6. 点击"执行对比"按钮
7. 查看实时输出日志
8. 下载生成的对比结果文件

### 配置说明
- **主键列索引**：从0开始，A列=0，B列=1，以此类推
- **特征起始列索引**：特征数据开始的列索引
- **转换特征值为数值**：勾选后会自动去除引号并尝试转换为数值
- **输出文件前缀**：用于区分不同的对比任务

### 注意事项
1. 确保两个文件的主键列数据类型一致
2. 特征列应该从指定的起始列开始
3. 转换功能会影响所有特征值的对比结果
4. 输出文件保存在 `outputdata/online_comparison/` 目录
