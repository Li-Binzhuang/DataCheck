# CDC 项目输出控制统一说明

## 概述

CDC 项目的四个板块（第一板块、第二板块、第三板块、BOSS板块）现在都支持灵活的输出控制，可以根据需要选择输出哪些文件。

---

## 统一的开关设计

所有板块（除 BOSS 板块外）都采用**双开关设计**：

### 1. 总开关：`WRITE_OUTPUTS`

控制是否输出任何文件。

```python
WRITE_OUTPUTS = True   # 启用输出
WRITE_OUTPUTS = False  # 完全关闭输出
```

### 2. 明细表开关：`WRITE_FLAT_CSV`

控制是否输出明细表（平铺数据）。

```python
WRITE_FLAT_CSV = True   # 输出明细表
WRITE_FLAT_CSV = False  # 跳过明细表（默认）
```

---

## 各板块输出文件对照表

| 板块 | 明细表文件 | 特征表文件 | 明细表行数（示例） | 特征列数 |
|------|-----------|-----------|------------------|---------|
| 第一板块 | `consultas_flat.csv` | `features_consultas.csv` | 10,365 | 988 |
| 第二板块 | `creditos_flat.csv` | `features_creditos.csv` | 697,815 | 16,647 |
| 第三板块 | `clave_prevencion_flat.csv` | `features_clave_prevencion.csv` | 19,233 | 120 |
| BOSS板块 | - | `boss_features_full_<timestamp>.csv` | - | 149 |

**注意**：
- BOSS 板块只有特征表，没有明细表
- BOSS 板块使用 `WRITE_FEATURES_CSV` 开关（带时间戳，不会覆盖）
- 其他三个板块的特征表会覆盖同名文件

---

## 推荐配置（默认配置）

### 第一、二、三板块

```python
WRITE_OUTPUTS = True      # 启用输出
WRITE_FLAT_CSV = False    # 不输出明细表（节省空间）
```

**输出结果**：
- ✅ 只输出特征表
- ❌ 跳过明细表

### BOSS 板块

```python
WRITE_FEATURES_CSV = True  # 输出特征表（带时间戳）
```

**输出结果**：
- ✅ 输出 `boss_features_full_<timestamp>.csv`

## 使用场景对照

### 场景1：只需要特征表（推荐）

**适用于**：建模、特征工程、生产环境

**配置**：
```python
# 第一、二、三板块
WRITE_OUTPUTS = True
WRITE_FLAT_CSV = False

# BOSS 板块
WRITE_FEATURES_CSV = True
```

**优点**：
- 节省磁盘空间（尤其是第二板块，明细表约 200MB）
- 输出速度快
- 文件管理简单

---

### 场景2：需要核对明细数据

**适用于**：数据质量检查、问题排查、数据分析

**配置**：
```python
# 第一、二、三板块
WRITE_OUTPUTS = True
WRITE_FLAT_CSV = True

# BOSS 板块
WRITE_FEATURES_CSV = True
```

**优点**：
- 可以查看原始明细记录
- 便于核对特征计算逻辑
- 便于发现数据异常

---

### 场景3：只在内存中计算（调试）

**适用于**：代码调试、测试运行

**配置**：
```python
# 第一、二、三板块
WRITE_OUTPUTS = False
WRITE_FLAT_CSV = False  # 此时无效

# BOSS 板块
WRITE_FEATURES_CSV = False
```

**优点**：
- 不产生文件
- 运行速度最快
- 可以在 notebook 中直接查看变量

## 常见问题

### Q1：为什么默认不输出明细表？

**A**：
1. 明细表文件很大（尤其是第二板块约 200MB）
2. 大多数情况下只需要特征表用于建模
3. 明细表主要用于数据核对，不是必需的
4. 可以在 notebook 中直接查看内存中的明细数据

### Q2：如何同时输出所有板块的文件？

**A**：
```python
# 第一板块
WRITE_OUTPUTS = True
WRITE_FLAT_CSV = True

# 第二板块
WRITE_OUTPUTS = True
WRITE_FLAT_CSV = True

# 第三板块
WRITE_OUTPUTS = True
WRITE_FLAT_CSV = True

# BOSS 板块
WRITE_FEATURES_CSV = True
```

然后依次运行所有 notebook。

### Q3：输出文件会覆盖吗？

**A**：
- 第一、二、三板块：会覆盖同名文件（固定文件名）
- BOSS 板块：不会覆盖（文件名带时间戳）

### Q4：如何只输出某一个板块的文件？

**A**：
- 只运行该板块的 notebook
- 或者关闭其他板块的 `WRITE_OUTPUTS` 开关

### Q5：输出文件的编码是什么？

**A**：
- 所有输出文件都使用 `UTF-8 with BOM` 编码
- 可以直接在 Excel 中打开，不会出现乱码
- 特征值统一保留 2 位小数
