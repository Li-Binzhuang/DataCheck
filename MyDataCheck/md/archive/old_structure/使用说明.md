# 场景1：接口数据对比 - 使用说明

## 配置方式

### 方式1：使用JSON配置文件（推荐，支持多场景）

1. **修改 `执行对比流程.py` 中的配置**：
   ```python
   CONFIG_JSON_FILE = "执行配置.json"  # 设置为你的JSON配置文件路径
   ```

2. **编辑 `执行配置.json` 文件**，添加你的场景配置：
   ```json
   {
     "scenarios": [
       {
         "name": "场景名称",
         "enabled": true,
         "input_csv_file": "输入文件.csv",
         "output_file_prefix": "输出前缀",
         "api_url": "接口URL",
         "thread_count": 150,
         "timeout": 60,
         "auto_detect_columns": true,
         "column_config": {
           "cust_no_column": 1,
           "use_create_time_column": 3,
           "feature_start_column": 4
         }
       }
     ],
     "global_config": {
       "default_thread_count": 150,
       "default_timeout": 60,
       "default_auto_detect_columns": true
     }
   }
   ```

3. **运行脚本**：
   ```bash
   python 执行对比流程.py
   ```

### 方式2：使用代码中的变量（单场景，向后兼容）

如果 `CONFIG_JSON_FILE` 设置为 `None`，则使用代码中的变量配置（单场景模式）。

## JSON配置说明

### 场景配置字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 场景名称（用于日志显示） |
| `enabled` | boolean | 否 | 是否启用此场景（默认：true） |
| `description` | string | 否 | 场景描述 |
| `input_csv_file` | string | 是 | 输入CSV文件名（相对于脚本目录） |
| `output_file_prefix` | string | 否 | 输出文件名前缀（例如："qx_0108"），如果为空字符串则不添加前缀 |
| `api_url` | string | 是 | 接口URL地址 |
| `thread_count` | integer | 否 | 并发线程数（如果不设置则使用全局默认值） |
| `timeout` | integer | 否 | 请求超时时间，单位：秒（如果不设置则使用全局默认值） |
| `auto_detect_columns` | boolean | 否 | 是否自动检测列索引（如果不设置则使用全局默认值） |
| `column_config` | object | 否 | 列索引配置（仅在 `auto_detect_columns=false` 时使用） |
| `column_config.cust_no_column` | integer | 否 | cust_no所在列索引（从0开始） |
| `column_config.use_create_time_column` | integer | 否 | 时间字段所在列索引（从0开始） |
| `column_config.feature_start_column` | integer | 否 | 特征开始列索引（从0开始） |

### 全局配置字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `default_thread_count` | integer | 默认并发线程数（场景中未指定时使用） |
| `default_timeout` | integer | 默认超时时间（场景中未指定时使用） |
| `default_auto_detect_columns` | boolean | 默认是否自动检测列索引（场景中未指定时使用） |

## 输出文件命名规则

输出文件名格式：`前缀_基础名_时间戳_类型.csv`

- **前缀**：由 `output_file_prefix` 配置（如果为空则不添加）
- **基础名**：输入文件名（去掉扩展名）
- **时间戳**：当前时间的 `MMDDHHmm` 格式（例如：`01081550` 表示1月8日15:50）
- **类型**：`api_data` 或 `compare`

### 示例

假设：
- 输入文件：`qx_0108_sqldata.csv`
- 前缀：`qx_0108`
- 执行时间：2026年1月8日 15:50:12

生成的文件：
- `qx_0108_qx_0108_sqldata_01081550_api_data.csv`
- `qx_0108_qx_0108_sqldata_01081550_compare.csv`
- `qx_0108_qx_0108_sqldata_01081550_compare_analysis_report.csv`
- `qx_0108_qx_0108_sqldata_01081550_compare_feature_stats.csv`

## 列索引自动检测

当 `auto_detect_columns` 为 `true` 时，脚本会自动检测：
- `cust_no` 列（支持：`cust_no`, `custNo`, `CUST_NO` 等）
- 时间字段列（支持：`use_create_time`, `ua_time`, `create_time` 等）

检测结果会保存到 `列索引配置_场景名.json` 文件中。

## 注意事项

1. 同一时间段内执行的所有场景会使用相同的时间戳后缀
2. 每个场景的列索引配置会单独保存
3. 如果某个场景执行失败，会继续执行下一个场景
4. 可以通过设置 `enabled: false` 来临时禁用某个场景
