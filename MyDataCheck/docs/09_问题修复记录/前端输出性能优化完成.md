## 问题描述

执行接口数据对比时，第一条数据请求并返回后，Web前端页面卡住不动，整个页面无法操作。

### 触发条件
- 线程数较多（100+）
- CSV文件行数较多（1000+）
- 每行都有输出
- 总输出行数超过5000行

---

## 问题原因

**DOM元素无限增长**

### 原始代码
```javascript
function appendOutput(message, type = 'output', tab = 'api') {
    const outputPanel = document.getElementById(`output-panel-${tab}`);
    const line = document.createElement('div');
    line.textContent = message;
    outputPanel.appendChild(line);  // ❌ 无限增长
    outputPanel.scrollTop = outputPanel.scrollHeight;
}
```

### 问题分析
1. **DOM元素无限增长**：每条输出都创建一个新的div元素
2. **内存占用增加**：大量DOM元素占用内存
3. **渲染性能下降**：浏览器需要渲染和重排大量元素
4. **页面卡死**：当输出超过几千行时，页面完全卡住

### 实际影响
- 输出1000行：页面开始卡顿
- 输出5000行：页面严重卡顿
- 输出10000行：页面完全卡死，无法操作

---

## 优化方案

### 核心策略
1. **限制最大输出行数**：最多保留1000行
2. **智能采样**：输出多时自动降低显示频率
3. **保留关键信息**：错误、成功、进度等重要信息总是显示

### 优化后的代码
```javascript
// 配置常量
const MAX_OUTPUT_LINES = 1000;  // 最多保留1000行
const SAMPLE_RATE = 5;          // 采样率（每5条显示1条）
let outputCounters = {};        // 每个tab的输出计数器

function appendOutput(message, type = 'output', tab = 'api') {
    // 初始化计数器
    if (!outputCounters[tab]) {
        outputCounters[tab] = 0;
    }
    outputCounters[tab]++;
    
    // 判断是否为重要信息（总是显示）
    const isImportant = message.includes('错误') || 
                       message.includes('❌') ||
                       message.includes('成功') || 
                       message.includes('✅') ||
                       message.includes('完成') ||
                       message.includes('开始') ||
                       message.includes('警告') ||
                       message.includes('⚠️') ||
                       message.includes('[') ||  // 进度信息 [1/100]
                       message.includes('=');    // 分隔线
    
    const outputPanel = document.getElementById(`output-panel-${tab}`);
    const currentLines = outputPanel.querySelectorAll('.output-line').length;
    
    // 如果输出行数超过500，开始采样
    const shouldSample = currentLines > 500;
    const shouldDisplay = !shouldSample || 
                         (outputCounters[tab] % SAMPLE_RATE === 0) || 
                         isImportant;
    
    if (shouldDisplay) {
        const line = document.createElement('div');
        line.className = 'output-line';
        
        // 添加类型样式
        if (message.includes('错误') || message.includes('❌') || message.includes('失败')) {
            line.className += ' error';
        } else if (message.includes('成功') || message.includes('✅') || message.includes('完成')) {
            line.className += ' success';
        } else if (message.includes('警告') || message.includes('⚠️')) {
            line.className += ' warning';
        } else {
            line.className += ' info';
        }
        
        line.textContent = message;
        outputPanel.appendChild(line);
        
        // 限制最大行数
        const lines = outputPanel.querySelectorAll('.output-line');
        if (lines.length > MAX_OUTPUT_LINES) {
            const removeCount = lines.length - MAX_OUTPUT_LINES;
            for (let i = 0; i < removeCount; i++) {
                lines[i].remove();
            }
        }
        
        // 滚动到底部（防抖）
        if (outputPanel._scrollTimeout) {
            clearTimeout(outputPanel._scrollTimeout);
        }
        outputPanel._scrollTimeout = setTimeout(() => {
            outputPanel.scrollTop = outputPanel.scrollHeight;
        }, 50);
    }
}
```

## 经验总结

### 问题根源
前端性能问题通常由以下原因导致：
1. DOM元素过多
2. 频繁的DOM操作
3. 大量的重排和重绘
4. 内存泄漏

### 解决思路
1. **限制数量**：控制DOM元素数量
2. **智能过滤**：只显示必要的信息
3. **批量操作**：减少DOM操作次数
4. **防抖节流**：降低操作频率

### 最佳实践
1. 始终考虑最坏情况（海量数据）
2. 添加性能监控和限制
3. 保留关键信息，过滤冗余信息
4. 提供配置选项，适应不同场景
