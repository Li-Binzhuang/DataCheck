### 3. 后端路由 (web/routes/compare_routes.py)

#### 新增路由1: 保存配置
```python
@compare_bp.route('/api/compare/decimal/config/save', methods=['POST'])
def save_decimal_config():
    """保存小数处理配置"""
    config_data = request.json
    config_path = os.path.join(data_comparison_dir, "decimal_config.json")
    
    with open(config_path, 'w', encoding='utf-8') as f:
        json.dump(config_data, f, ensure_ascii=False, indent=2)
    
    return jsonify({'success': True, 'message': '✅ 配置保存成功'})
```

#### 新增路由2: 加载配置
```python
@compare_bp.route('/api/compare/decimal/config/load', methods=['GET'])
def load_decimal_config():
    """加载小数处理配置"""
    config_path = os.path.join(data_comparison_dir, "decimal_config.json")
    
    if not os.path.exists(config_path):
        return jsonify({'success': False, 'error': '配置文件不存在'})
    
    with open(config_path, 'r', encoding='utf-8') as f:
        config = json.load(f)
    
    return jsonify({'success': True, 'config': config})
```

## 📊 配置文件格式

### 文件位置
`data_comparison/decimal_config.json`

### 配置结构
```json
{
  "file_path": "PATH:/data/diff.csv",
  "source_column": "API值",
  "reference_column": "CSV值",
  "decimal_method": "round",
  "compare_mode": "exact",
  "tolerance_value": 0.01,
  "output_prefix": "decimal_processed",
  "output_full_data": false,
  "file_input_mode": "path"
}
```

### 配置字段说明

| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| file_path | string | 文件路径 | 任意路径 |
| source_column | string | 源列名称 | 任意列名 |
| reference_column | string | 参考列名称 | 任意列名 |
| decimal_method | string | 小数处理方式 | none, round, double_round, truncate, ceil |
| compare_mode | string | 对比方式 | exact, tolerance, last_digit, last_digit_2 |
| tolerance_value | number | 容差值 | 任意正数 |
| output_prefix | string | 输出文件前缀 | 任意字符串 |
| output_full_data | boolean | 输出全量数据 | true, false |
| file_input_mode | string | 文件输入模式 | upload, path |

### 技巧2：团队共享配置
- 配置文件保存在服务器
- 团队成员可以共享同一个配置
- 适合标准化的处理流程

### 技巧3：配置模板
- 为常用的处理任务创建配置模板

## ⚠️ 注意事项

1. **配置文件共享**
   - 当前所有用户共享同一个配置文件
   - 后续保存会覆盖前面的配置
   - 如需多用户独立配置，需要扩展实现

2. **文件路径有效性**
   - 配置中保存的文件路径可能失效
   - 自动加载时如果文件不存在，需要重新上传

3. **自动加载行为**
   - 页面加载时自动加载配置
   - 不会覆盖用户已上传的文件
   - 只有点击"加载配置"按钮才会强制覆盖
