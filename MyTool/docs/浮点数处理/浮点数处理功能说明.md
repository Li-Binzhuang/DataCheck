# CDC板块衍生脚本 - 浮点数精度处理说明

## 修改日期
2026-01-28

## 问题描述
在检查四个板块衍生脚本时，发现BOSS板块在输出CSV文件时**没有对浮点数进行精度控制**，导致特征值可能被截断或显示不完整。

例如：`cdc_boss_num_pagos_mean_607` 这类均值特征，如果不做精度控制，可能只显示2位小数。

## 解决方案

### 统一处理逻辑
所有四个板块现在都使用相同的浮点数精度处理逻辑：

```python
# zlf update: 对数值特征列保留6位小数
_features_to_write = features_df.copy()
_round_cols = [c for c in _features_to_write.columns if c not in {"apply_id", "request_time"}]
_features_to_write[_round_cols] = _features_to_write[_round_cols].apply(pd.to_numeric, errors="coerce").round(6)
_features_to_write.to_csv(csv_path, index=False, encoding="utf-8-sig")
```

### 处理规则
1. **排除字段**：`apply_id` 和 `request_time` 不做精度处理
2. **数值转换**：使用 `pd.to_numeric(errors="coerce")` 确保列为数值类型
3. **精度控制**：统一保留 **6位小数** (`round(6)`)
4. **应用范围**：全量数据和sample200样本都使用相同处理

## 注意事项

1. **重新运行脚本**：修改后需要重新运行BOSS板块衍生脚本才能生成新的输出文件
2. **数据一致性**：确保四个板块的输出精度保持一致
3. **下游影响**：如果下游系统对精度有特殊要求，需要相应调整
4. **性能影响**：`.round(6)` 操作对性能影响很小，可以忽略

## 与空值填充的关系

本次修改与之前的空值填充修改（fillna(-999)）是独立的：
- **空值填充**：处理缺失值，填充为 -999
- **精度控制**：处理有效数值，保留6位小数

两者配合使用，确保输出数据的质量和一致性。
