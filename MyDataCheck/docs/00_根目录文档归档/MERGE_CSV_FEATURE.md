# CSV文件合并功能说明

## 功能概述

在"小工具集合"菜单下新增了"合并表格文件"功能，支持上传多个CSV文件并合并为一个CSV文件。

## 功能特性

### 1. 合并方式

- **纵向合并（追加行）**：将多个文件的行拼接在一起
  - 要求：所有文件的列结构必须相同
  - 适用场景：合并多个结构相同的数据文件
  
- **横向合并（追加列）**：将多个文件的列拼接在一起
  - 支持自动排除重复列
  - 可指定要排除的列名，或自动检测所有重复列
  - 适用场景：合并不同来源的特征数据

### 2. 重复列处理（横向合并专用）

- **自动移除重复列**：勾选后，横向合并时会自动处理重复列
  - 保留第一个文件中的列
  - 从后续文件中移除重复的列
  
- **指定重复列名**：
  - 留空：自动检测所有与第一个文件重复的列名
  - 填写：只移除指定的列（多个列名用逗号分隔）
  - 示例：`user_id,order_id` 表示只移除这两列

### 3. 文件上传

- 支持多文件选择（按住Ctrl或Cmd多选）
- 仅支持CSV格式文件
- 至少需要选择2个文件进行合并

### 3. 输出配置

- 可自定义输出文件名前缀
- 自动添加时间戳后缀（格式：YYYYMMDD_HHMMSS）
- 输出文件保存在 `outputdata/merge_csv/` 目录
- 合并完成后自动下载结果文件

## 使用步骤

1. 在左侧菜单点击"小工具集合" → "合并表格文件"
2. 点击"选择CSV文件"，按住Ctrl或Cmd选择多个CSV文件
3. 选择合并方式：
   - **纵向合并**：适合合并相同结构的数据
   - **横向合并**：适合合并不同特征列
4. 如果选择横向合并：
   - 勾选"自动移除重复列"（推荐）
   - 可选：指定要移除的列名（如 `user_id,order_id`），留空则自动检测
5. 输入输出文件名（可选，默认为"merged"）
6. 点击"执行合并"按钮
7. 等待处理完成，文件会自动下载

## 注意事项

1. **纵向合并**：
   - 所有文件必须有相同的列名和列数
   - 如果列名不同，合并后会包含所有列，缺失值填充为NaN

2. **横向合并**：
   - 建议勾选"自动移除重复列"，避免列名冲突
   - 如果不移除重复列，pandas会自动添加后缀（_x, _y等）
   - 指定列名时，只会移除指定的列，其他重复列仍会保留

3. **重复列处理逻辑**：
   - 始终保留第一个文件中的列
   - 从第2、3、4...个文件中移除重复列
   - 示例：3个文件都有`user_id`列，最终只保留第1个文件的`user_id`

4. **文件大小**：
   - 受Flask配置的最大上传限制（当前为5GB）
   - 大文件合并可能需要较长时间

5. **编码格式**：
   - 输出文件使用UTF-8-BOM编码，确保Excel正确显示中文

## 示例场景

### 场景1：合并多天的数据
```
文件1: data_20240301.csv (1000行 × 10列)
文件2: data_20240302.csv (1000行 × 10列)
文件3: data_20240303.csv (1000行 × 10列)

选择：纵向合并
结果：merged_20240309_143022.csv (3000行 × 10列)
```

### 场景2：合并不同特征源（自动排除重复列）
```
文件1: user_basic.csv (1000行 × 5列：user_id、姓名、年龄、性别、城市)
文件2: user_features.csv (1000行 × 21列：user_id、特征1-20)

配置：
- 选择：横向合并
- 勾选：自动移除重复列
- 指定列名：留空（自动检测）

结果：merged_20240309_143022.csv (1000行 × 25列)
说明：第2个文件的user_id列被自动移除，避免重复
```

### 场景3：合并多个特征表（指定排除列）
```
文件1: base.csv (1000行 × 3列：user_id、order_id、amount)
文件2: features_a.csv (1000行 × 12列：user_id、order_id、特征A1-10)
文件3: features_b.csv (1000行 × 12列：user_id、order_id、特征B1-10)

配置：
- 选择：横向合并
- 勾选：自动移除重复列
- 指定列名：user_id,order_id

结果：merged_20240309_143022.csv (1000行 × 23列)
说明：文件2和文件3的user_id、order_id列被移除，只保留文件1的
```
