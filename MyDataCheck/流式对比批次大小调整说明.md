# 流式对比批次大小调整说明

## 🎯 调整内容

将接口数据对比中的流式对比批次大小从 **每批500条** 调整为 **每批50条**，并将进度显示频率从 **每10批显示一次** 调整为 **每批都显示**。

## 📋 修改详情

### 1. 批次大小调整

**修改前**: 每批处理 1000 条数据  
**修改后**: 每批处理 50 条数据

### 2. 进度显示频率调整

**修改前**: 每 10 个批次显示一次进度  
**修改后**: 每个批次都显示进度（每 50 条显示一次）

## 🔧 修改的文件

### 1. `MyDataCheck/api_comparison/job/streaming_comparator.py`

**修改位置 1**: 第 48 行 - 默认批次大小
```python
# 修改前
batch_size: int = 1000,  # 批次大小

# 修改后
batch_size: int = 50,  # 批次大小（每批50条）
```

**修改位置 2**: 第 332-335 行 - 进度显示逻辑
```python
# 修改前
# 每10个批次或最后一个批次显示进度
if batch_idx % 10 == 0 or batch_idx == 1:
    total_batches = (len(rows) + self.batch_size - 1) // self.batch_size
    print(f"处理批次 {batch_idx}/{total_batches}: {len(batch)} 行")

# 修改后
# 每个批次都显示进度（每50条显示一次）
total_batches = (len(rows) + self.batch_size - 1) // self.batch_size
print(f"处理批次 {batch_idx}/{total_batches}: {len(batch)} 行")
```

### 2. `MyDataCheck/api_comparison/job/process_executor.py`

**修改位置 1**: 第 200 行 - 函数默认参数
```python
# 修改前
batch_size: int = 1000):

# 修改后
batch_size: int = 50):
```

**修改位置 2**: 第 421 行 - 调用时的参数
```python
# 修改前
batch_size=1000  # 批次大小

# 修改后
batch_size=50  # 批次大小（每批50条）
```

## ✨ 优势

### 1. 更细粒度的进度反馈

- **修改前**: 每 10,000 条才显示一次进度（10 批 × 1000 条）
- **修改后**: 每 50 条就显示一次进度
- **提升**: 进度反馈频率提升 **200 倍**

### 2. 更好的用户体验

- 用户可以实时看到处理进度
- 不会长时间没有反馈
- 更容易判断程序是否正常运行

### 3. 更容易发现问题

- 如果某批数据处理失败，可以更快发现
- 更精确地定位问题数据的位置

## ⚠️ 注意事项

### 1. 性能影响

- **批次大小减小**: 从 1000 → 50
- **批次数量增加**: 增加 20 倍
- **进度输出增加**: 增加 200 倍

**性能影响评估**:
- ✅ 接口请求性能: **无影响**（仍然使用线程池并发）
- ✅ 数据处理性能: **无影响**（处理逻辑不变）
- ⚠️ 控制台输出: **略有增加**（但可忽略不计）

### 2. 内存占用

- **修改前**: 每批在内存中保持 1000 条数据
- **修改后**: 每批在内存中保持 50 条数据
- **优势**: 内存占用进一步降低 **95%**

### 3. 适用场景

✅ **适合**:
- 需要实时查看进度的场景
- 数据量较大的场景（几千到几万条）
- 需要精确定位问题的场景

❌ **不适合**:
- 如果觉得进度输出太频繁，可以调整批次大小
- 如果需要更快的处理速度（但实际影响很小）

## 🔄 如何调整批次大小

如果需要调整批次大小，可以修改以下位置：

### 方法 1: 修改默认值（推荐）

修改 `streaming_comparator.py` 第 48 行：
```python
batch_size: int = 50,  # 改为你想要的批次大小
```

### 方法 2: 修改调用参数

修改 `process_executor.py` 第 421 行：
```python
batch_size=50  # 改为你想要的批次大小
```

### 推荐的批次大小

| 数据量 | 推荐批次大小 | 说明 |
|--------|-------------|------|
| < 1000 | 50 | 实时反馈 |
| 1000-5000 | 50-100 | 平衡反馈和性能 |
| 5000-10000 | 100-200 | 适度反馈 |
| > 10000 | 200-500 | 减少输出 |
