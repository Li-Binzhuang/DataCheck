# 授信申请表分析报告

## 一、表概述

**表名**: `ods_mx_aprv_approve_credit_apply_df` (授信申请表)

**业务定位**: 授信申请表是业务流程的起点，记录客户的授信申请信息，是整个贷款业务流程的第一个环节。

**业务流程位置**:
```
授信申请 → 授信产品 → 用信申请 → 资产申请(借款申请) → 额度 → 借据(贷款)
```

## 十一、业务流程理解

### 11.1 授信申请流程
1. **申请提交**: 客户通过Android平台提交授信申请
2. **规则执行**: 系统执行CREDIT_APPLY规则
3. **审核审批**: 经过审核（audit_status=PASS）和审批（approve_state=CYCLE_PASS）
4. **状态更新**: 状态更新为400（Success）
5. **有效期设置**: 设置有效期结束日期（通常为15天）
6. **产品配置**: 在授信产品表中生成对应的产品配置

### 11.2 状态流转路径
```
100 (INIT) → 200 (Processing) → 400 (Success) 或 300 (Fail)
```

当前数据集中只看到最终状态400（Success），说明：
- 数据可能是最终状态的快照
- 或者只导出了成功通过的申请
