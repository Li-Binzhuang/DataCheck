# 忽略0和NaN的差异 - 快速开始

## 一句话说明
在 CSV 数据对比时，勾选此选项可以将 `0`、`NaN`、空格 和其他空值表示视为相同的值。

## 在哪里找到这个选项？

1. 打开 Web 界面
2. 左侧菜单 → **CSV数据对比**
3. 配置面板中找到：
   - ✅ 忽略默认填充值
   - ✅ **忽略0和NaN的差异** ← 新选项在这里
   - ✅ 忽略小数差异

## 何时使用？

### ✅ 应该勾选的情况

- 数据中 `0`、`NaN`、空格 表示同一个含义（如都表示缺失）
- 不同系统对缺失值的表示方式不同
- 只关心实质性的数据差异，不关心 `0` vs `NaN` vs 空格
- 特征值为 `0` 和 `" "` (空格) 的差异

### ❌ 不应该勾选的情况

- 需要严格区分 `0`、`NaN` 和空格
- 这些值在业务上有不同含义
- 需要精确的数据对比

## 效果对比

### 未勾选（默认）
```
CSV值: 0      API值: NaN     → ❌ 差异
CSV值: 0      API值: (空格)  → ❌ 差异
CSV值: 0      API值: 0       → ✅ 一致
CSV值: NaN    API值: NaN     → ✅ 一致
```

### 勾选后
```
CSV值: 0      API值: NaN     → ✅ 一致 ← 变化
CSV值: 0      API值: (空格)  → ✅ 一致 ← 变化（关键修复）
CSV值: 0      API值: 0       → ✅ 一致
CSV值: NaN    API值: NaN     → ✅ 一致
```

## 支持的等价值

启用此选项后，以下值被视为等价：

| 类型 | 值 | 说明 |
|------|-----|------|
| 数值 | `0`, `0.0` | 数字零 |
| 字符串 | `"0"` | 字符串零 |
| 空值 | `None`, `""`, `" "`, `"  "` | 各种空值表示 |
| 字符串 | `"null"`, `"none"` | 字符串空值 |
| NaN | `float('nan')` | 浮点数 NaN |
| 字符串 | `"NaN"`, `"nan"`, `"NA"`, `"N/A"` | 字符串 NaN 表示 |

## 与其他选项的区别

| 选项 | 处理 | 例子 |
|------|------|------|
| 忽略默认填充值 | `-999` 和 `null` | `-999` ≈ `null` |
| **忽略0和NaN的差异** | **`0`、`NaN`、空格** | **`0` ≈ `NaN` ≈ `" "`** |
| 忽略小数差异 | 浮点数精度 | `8.0001` ≈ `8.0002` |

## 配置保存

勾选此选项后，点击"💾 保存配置"可以保存设置。

下次加载配置时，此选项会自动恢复。

## 常见问题

**Q: 勾选此选项会影响 `null` 吗？**
A: 会。`null` 现在被视为与 `0` 和 `NaN` 等价。如需更精细的控制，请使用"忽略默认填充值"。

**Q: 可以和其他选项一起使用吗？**
A: 可以。可以同时勾选多个选项，它们互不影响。

**Q: 字符串 "NaN" 会被处理吗？**
A: 会。字符串 `"NaN"`、`"nan"`、`"NA"`、`"N/A"` 都会被转换为 NaN 后进行比较。

**Q: 空格会被处理吗？**
A: 会。空格 `" "`、多个空格 `"  "` 都会被视为空值。

**Q: 性能会受影响吗？**
A: 几乎没有影响。只是增加了简单的字符串检查。

## 使用示例

### 场景：处理不同系统的缺失值表示

**问题**: 
- CSV 文件中用 `0` 表示缺失
- API 返回用空格 `" "` 表示缺失
- 对比时出现大量"差异"

**解决方案**:
1. 勾选"忽略0和NaN的差异"
2. 执行对比
3. 这些差异会被忽略，只显示真正的差异
