# Web界面使用说明

## 概述

Web界面提供了一个友好的图形化界面，用于输入执行配置和执行数据对比流程。

## 安装依赖

```bash
pip install -r requirements.txt
```

## 启动服务

```bash
python web_app.py
```

启动后，访问 `http://localhost:5001` 即可使用Web界面。

**注意**：默认端口为5001（避免macOS上AirPlay Receiver占用5000端口）。如果需要使用其他端口，可以使用：
```bash
python web_app.py --port 8080
```

## 功能说明

### 1. 加载配置

点击 **"📥 加载配置"** 按钮，会自动从 `执行配置.json` 文件加载当前配置到编辑框中。

### 2. 编辑配置

在配置编辑框中，可以直接编辑JSON配置内容。配置格式与 `执行配置.json` 文件格式相同。

### 3. 保存配置

点击 **"💾 保存配置"** 按钮，会将当前编辑框中的配置保存到 `执行配置.json` 文件。

### 4. 执行对比

点击 **"▶️ 执行对比"** 按钮，会使用当前配置执行数据对比流程。执行过程中：

- 状态指示器会显示执行状态（运行中/成功/失败）
- 输出面板会实时显示执行日志
- 执行完成后会显示总结信息

### 5. 清空输出

点击 **"🗑️ 清空输出"** 按钮，会清空输出面板中的内容。

## 配置格式

配置格式与 `执行配置.json` 文件格式相同，包含以下主要部分：

- `scenarios`: 场景配置列表
  - `name`: 场景名称
  - `enabled`: 是否启用
  - `input_csv_file`: 输入CSV文件
  - `api_url`: API接口地址
  - `thread_count`: 线程数
  - `timeout`: 超时时间
  - `convert_feature_to_number`: 是否转换特征值为数值
  - `column_config`: 列配置
    - `cust_no_column`: 客户号列索引
    - `use_create_time_column`: 时间列索引
    - `feature_start_column`: 特征起始列索引
- `global_config`: 全局配置
  - `default_thread_count`: 默认线程数
  - `default_timeout`: 默认超时时间
  - `default_convert_feature_to_number`: 默认是否转换特征值为数值

## 注意事项

1. 执行过程中请勿关闭浏览器或停止服务
2. 配置JSON格式必须正确，否则无法执行
3. 执行时间取决于数据量和网络情况
4. 输出面板会自动滚动到最新内容

## 故障排除

### 问题：无法加载配置

**解决方案**：检查 `执行配置.json` 文件是否存在且格式正确。

### 问题：执行失败

**解决方案**：
1. 检查配置中的文件路径是否正确
2. 检查API接口地址是否可访问
3. 查看输出面板中的错误信息

### 问题：输出不显示

**解决方案**：
1. 检查浏览器控制台是否有错误
2. 尝试刷新页面重新执行
3. 检查网络连接是否正常
